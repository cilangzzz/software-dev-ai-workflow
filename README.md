# 软件开发工作流 AI.SKILL

> *"让AI成为软件开发流程中的智能伙伴，从需求到运维的全流程赋能！*

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Claude Code](https://img.shields.io/badge/Claude%20Code-Skill-blueviolet)](https://claude.ai/code)
[![AgentSkills](https://img.shields.io/badge/AgentSkills-Standard-green)](https://agentskills.io)

<br>

**一套 AI 驱动的软件开发工作流框架**，将软件工程从需求到运维的每个环节都交给 AI Agent 协作完成。你只需要描述想做什么，AI 就能：

- **自动生成产品方案**：PRD、用户故事、验收标准（Gherkin）、设计系统文档
- **自动输出研发方案**：技术选型报告、架构设计（ADR）、数据模型、各模块功能实现方案
- **自动编写代码**：基于方案直接生成完整可用的项目代码（Vue / React / Java / Go 等）
- **自动编写教程**：从零到一的开发教程、快速入门指南

**核心理念**：人类负责定义目标和决策，AI 负责执行和产出——用标准化的 Agent 角色和 Skill 工具，让 AI 贯穿软件开发全流程。

## 流程使用方式

### 快速开始

```
1. 选择角色Agent → 从 1.0-软件开发流程角色agent模型/ 选择对应角色
2. 确定生命周期阶段 → 参考 2.0-软件开发流程生命周期/
3. 获取产出模板 → 使用 3.0-软件开发流程产出/ 中的模板
4. 执行质量门控 → 通过 5.0-系统模型/ 中的质量检查
5. 输出结果 → 生成至 output/ 目录
```

### 各阶段产出物

| 阶段 | 角色 | 产出物 | 参考目录 |
|------|------|--------|----------|
| 需求分析 | 产品Agent | PRD、用户故事、验收标准 | `3.0-软件开发流程产出/产品/` |
| 架构设计 | 研发Agent | 架构方案、ADR、技术选型 | `3.0-软件开发流程产出/研发/` |
| 代码开发 | 研发Agent | 源码、单元测试、API文档 | `6.0-基础开发系统/` |
| 测试验证 | 测试Agent | 测试用例、测试报告、缺陷分析 | `3.0-软件开发流程产出/测试/` |
| 部署上线 | 运维Agent | 部署方案、回滚预案、监控配置 | `3.0-软件开发流程产出/运维/` |

### 使用示例

1. **需求分析阶段**
   - 加载 `1.0-软件开发流程角色agent模型/产品/` 中的产品Agent
   - 输入需求描述，AI自动生成结构化PRD
   - 产出物存放至 `output/产品/`

2. **架构设计阶段**
   - 加载 `1.0-软件开发流程角色agent模型/研发/` 中的架构Agent
   - 输入PRD，AI生成架构方案和ADR
   - 参考 `5.0-系统模型/` 中对应系统的架构模板

3. **测试验证阶段**
   - 加载 `1.0-软件开发流程角色agent模型/测试/` 中的测试Agent
   - 输入功能规格，AI生成测试用例和测试报告
   - 通过质量门控检查后进入下一阶段

### MES制造执行系统

> [yudao-aisk-mes](https://github.com/cilangzzz/yudao-aisk-mes) - 基于 AI.SKILL 工作流生成的 MES 系统示例

该示例展示了如何使用本工作流完整地构建一个制造执行系统（MES），包括：

- **需求阶段产出物**：PRD文档、用户故事、验收标准
- **设计阶段产出物**：系统架构设计、数据库设计、API设计
- **开发阶段产出物**：基于 yudao-cloud 框架的后端源码
- **测试阶段产出物**：测试用例、测试报告
- **部署阶段产出物**：部署方案、运维文档

技术栈：
- 后端框架：yudao-cloud（Spring Cloud Alibaba）
- 前端框架：Vue3 + Element Plus
- 数据库：MySQL + Redis

## 质量门控体系

```
Gate 1: 需求评审 → 完整性检查、冲突检测
Gate 2: 设计验收 → 设计规范检查
Gate 3: 代码审查 → 代码质量、安全扫描
Gate 4: 测试验收 → 功能测试通过
Gate 5: 发布就绪 → 部署方案、回滚预案
```

## 协作接口

### 产品→研发
- PRD文档 + 用户故事 + 验收标准

### 研发→测试
- 可测试版本 + 变更说明 + API文档

### 测试→运维
- 测试报告 + 回滚方案
