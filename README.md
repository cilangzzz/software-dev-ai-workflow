# AI驱动软件开发工作流

一套完整的AI驱动软件开发工作流Skill集合，覆盖从需求到运维的全流程。

## 项目简介

本项目旨在通过AI能力辅助软件开发全流程，提升研发效率和质量。包含产品设计、研发开发、测试验证、运维部署、安全审计、数据分析等多个部门的AI Skill。

## Skill清单

### 产品部 (3个P0 Skill)
| Skill | 功能 |
|-------|------|
| `requirement-analyzer` | 需求分析，生成结构化PRD框架 |
| `user-story-generator` | 用户故事生成，INVEST标准 |
| `acceptance-criteria-writer` | 验收标准编写，Gherkin格式 |

### 研发部 (5个P0 Skill)
| Skill | 功能 |
|-------|------|
| `architect` | 架构设计，生成架构方案+ADR |
| `adr` | 架构决策记录管理 |
| `scaffold` | 项目脚手架生成 |
| `implement` | 功能实现，代码+测试生成 |
| `code-review` | 代码审查，质量+安全检查 |

### 测试部 (3个P0 Skill)
| Skill | 功能 |
|-------|------|
| `test-case-generator` | 测试用例自动生成 |
| `test-executor` | 测试执行和报告生成 |
| `bug-analyzer` | 缺陷分析和根因定位 |

### 运维部 (2个P0 Skill)
| Skill | 功能 |
|-------|------|
| `deploy-analyzer` | 部署风险评估和计划生成 |
| `ci-pipeline-assistant` | CI/CD故障诊断和优化 |

### 设计部 (2个P0 Skill)
| Skill | 功能 |
|-------|------|
| `design-system-analyzer` | 设计系统一致性检查 |
| `design-to-code` | 设计稿转前端代码 |

### 安全部 (3个P0 Skill)
| Skill | 功能 |
|-------|------|
| `security-threat-model` | 威胁建模，STRIDE分析 |
| `security-code-review` | 安全代码审查 |
| `security-scan` | 安全扫描集成 |

### 数据部 (2个P0 Skill)
| Skill | 功能 |
|-------|------|
| `data-quality-check` | 数据质量检查和异常检测 |
| `data-lineage-trace` | 数据血缘追踪和影响分析 |

### 项目管理 (2个P0 Skill)
| Skill | 功能 |
|-------|------|
| `project-status` | 项目状态报告生成 |
| `gate-check` | 阶段质量门控检查 |

### 文档管理 (1个P0 Skill)
| Skill | 功能 |
|-------|------|
| `readme-generator` | README文档自动生成 |

## 使用方式

每个Skill文件包含完整的定义：
- 触发条件和命令
- 输入参数说明
- 执行流程
- 输出格式示例
- 质量标准
- 使用示例

将Skill文件配置到对应的AI工具中即可使用。

## 质量门控体系

```
Gate 1: 需求评审 → 完整性检查、冲突检测
Gate 2: 设计验收 → 设计规范检查
Gate 3: 代码审查 → 代码质量、安全扫描
Gate 4: 测试验收 → 功能测试通过
Gate 5: 发布就绪 → 部署方案、回滚预案
```

## 协作接口

### 产品→研发
- PRD文档 + 用户故事 + 验收标准

### 研发→测试
- 可测试版本 + 变更说明 + API文档

### 测试→运维
- 测试报告 + 回滚方案
