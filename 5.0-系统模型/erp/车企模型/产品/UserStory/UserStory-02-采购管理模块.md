## 2. 采购申请管理用户故事

### US-PR-001: 创建采购申请单

**作为** 采购员
**我想要** 创建采购申请单
**以便于** 发起采购需求并进入审批流程

**用户故事描述**:
采购员根据生产需求或库存情况创建采购申请单，填写物料信息、需求数量、需求日期等关键信息，系统自动计算申请金额并触发相应的审批流程。

**验收条件**:
- [ ] 可以选择物料并自动带出物料基础信息
- [ ] 可以填写需求数量、需求日期、申请原因
- [ ] 系统自动根据价格信息计算申请金额
- [ ] 可以标记是否为紧急申请
- [ ] 申请单保存后自动生成唯一申请单号
- [ ] 支持添加多个物料明细行
- [ ] 必填字段未填写时给出提示

**优先级**: P0
**故事点**: 5
**所属迭代**: Sprint 1

---

### US-PR-002: 审批采购申请

**作为** 采购主管/经理
**我想要** 审批采购申请单
**以便于** 控制采购成本并确保采购合规

**用户故事描述**:
审批人在待办事项中收到审批任务，可以查看申请详情、历史记录、预算情况，并做出批准或驳回决定。

**验收条件**:
- [ ] 审批人能在待办列表看到待审批的申请
- [ ] 可以查看申请单完整信息和物料明细
- [ ] 可以查看申请人的历史申请记录
- [ ] 可以查看预算执行情况
- [ ] 驳回时必须填写驳回原因
- [ ] 审批后自动流转到下一节点或结束流程
- [ ] 审批结果通过消息通知申请人

**优先级**: P0
**故事点**: 3
**所属迭代**: Sprint 1

---

### US-PR-003: 导入MRP采购需求

**作为** 生产计划员
**我想要** 将MRP运算结果导入采购系统
**以便于** 自动生成采购申请并提高效率

**用户故事描述**:

**验收条件**:
- [ ] 支持Excel格式的MRP需求文件导入
- [ ] 导入前进行数据格式校验
- [ ] 自动匹配物料编码和供应商
- [ ] 显示导入结果和失败明细
- [ ] 支持按MRP批次生成申请单
- [ ] 可以预览导入数据后再确认

**优先级**: P0
**故事点**: 8
**所属迭代**: Sprint 2

---

### US-PR-004: 合并采购申请

**作为** 采购员
**我想要** 合并多个采购申请
**以便于** 减少采购订单数量并提高议价能力

**用户故事描述**:
采购员可以将相同供应商、相同交货日期的多个采购申请合并处理，生成一个采购订单，提高采购效率。

**验收条件**:
- [ ] 可以按条件筛选可合并的申请单
- [ ] 系统自动推荐可合并的申请组合
- [ ] 支持手工选择需要合并的申请
- [ ] 合并后汇总数量和金额
- [ ] 保留原申请单的关联关系
- [ ] 合并操作需要审批确认

**优先级**: P1
**故事点**: 5
**所属迭代**: Sprint 3

- [ ] 可以查看收货和入库情况
