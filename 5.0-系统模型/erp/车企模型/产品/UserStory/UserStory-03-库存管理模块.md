## 1. 文档概述

### 1.1 目的

本文档描述库存管理模块的用户故事，从用户视角出发，详细说明各角色的业务需求和期望价值。

### 1.2 用户角色定义

| 角色ID | 角色名称 | 角色描述 | 主要职责 |
|--------|----------|----------|----------|
| R01 | 仓库管理员 | 负责日常仓库操作的人员 | 出入库操作、库存盘点、库位维护 |
| R02 | 仓库主管 | 仓库管理人员 | 库存监控、预警处理、报表分析 |
| R03 | 采购人员 | 负责采购业务的业务人员 | 采购入库跟踪、VMI库存管理 |
| R04 | 生产计划员 | 负责生产计划的人员 | 物料需求查看、缺料预警 |
| R05 | 生产领料员 | 生产线物料领取人员 | 领料申请、物料交接 |
| R06 | 质检人员 | 负责质量检验的人员 | 入库质检、库存质检 |
| R07 | 财务人员 | 负责财务核算的人员 | 库存成本核算、价值分析 |
| R08 | 销售人员 | 负责销售业务的业务人员 | 库存查询、销售出库 |
| R09 | 系统管理员 | 系统运维管理人员 | 系统配置、权限管理 |

## 2. 库存台账用户故事

### 2.1 实时库存查询

#### US-WM-001 查询实时库存

| 项目 | 内容 |
|------|------|
| 用户故事ID | US-WM-001 |
| 用户故事名称 | 查询实时库存 |
| 作为 | 仓库管理员 |
| 我想要 | 能够按多种条件查询实时库存 |
| 以便于 | 快速了解当前库存状况，支持日常仓库管理 |
| 优先级 | P0 |
| 故事点 | 5 |

**验收标准**：
- Given 我已登录系统并拥有库存查询权限
- When 我进入库存查询页面，选择仓库、输入物料编码或名称
- Then 系统应显示符合条件的库存列表，包含物料信息、数量、库位等
- And 查询响应时间应小于2秒
- And 支持导出查询结果到Excel

**业务规则**：
- 支持按仓库、物料编码、物料名称、批次号、VIN码等条件查询
- 支持模糊查询和精确查询
- 默认按仓库+物料编码排序

## 3. 入库管理用户故事

### 3.1 采购入库

#### US-WM-009 采购入库登记

| 项目 | 内容 |
|------|------|
| 用户故事ID | US-WM-009 |
| 用户故事名称 | 采购入库登记 |
| 作为 | 仓库管理员 |
| 我想要 | 能够根据采购订单进行入库登记 |
| 以便于 | 完成采购物料入库，更新库存 |
| 优先级 | P0 |
| 故事点 | 8 |

**验收标准**：
- Given 采购订单已到货，我已登录系统
- When 我进入采购入库页面，选择采购订单
- Then 系统应显示订单明细信息
- And 我可以输入实收数量、批次号、库位等信息
- And 系统校验入库数量不超过订单数量的一定比例（超收控制）
- When 我确认入库
- Then 系统生成入库单据，更新库存台账

**业务规则**：
- 支持部分入库
- 超收比例可在系统参数中配置（默认5%）
- 入库时自动生成或手工录入批次号
- 需质检物料先进入质检区
