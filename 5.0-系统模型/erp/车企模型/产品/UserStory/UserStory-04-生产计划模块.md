## 2. 用户故事列表

### 2.1 主生产计划(MPS) - Epic

#### Epic描述
作为生产计划经理，我需要管理主生产计划的完整生命周期，从需求收集到计划发布，确保生产计划能够满足市场需求并与产能相匹配。

#### 用户故事 US-MPS-001: 销售需求汇总

```
作为 主计划员
我希望 能够快速汇总来自不同渠道的销售需求
以便 进行主生产计划的编制工作

验收标准:
1. 能够导入销售订单数据，支持按时间范围、产品、客户筛选
2. 能够录入和管理销售预测数据，支持多版本对比
3. 能够自动按产品、时间维度汇总各类需求
4. 汇总结果能够按周/月展示，支持钻取查看明细
5. 能够标识需求来源(订单/预测/安全库存)

故事点: 8
优先级: P0
迭代: Sprint 1
```

#### 用户故事 US-MPS-002: ATP可承诺量计算

```
作为 销售协调员
我希望 能够快速查询产品的可承诺交付量(ATP)
以便 准确回复客户的交期承诺

验收标准:
1. 能够输入产品编码和日期范围查询ATP
2. ATP计算需考虑:现有库存、在产订单、在途物料、已分配量
3. 能够按时间窗口展示ATP(周/旬/月)
4. 支持模拟新订单对ATP的影响
5. 能够关联查看ATP计算的明细数据

故事点: 8
优先级: P0
迭代: Sprint 1
```

#### 用户故事 US-MPS-003: 主生产计划编制

```
作为 主计划员
我希望 能够编制主生产计划并自动进行粗能力校验
以便 确保计划的可行性

验收标准:
1. 能够按产品和时间周期录入计划产量
2. 支持从需求自动生成计划，也可手工调整
3. 计划编制时自动进行粗能力需求计划(RCCP)计算
4. 当能力不足时给出预警和调整建议
5. 支持计划版本管理，可对比不同版本差异
6. 计划能够保存为草稿，支持多次修改

故事点: 13
优先级: P0
迭代: Sprint 2
```

#### 用户故事 US-MPS-004: MPS计划审批发布

```
作为 生产计划经理
我希望 能够审批主生产计划并控制发布流程
以便 确保只有经过审核的计划才能执行

验收标准:
1. 计划提交审批时自动检查完整性
2. 审批流程支持多级审批，可配置审批人
3. 审批过程中可退回修改
4. 审批通过后自动发布，状态变为已发布
5. 发布后通知相关人员(邮件/系统消息)
6. 已发布的计划在一定时间范围内冻结，不可随意修改

故事点: 8
优先级: P0
迭代: Sprint 2
```

#### 用户故事 US-MPS-005: MPS计划调整

```
作为 主计划员
我希望 能够处理计划的变更请求
以便 应对市场需求的变化

验收标准:
1. 能够发起计划调整申请，说明调整原因
2. 调整时重新进行粗能力校验
3. 重大调整需要重新走审批流程
4. 记录所有调整历史，可追溯
5. 调整后自动通知相关部门(采购、生产、销售)
6. 支持查看调整前后对比

故事点: 8
优先级: P1
迭代: Sprint 3
```

#### 用户故事 US-MPS-006: 预测冲销管理

```
作为 主计划员
我希望 管理预测和订单的冲销关系
以便 准确计算净需求

验收标准:
1. 能够配置预测冲销规则(先进先出、时间窗口等)
2. 系统自动计算预测被订单冲销的数量
3. 能够查看预测冲销明细
4. 支持手工调整冲销关系
5. 未被冲销的预测可设置过期处理规则

故事点: 5
优先级: P1
迭代: Sprint 3
```

## 5. 验收测试场景

### 5.1 核心业务场景

#### 场景1: 完整的MPS-MRP-排程流程

```
前置条件:
- 系统中已有销售订单和销售预测数据
- 物料主数据、BOM数据、工艺路线数据已维护完整

测试步骤:
1. 主计划员汇总销售需求，生成需求汇总表
2. 主计划员编制主生产计划，系统自动进行RCCP校验
3. 计划经理审批通过MPS
4. 物料计划员运行MRP，生成计划订单
5. 物料计划员确认计划订单，生成生产订单和采购申请
6. 生产调度员进行生产排程，优化混线顺序
7. 生产调度员下达工单到MES

预期结果:
- MPS计划数量与需求匹配
- MRP计划订单覆盖所有需求
- 排程方案可行，无资源冲突
- MES成功接收工单
```

#### 场景2: JIT物料拉动流程

```
前置条件:
- 生产排程已完成
- JIT物料拉动规则已配置
- 与WMS的接口已联通

测试步骤:
1. 根据排程结果，系统自动计算物料需求时序
2. 按JIT规则生成物料拉动指令
3. 拉动指令发送给WMS
4. WMS执行物料配送
5. 物料到达后确认收货
6. 系统更新线边仓库存

预期结果:
- 物料拉动时间准确，符合JIT要求
- 拉动指令与排程同步
- 物料准时到达生产现场
```

#### 场景3: 产能不足处理流程

```
前置条件:
- MPS计划已编制
- 工作中心能力已定义

测试步骤:
1. 系统进行RCCP计算
2. 发现某关键工作中心产能不足
3. 系统给出产能优化建议(加班/外协)
4. 计划员选择加班方案
5. 调整工作中心能力日历
6. 重新进行RCCP计算，产能满足

预期结果:
- 准确识别产能瓶颈
- 提供可行的优化方案
- 调整后产能满足需求
```
