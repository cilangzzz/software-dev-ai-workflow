## 1. 验收标准概述

### 1.1 验收原则

| 原则 | 说明 |
|------|------|
| 完整性 | 所有功能需求必须有对应的验收标准 |
| 可测试性 | 验收标准必须可被测试验证 |
| 明确性 | 验收标准表述清晰，无歧义 |
| 可追溯性 | 验收标准可追溯到用户故事和PRD需求 |

### 1.2 验收标准格式

```
AC-[模块编号]-[功能编号]-[标准序号]
```

### 1.3 验收条件定义

| 条件 | 说明 |
|------|------|
| 前置条件 | 执行测试前必须满足的条件 |
| 操作步骤 | 测试执行的具体步骤 |
| 预期结果 | 期望的系统响应和结果 |
| 后置条件 | 测试完成后的系统状态 |

## 2. 设备台账管理验收标准

### 2.1 设备档案管理

#### AC-09-01-01: 设备档案创建

| 项目 | 内容 |
|------|------|
| **前置条件** | 1. 用户已登录系统，具有设备档案创建权限<br>2. 设备分类数据已存在 |
| **测试步骤** | 1. 进入设备档案管理页面<br>2. 点击"新增设备"按钮<br>3. 选择设备分类，系统自动加载分类属性模板<br>4. 填写设备基本信息（名称、型号、品牌、供应商等）<br>5. 填写技术参数（根据分类模板动态展示）<br>6. 填写资产信息（购置日期、原值、折旧年限等）<br>7. 选择设备位置（工厂-车间-产线-工位）<br>8. 选择归属部门和使用部门<br>9. 上传设备图片和技术文档<br>10. 点击"保存"按钮 |
| **预期结果** | 1. 系统自动生成设备编码，格式：EQ+年月+4位序号<br>2. 设备编码全局唯一<br>3. 必填字段校验通过<br>4. 设备档案保存成功，状态为"在用"<br>5. 保存操作记录到审计日志<br>6. 页面提示"保存成功" |
| **后置条件** | 设备档案数据新增一条记录，可被查询和使用 |

#### AC-09-01-02: 设备编码规则

| 项目 | 内容 |
|------|------|
| **前置条件** | 系统参数配置中设置了设备编码规则 |
| **测试步骤** | 1. 创建新设备档案<br>2. 查看生成的设备编码<br>3. 连续创建多个设备，检查编码递增规则<br>4. 跨月创建设备，检查月份更新 |
| **预期结果** | 1. 编码格式：EQ+YYYYMM+4位流水号<br>2. 示例：EQ2026030001<br>3. 流水号按月重置<br>4. 编码不允许手动修改<br>5. 删除的设备编码不重复使用 |
| **后置条件** | 设备编码唯一且符合规则 |

#### AC-09-01-03: 设备档案查询

| 项目 | 内容 |
|------|------|
| **前置条件** | 系统中已存在设备档案数据 |
| **测试步骤** | 1. 进入设备档案列表页面<br>2. 测试按设备名称模糊查询<br>3. 测试按设备编码精确查询<br>4. 测试按设备分类筛选<br>5. 测试按设备状态筛选<br>6. 测试按位置筛选<br>7. 测试按部门筛选<br>8. 测试组合条件查询<br>9. 测试查询结果排序<br>10. 测试分页功能 |
| **预期结果** | 1. 查询响应时间≤2秒<br>2. 查询结果准确，符合筛选条件<br>3. 支持多条件组合查询<br>4. 默认按创建时间倒序排列<br>5. 每页默认显示20条，可调整<br>6. 查询条件可保存为常用查询 |
| **后置条件** | 查询结果正确展示 |

#### AC-09-01-04: 设备档案编辑

| 项目 | 内容 |
|------|------|
| **前置条件** | 用户具有设备档案编辑权限，设备档案已存在 |
| **测试步骤** | 1. 选择一条设备档案<br>2. 点击"编辑"按钮<br>3. 修改设备名称<br>4. 修改技术参数<br>5. 修改位置信息<br>6. 点击"保存"按钮 |
| **预期结果** | 1. 设备编码不可修改<br>2. 部分关键字段修改需审批（如原值、状态）<br>3. 修改记录保存到变更历史<br>4. 保存成功后页面刷新展示最新数据<br>5. 操作记录到审计日志 |
| **后置条件** | 设备档案数据更新成功 |

#### AC-09-01-05: 设备档案删除

| 项目 | 内容 |
|------|------|
| **前置条件** | 用户具有设备档案删除权限 |
| **测试步骤** | 1. 选择一条设备档案<br>2. 点击"删除"按钮<br>3. 系统校验设备是否有关联数据<br>4. 确认删除操作 |
| **预期结果** | 1. 有关联数据的设备不允许删除，提示原因<br>2. 关联数据包括：维护记录、维修记录、备件关联等<br>3. 无关联数据可删除，需二次确认<br>4. 删除为逻辑删除，数据保留在数据库<br>5. 删除操作记录到审计日志 |
| **后置条件** | 设备档案标记为已删除，不可见 |
