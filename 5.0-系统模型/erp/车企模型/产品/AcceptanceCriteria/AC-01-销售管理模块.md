## 验收标准说明

### Gherkin语法格式

验收标准采用Gherkin格式编写，结构如下：

```
Feature: 功能特性描述

Scenario: 场景名称
  Given 前置条件
  When 执行操作
  Then 预期结果
  And 附加条件或结果
```

### 关键字说明

- **Feature**: 功能特性，对应一个功能模块
- **Scenario**: 测试场景，对应一个具体测试用例
- **Given**: 前置条件，测试前需要满足的条件
- **When**: 执行操作，测试执行的动作
- **Then**: 预期结果，执行后应该产生的结果
- **And**: 附加条件，补充说明
- **But**: 排除条件，不应出现的情况

## 1. 客户管理子模块验收标准

### Feature: 客户档案管理

#### Scenario: 成功创建客户档案

```gherkin
Scenario: 销售代表成功创建个人客户档案
  Given 销售代表已登录系统
  And 销售代表具有客户创建权限
  When 销售代表进入客户管理页面
  And 销售代表点击"新建客户"按钮
  And 销售代表选择客户类型为"个人客户"
  And 销售代表填写客户信息：
    | 字段 | 值 |
    | 姓名 | 张三 |
    | 手机号 | 13800138000 |
    | 性别 | 男 |
    | 年龄 | 35 |
    | 购车意向车型 | Model A |
    | 信息来源 | 官网 |
  And 销售代表点击"保存"按钮
  Then 系统显示"客户创建成功"提示
  And 系统生成客户编号，格式为"CU" + 年月日(8位) + 序号(4位)
  And 客户信息保存到数据库
  And 客户创建人和创建时间被记录
```

