## 1. 模块概述

### 1.1 模块定位

供应商管理模块(SRM - Supplier Relationship Management)是汽车制造业ERP系统的核心协同模块，旨在建立企业与供应商之间的数字化协作平台，实现从供应商准入、采购协同、质量协同到财务结算的全生命周期管理。

### 1.2 业务背景

汽车制造业具有供应链长、零部件种类多、质量要求严格的特点。传统的供应商管理存在以下问题：

- 供应商信息分散，资质管理不透明
- 采购协同效率低，沟通成本高
- 质量问题追溯困难，整改跟踪不及时
- 结算对账繁琐，财务风险难以控制
- 缺乏有效的供应商绩效评估体系

### 1.3 模块价值

| 价值维度 | 具体收益 |
|---------|---------|
| 效率提升 | 采购协同周期缩短40%，对账效率提升60% |
| 成本降低 | 供应商管理成本降低25%，采购成本降低8-12% |
| 质量保障 | 质量问题响应时间缩短50%，来料合格率提升至98% |
| 风险控制 | 供应商风险预警提前30天，财务风险可控率95% |
| 决策支持 | 供应商绩效数据实时可视化，采购决策更科学 |

## 3. 功能需求详细说明

### 3.1 供应商门户

#### 3.1.1 供应商注册与认证

**功能描述**

供应商通过门户进行在线注册，提交企业基本信息和资质材料，经企业审核后成为合格供应商。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 在线注册 | 供应商填写企业基本信息、联系人信息、银行账户信息 |
| 资质上传 | 上传营业执照、税务登记证、组织机构代码证、行业资质证书等 |
| 分类申请 | 根据供应物料类别选择供应商分类进行申请 |
| 审核流程 | 支持多级审核，可配置审核节点和审批人 |
| 认证状态 | 实时查看注册审核进度和结果 |

**数据要求**

- 企业基本信息：公司名称、统一社会信用代码、注册地址、注册资本、成立日期、经营范围
- 联系人信息：姓名、职位、电话、邮箱、地址
- 银行信息：开户银行、银行账号、开户名称
- 资质信息：证照类型、证照编号、有效期、附件

**业务规则**

1. 统一社会信用代码唯一性校验
2. 资质证照必须在有效期内
3. 审核通过前供应商只能查看不能操作
4. 审核驳回需填写驳回原因

#### 3.1.2 供应商信息维护

**功能描述**

供应商维护自身的企业信息、产品信息、产能信息等，企业可查看和管理供应商档案。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 基本信息维护 | 供应商修改企业基本信息，部分关键字段修改需审核 |
| 产品信息维护 | 维护供应商可供应的产品清单及价格信息 |
| 产能信息维护 | 维护产能数据，支持月度产能填报 |
| 联系人管理 | 管理多个联系人，设置主联系人 |
| 银行账户管理 | 管理多个银行账户，设置主账户 |

**权限控制**

- 供应商：仅可维护本企业信息
- 采购员：可查看分管供应商信息
- 采购主管：可修改供应商部分信息
- 系统管理员：所有权限

#### 3.1.3 资质证照管理

**功能描述**

管理供应商各类资质证照的全生命周期，包括新增、更新、到期提醒、审核等功能。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 证照类型配置 | 配置不同供应商类别所需的证照类型 |
| 证照上传 | 供应商上传证照扫描件，填写有效期 |
| 到期预警 | 证照到期前30/15/7天自动预警 |
| 证照审核 | 采购方审核证照的真实性和有效性 |
| 证照档案 | 证照历史版本归档，支持查看历史记录 |

**证照类型**

| 证照类型 | 是否必需 | 有效期要求 |
|---------|---------|-----------|
| 营业执照 | 是 | 长期或有期限 |
| IATF16949认证 | 是(核心供应商) | 有期限 |
| ISO9001认证 | 是 | 有期限 |
| ISO14001认证 | 否 | 有期限 |
| 特种行业许可证 | 按需 | 有期限 |
| 3C认证 | 按需 | 有期限 |

#### 3.1.4 通知公告中心

**功能描述**

企业向供应商发布通知公告，供应商可查看历史公告信息。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 公告发布 | 发布采购政策、质量要求、放假通知等公告 |
| 定向推送 | 支持按供应商类别、地区等条件定向推送 |
| 阅读确认 | 记录供应商阅读状态，支持阅读回执 |
| 消息提醒 | 新公告通过站内信、邮件、短信提醒 |

### 3.2 采购协同

#### 3.2.1 询价报价管理

**功能描述**

采购方向供应商发起询价，供应商在线报价，支持多轮询报价和比价功能。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 询价发布 | 选择询价物料、供应商，设定询价截止时间 |
| 询价模板 | 支持询价单模板，快速创建询价 |
| 在线报价 | 供应商在线填写报价信息，上传报价单 |
| 多轮询价 | 支持多轮询价报价，记录完整过程 |
| 比价分析 | 自动汇总比价，生成比价报告 |
| 定标确认 | 确认中标供应商，生成报价协议 |

**报价信息**

- 物料编码、名称、规格型号
- 报价单价、币种、税率
- 最小起订量、包装规格
- 交货周期、有效期
- 技术参数偏离说明

**业务规则**

1. 询价截止时间后不能再报价
2. 报价需在有效期内有效
3. 支持公开询价和定向询价两种模式
4. 定标后自动生成价格协议

#### 3.2.2 采购订单协同

**功能描述**

采购方向供应商发布采购订单，供应商在线确认、安排生产、反馈交货计划。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 订单发布 | 从请购单或计划生成采购订单，推送给供应商 |
| 订单确认 | 供应商确认订单，可提出交期调整申请 |
| 订单变更 | 支持订单数量、交期等变更，记录变更历史 |
| 订单跟踪 | 实时跟踪订单执行状态 |
| 订单关闭 | 订单完成或取消后关闭 |

**订单状态流转**

```
待确认 -> 已确认 -> 生产中 -> 已发货 -> 已到货 -> 已检验 -> 已入库 -> 已结算 -> 已关闭
         |
         -> 已拒绝
         |
         -> 变更中 -> 待确认(变更)
```

**订单信息**

| 信息类别 | 字段 |
|---------|------|
| 基本信息 | 订单号、订单类型、采购组织、采购员、下单时间 |
| 供应商信息 | 供应商编码、名称、联系人、收货地址 |
| 物料信息 | 物料编码、名称、规格、单位、数量、单价、金额 |
| 交货信息 | 交货日期、交货地址、包装要求、运输方式 |
| 结算信息 | 结算方式、付款条件、发票类型 |

#### 3.2.3 交货计划协同

**功能描述**

供应商维护交货计划，采购方查看和确认交货计划，实现供需协同。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 交货计划维护 | 供应商按订单维护详细交货计划 |
| 计划确认 | 采购方确认交货计划，提出调整要求 |
| ASN预报 | 发货前生成ASN发货通知单 |
| 计划变更 | 支持交货计划变更申请和审批 |
| 计划达成率 | 统计交货计划达成情况 |

**ASN信息**

- ASN编号、订单号
- 物料清单：物料编码、名称、数量、批次号
- 发货信息：发货时间、预计到货时间、运输方式、物流单号
- 包装信息：包装类型、箱数、毛重、体积

#### 3.2.4 到货确认管理

**功能描述**

采购方确认到货信息，供应商查看到货确认状态。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 到货登记 | 登记到货信息，关联ASN和采购订单 |
| 数量确认 | 确认实收数量，处理差异 |
| 待检入库 | 生成待检单，推送质量管理模块 |
| 到货通知 | 自动通知供应商到货确认结果 |
| 差异处理 | 处理到货数量与发货数量差异 |

### 3.3 质量协同

#### 3.3.1 来料检验协同

**功能描述**

采购方完成来料检验后，将检验结果推送给供应商，供应商查看和确认。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 检验结果推送 | 检验完成后自动推送结果给供应商 |
| 检验报告查看 | 供应商查看详细检验报告 |
| 检验标准维护 | 维护各物料的检验标准和方法 |
| 特采申请 | 对不合格批次申请特采使用 |
| 检验数据统计 | 统计供应商来料检验合格率 |

**检验结果等级**

| 等级 | 说明 | 处理方式 |
|-----|------|---------|
| 合格 | 检验合格 | 正常入库 |
| 让步接收 | 轻微不合格，经评审可使用 | 让步使用，记录问题 |
| 挑选/返工 | 需挑选或返工后使用 | 供应商派人处理 |
| 退货 | 严重不合格 | 退回供应商 |
| 特采 | 紧急情况下的特殊批准 | 特采记录，跟踪使用情况 |

#### 3.3.2 不良品处理

**功能描述**

管理来料不良品的处理流程，包括8D报告、退货、索赔等处理方式。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 不良登记 | 登记不良品信息，关联来料批次 |
| 原因分析 | 供应商进行原因分析，提交分析报告 |
| 8D报告 | 供应商提交8D整改报告 |
| 处理跟踪 | 跟踪不良品处理进度 |
| 费用索赔 | 对不良造成的损失进行索赔 |

**不良品处理流程**

```
不良发现 -> 不良登记 -> 原因分析 -> 处理方案 -> 方案审批 -> 执行处理 -> 效果验证 -> 案例关闭
```

**8D报告内容**

1. D1: 建立团队
2. D2: 问题描述
3. D3: 临时措施
4. D4: 根本原因
5. D5: 永久措施
6. D6: 措施实施
7. D7: 预防措施
8. D8: 团队祝贺

#### 3.3.3 质量整改跟踪

**功能描述**

对供应商质量问题进行整改跟踪，确保问题得到有效解决。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 整改任务下达 | 下达质量整改任务，设定整改期限 |
| 整改计划提交 | 供应商提交整改计划和措施 |
| 整改跟踪 | 跟踪整改进度和结果 |
| 效果验证 | 验证整改效果，必要时现场审核 |
| 整改评价 | 评价整改效果，纳入绩效评估 |

#### 3.3.4 PPAP管理

**功能描述**

管理生产件批准程序(PPAP)，确保供应商产品满足质量要求。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| PPAP申请 | 新产品或变更产品提交PPAP申请 |
| 文档提交 | 提交PPAP要求的各项文档 |
| 文档审核 | 审核PPAP文档的完整性和正确性 |
| 样件验证 | 进行样件试装和验证 |
| 批准状态 | PPAP批准、临时批准、拒绝 |

**PPAP文档要求**

| 文档等级 | 要求文档 |
|---------|---------|
| 等级1 | 仅提交保证书 |
| 等级2 | 保证书+外观件批准报告+样品 |
| 等级3 | 保证书+外观件批准报告+全套文档 |
| 等级4 | 保证书+其他要求文档 |
| 等级5 | 保证书+全套文档+样品(保留在企业) |

**PPAP状态**

| 状态 | 说明 |
|-----|------|
| 待提交 | 等待供应商提交PPAP文档 |
| 审核中 | 采购方审核文档中 |
| 样件验证 | 样件试装验证中 |
| 已批准 | PPAP通过，可批量供货 |
| 临时批准 | 临时批准，需在限期内完善 |
| 已拒绝 | PPAP未通过，需重新提交 |

### 3.4 结算协同

#### 3.4.1 对账单管理

**功能描述**

采购方生成对账单发送给供应商，供应商在线确认，实现快速对账。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 对账单生成 | 按期间自动生成对账单 |
| 对账单发送 | 发送对账单给供应商确认 |
| 在线对账 | 供应商在线核对并确认对账单 |
| 差异处理 | 处理对账差异，协商解决 |
| 对账确认 | 双方确认后生成结算依据 |

**对账单信息**

- 对账期间
- 入库明细：入库日期、物料、数量、单价、金额
- 退货明细：退货日期、物料、数量、金额
- 调整明细：调整原因、金额
- 对账金额：应收/应付金额汇总

**对账流程**

```
生成对账单 -> 发送供应商 -> 供应商核对 -> 确认/差异反馈 -> 差异处理 -> 双方确认
```

#### 3.4.2 发票协同

**功能描述**

供应商在线提交发票信息，采购方进行发票校验和入账处理。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 发票提交 | 供应商提交电子发票信息 |
| 发票校验 | 校验发票与入库单、对账单是否匹配 |
| 三单匹配 | 订单、入库单、发票三单匹配校验 |
| 发票入账 | 校验通过后推送财务入账 |
| 发票查询 | 查询发票状态和历史记录 |

**发票信息**

- 发票类型：增值税专用发票、普通发票
- 发票代码、发票号码
- 开票日期、开票金额
- 税率、税额、价税合计
- 发票扫描件/电子发票文件

**三单匹配规则**

| 校验项 | 说明 |
|-------|------|
| 供应商一致性 | 发票供应商=订单供应商=入库供应商 |
| 物料一致性 | 发票物料=订单物料=入库物料 |
| 数量一致性 | 发票数量<=订单数量，且<=入库数量 |
| 价格一致性 | 发票单价=订单单价(或约定价格) |
| 金额一致性 | 发票金额=数量*单价 |

#### 3.4.3 付款申请

**功能描述**

供应商查看应付账款，提交付款申请，跟踪付款进度。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 应付查询 | 查询应付账款明细和汇总 |
| 付款申请 | 提交付款申请，附上相关凭证 |
| 付款审批 | 采购方审批付款申请 |
| 付款执行 | 财务执行付款，登记付款信息 |
| 付款通知 | 通知供应商付款结果 |

**付款条件**

| 条件类型 | 说明 |
|---------|------|
| 月结30天 | 月底结算后30天付款 |
| 月结60天 | 月底结算后60天付款 |
| 月结90天 | 月底结算后90天付款 |
| 货到付款 | 货到验收合格后付款 |
| 预付款 | 发货前支付部分货款 |

#### 3.4.4 结算查询

**功能描述**

供应商查询与采购方的所有结算相关信息，包括订单、入库、对账、发票、付款等。

**功能要点**

| 功能项 | 说明 |
|-------|------|
| 订单结算查询 | 查询订单的结算状态 |
| 入库明细查询 | 查询入库明细和对账状态 |
| 发票状态查询 | 查询发票校验和入账状态 |
| 付款记录查询 | 查询付款记录和付款进度 |
| 结算汇总报表 | 汇总展示结算相关数据 |
