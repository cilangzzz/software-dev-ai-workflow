## 1. 功能概述

### 1.1 模块定位

销售管理模块(SD)是汽车制造业ERP系统的核心业务模块之一，负责管理从客户开发到车辆交付的完整销售生命周期。本模块深度结合汽车行业特性，支持经销商网络管理、车辆VIN码追踪、多级销售渠道管理以及车辆交付PDI检查等业务场景。

### 1.2 业务目标

- 建立统一的客户资源管理平台，实现客户全生命周期管理
- 规范销售订单处理流程，提升订单处理效率30%以上
- 实现车辆从订单到交付的全程可追溯
- 支持多级经销商网络的高效协同
- 提供实时销售数据分析，辅助决策

### 1.3 模块范围

| 范围内 | 范围外 |
|--------|--------|
| 客户信息管理 | 财务结算处理 |
| 销售订单全流程管理 | 生产计划编制 |
| 车辆配置与定价 | 售后维修服务 |
| 交付与PDI检查 | 车辆召回管理 |
| 销售数据分析 | 营销活动管理 |

## 6. 汽车行业特性说明

### 6.1 经销商网络管理

#### 6.1.1 多级经销商体系

汽车行业采用多级经销商网络，系统需支持：

- **一级经销商(4S店)**：直接与主机厂签约，享有优先供货权
- **二级经销商**：从一级经销商进货，区域覆盖更广
- **特约经销商**：针对特定车型或区域的授权经销商

#### 6.1.2 区域保护机制

- 按省份/城市划分销售区域
- 每个区域设置授权经销商
- 跨区域销售需要特殊审批
- 区域销售配额管理

#### 6.1.3 经销商务人员管理

- 经销商务人员需在系统注册
- 业绩归属到具体销售人员
- 支持销售人员调动历史追溯

### 6.2 车辆VIN码管理

#### 6.2.1 VIN码规则

VIN码(Vehicle Identification Number)是车辆唯一标识：

- 17位字符，包含：WMI(世界制造商代码) + VDS(车辆说明部分) + VIS(车辆指示部分)
- 生产年份、产地、车型等信息编码在VIN中
- 支持VIN码校验算法验证

#### 6.2.2 VIN码生命周期

```
VIN码生成(生产计划) --> VIN码预分配(订单确认) --> VIN码正式绑定(车辆下线) --> VIN码追溯(售后)
```

#### 6.2.3 VIN码业务规则

- 每个销售订单最终必须关联确定的VIN码
- VIN码一旦绑定不可随意变更
- VIN码变更需要走特殊审批流程
- 通过VIN码可追溯完整的生产、销售、售后信息

### 6.3 多级销售渠道管理

#### 6.3.1 渠道类型

| 渠道类型 | 说明 | 特点 |
|----------|------|------|
| 直营店 | 主机厂直营 | 价格统一，服务标准高 |
| 4S店 | 授权经销商 | 集销售、服务、配件、信息反馈于一体 |
| 二级网点 | 二级经销商 | 覆盖更广泛的市场 |
| 线上渠道 | 官网、电商平台 | 收集线索，引流到店 |
| 大客户渠道 | 企业团购、政府采购 | 批量销售，特殊价格 |

#### 6.3.2 渠道价格管理

- 不同渠道可设置不同的价格体系
- 经销商价格 = 批发价 = 指导价 - 经销商返利
- 终端零售价受指导价约束
- 促销价格需要审批后生效

### 6.4 车辆交付PDI检查

#### 6.4.1 PDI检查定义

PDI(Pre-Delivery Inspection)即交付前检查，是车辆交付给客户前的必要环节：

- 确保车辆在运输过程中没有损坏
- 检查车辆各项功能是否正常
- 确保交付给客户的是合格产品

#### 6.4.2 PDI检查项目

| 检查类别 | 检查项目 |
|----------|----------|
| 外观检查 | 车漆、玻璃、轮胎、轮毂、车身划痕 |
| 内饰检查 | 座椅、仪表盘、内饰件、电子设备 |
| 发动机舱 | 发动机、变速箱、油液、皮带 |
| 底盘检查 | 悬挂、制动系统、排气管 |
| 电气系统 | 灯光、音响、空调、电动车窗 |
| 随车物品 | 备胎、工具、证件、说明书 |

#### 6.4.3 PDI处理流程

```
车辆到达交付中心 --> PDI检查 --> 检查结果记录
  |
  +---> 合格 --> 准备交付
  |
  +---> 不合格 --> 问题分类 --> 返修/更换 --> 重新PDI
```

### 6.5 车辆交付流程

#### 6.5.1 交付前准备

- 核对订单信息与车辆信息
- 准备交付单据：合格证、发票、说明书、保养手册
- 准备随车物品：钥匙(至少2把)、备胎、工具包
- 车辆清洁：外观清洗、内饰清洁

#### 6.5.2 交付确认

- 客户/经销商现场验车
- 确认车辆外观、配置无误
- 签署交付确认单
- 移交车辆及相关单据
- 系统记录交付信息
