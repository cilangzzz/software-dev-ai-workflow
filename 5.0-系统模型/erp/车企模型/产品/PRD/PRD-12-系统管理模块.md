# PRD-12 系统管理模块产品需求文档

## 文档信息
- **文档编号**: PRD-12
- **模块名称**: 系统管理模块 (System Management Module)
- **版本号**: v1.0
- **创建日期**: 2026-03-24
- **优先级**: P0（核心基础模块）
- **产品负责人**: ERP产品团队

## 1. 模块概述

### 1.1 模块定位
系统管理模块是整个ERP系统的基础支撑模块，提供用户身份认证、组织架构管理、多租户隔离、系统配置、操作日志和接口管理等核心功能。该模块为所有业务模块提供统一的用户权限控制、组织体系支撑和系统运行保障。

### 1.2 业务价值
- **统一身份认证**: 实现单点登录，降低用户管理成本60%
- **多租户隔离**: 支持SaaS化部署，实现租户数据100%逻辑隔离
- **权限精细化**: 基于RBAC模型实现细粒度权限控制，保障数据安全
- **操作可追溯**: 全链路操作日志，满足审计合规要求
- **系统高可用**: 配置管理和监控预警，保障系统稳定运行

### 1.3 适用范围
- 所有需要登录ERP系统的用户
- 系统管理员、租户管理员
- 需要与ERP集成的外部系统
- 审计和合规部门

## 3. 功能需求详细说明

### 3.1 用户管理

#### 3.1.1 用户账号管理
**功能描述**:
提供用户账号的全生命周期管理，包括创建、修改、删除、查询、导入导出等功能。

**功能要点**:
- 用户基本信息：用户名、姓名、工号、手机、邮箱、所属组织
- 用户状态：待激活、正常、禁用、锁定、注销
- 用户类型：系统管理员、租户管理员、普通用户
- 批量导入：支持Excel模板批量导入用户
- 批量导出：支持按条件导出用户列表

**业务规则**:
1. 用户名全局唯一，长度4-20位，仅支持字母、数字、下划线
2. 工号在租户范围内唯一
3. 手机号需验证格式，作为密码找回途径
4. 邮箱需验证格式，作为系统通知途径
5. 用户创建后默认状态为"待激活"，需首次登录修改密码后激活
6. 用户删除采用逻辑删除，保留历史数据关联

**界面设计要点**:
- 列表页：支持多条件筛选、分页、排序
- 新增/编辑：表单验证、组织树选择器
- 详情页：展示用户完整信息、角色列表、操作日志
- 批量操作：批量启用/禁用/删除

#### 3.1.2 用户角色分配
**功能描述**:
为用户分配一个或多个角色，实现权限的灵活组合。

**功能要点**:
- 角色列表展示（按租户隔离）
- 多角色分配支持
- 角色有效期设置
- 主角色标识

**业务规则**:
1. 一个用户可分配多个角色
2. 角色权限取并集
3. 可设置角色有效期，到期自动失效
4. 必须指定一个主角色

#### 3.1.3 密码策略管理
**功能描述**:
配置和管理用户密码安全策略。

**功能要点**:
- 密码复杂度要求（长度、字符类型）
- 密码有效期
- 密码历史记录（防止重复使用）
- 密码找回机制

**业务规则**:
1. 默认密码长度不少于8位
2. 必须包含大写字母、小写字母、数字、特殊字符中的三种
3. 密码有效期默认90天
4. 新密码不能与最近5次密码相同
5. 连续输错5次密码，账号锁定30分钟

### 3.2 组织管理

#### 3.2.1 组织架构管理
**功能描述**:
管理系统组织架构，支持多层级树形结构。

**功能要点**:
- 组织树形结构管理
- 组织层级：集团 > 公司 > 事业部 > 部门 > 班组
- 组织负责人设置
- 组织编码管理
- 组织状态管理

**业务规则**:
1. 组织架构支持最多6级层级
2. 组织编码在租户范围内唯一
3. 组织删除前需检查是否有关联用户
4. 组织合并需处理人员归属

**界面设计要点**:
- 左侧树形导航，右侧详情面板
- 支持拖拽调整组织层级
- 组织架构图可视化展示

#### 3.2.2 岗位管理
**功能描述**:
管理组织内的岗位信息。

**功能要点**:
- 岗位基本信息：编码、名称、所属组织
- 岗位职责描述
- 岗位权限配置
- 岗位人员编制

**业务规则**:
1. 岗位编码在租户范围内唯一
2. 岗位需关联到具体组织
3. 支持岗位继承上级组织权限

### 3.3 多租户管理

#### 3.3.1 租户注册与开通
**功能描述**:
支持租户在线注册和系统管理员审核开通。

**功能要点**:
- 租户注册申请
- 租户信息审核
- 租户账号开通
- 租户初始化配置

**业务规则**:
1. 租户注册需提供企业名称、统一社会信用代码、联系人信息
2. 管理员审核通过后自动创建租户
3. 租户创建后自动初始化默认角色和权限
4. 租户可选择套餐（基础版/专业版/企业版）

#### 3.3.2 租户数据隔离
**功能描述**:
确保多租户数据的逻辑隔离和安全性。

**功能要点**:
- tenant_id字段自动注入
- 数据查询自动添加租户过滤
- 租户数据独立备份
- 租户数据迁移支持

**业务规则**:
1. 所有业务表必须包含tenant_id字段
2. 数据查询、修改、删除自动添加租户过滤条件
3. 跨租户数据访问需特殊授权
4. 租户删除采用逻辑删除，保留数据

#### 3.3.3 租户资源配置
**功能描述**:
为每个租户配置独立的资源限额。

**功能要点**:
- 用户数量限制
- 存储空间限制
- API调用次数限制
- 功能模块授权

**业务规则**:
1. 不同套餐有不同的资源配置
2. 租户资源使用接近上限时预警
3. 超出限额时限制相应功能

### 3.4 系统配置

#### 3.4.1 参数配置管理
**功能描述**:
管理系统运行参数，支持租户级和系统级配置。

**功能要点**:
- 参数分类管理
- 参数键值对配置
- 参数层级：系统级 > 租户级 > 用户级
- 参数版本管理

**业务规则**:
1. 参数优先级：用户级 > 租户级 > 系统级
2. 敏感参数需加密存储
3. 参数修改需记录日志
4. 支持参数热加载

**参数类型示例**:
- 系统参数：系统名称、LOGO、版权信息
- 业务参数：单据编号规则、审批流程配置
- 安全参数：密码策略、会话超时时间
- 集成参数：第三方系统接口配置

#### 3.4.2 数据字典管理
**功能描述**:
管理系统数据字典，支持多种字典类型。

**功能要点**:
- 字典类型管理
- 字典项管理
- 字典值国际化
- 字典缓存管理

**业务规则**:
1. 字典类型编码全局唯一
2. 字典项在字典类型内唯一
3. 字典支持启用/禁用状态
4. 字典修改需刷新缓存

**字典类型示例**:
- 用户状态：待激活/正常/禁用/锁定/注销
- 组织类型：集团/公司/事业部/部门/班组
- 审批状态：待审批/已通过/已拒绝
- 单据状态：草稿/已提交/已审批/已完成/已作废

### 3.5 日志管理

#### 3.5.1 操作日志
**功能描述**:
记录用户所有操作行为，支持审计追溯。

**功能要点**:
- 操作人员信息
- 操作时间
- 操作类型（新增/修改/删除/查询/导出）
- 操作模块
- 操作内容（变更前后对比）
- IP地址和设备信息

**业务规则**:
1. 所有关键操作必须记录日志
2. 日志数据只增不改
3. 日志保留期限可配置（默认3年）
4. 敏感操作需特别标注

#### 3.5.2 登录日志
**功能描述**:
记录用户登录行为，支持安全审计。

**功能要点**:
- 登录时间
- 登录方式（账号密码/SSO/扫码）
- 登录IP
- 登录设备
- 登录结果（成功/失败）
- 失败原因

**业务规则**:
1. 登录失败需记录原因（密码错误/账号锁定/权限不足）
2. 异常登录行为预警（异地登录、频繁失败）
3. 支持强制下线功能

#### 3.5.3 接口访问日志
**功能描述**:
记录API接口调用情况，支持性能分析。

**功能要点**:
- 接口地址
- 请求方法
- 请求参数
- 响应结果
- 响应时间
- 调用来源

**业务规则**:
1. 接口调用超时需特别记录
2. 接口异常需记录堆栈信息
3. 支持接口调用统计和分析

### 3.6 接口管理

#### 3.6.1 API接口注册
**功能描述**:
注册和管理系统提供的API接口。

**功能要点**:
- 接口基本信息：名称、路径、方法、描述
- 接口参数定义
- 接口返回值定义
- 接口状态（开发中/已发布/已下线）

**业务规则**:
1. 接口路径需遵循RESTful规范
2. 接口版本号管理
3. 接口变更需记录版本历史

#### 3.6.2 接口权限配置
**功能描述**:
配置接口访问权限，支持细粒度控制。

**功能要点**:
- 接口与角色关联
- 接口与用户关联
- 接口访问频率限制
- IP白名单配置

**业务规则**:
1. 接口默认需要认证才能访问
2. 公开接口需特别标注
3. 敏感接口需额外权限验证

#### 3.6.3 接口调用统计
**功能描述**:
统计和分析接口调用情况。

**功能要点**:
- 调用量统计
- 响应时间统计
- 错误率统计
- 调用来源分析

**业务规则**:
1. 实时统计接口调用情况
2. 异常调用自动告警
3. 支持接口性能排行榜

## 4. 非功能需求

### 4.1 性能需求
- 用户登录响应时间 < 1秒
- 用户列表查询响应时间 < 2秒（10000条数据）
- 组织树加载时间 < 1秒（500个节点）
- 接口调用响应时间 < 200ms
- 支持并发用户数 ≥ 5000

### 4.2 安全需求
- 密码加密存储（BCrypt）
- 敏感数据传输加密（HTTPS）
- SQL注入防护
- XSS攻击防护
- CSRF防护
- 接口防重放攻击
- 操作日志不可篡改

### 4.3 可用性需求
- 系统可用性 ≥ 99.9%
- 支持故障自动切换
- 支持数据自动备份
- 支持灰度发布

### 4.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Edge、Safari）
- 支持移动端适配
- 支持第三方系统集成（SSO、LDAP）

### 4.5 可扩展性需求
- 支持水平扩展
- 支持插件化扩展
- 支持自定义认证方式

## 8. 业务流程

### 8.1 用户注册激活流程
1. 管理员创建用户账号（待激活状态）
2. 系统发送激活邮件/短信
3. 用户点击链接，设置密码
4. 账号激活成功，状态变更为正常
5. 用户可正常登录系统

### 8.2 用户登录流程
1. 用户输入用户名密码
2. 系统验证用户信息
3. 验证通过，生成Token
4. 记录登录日志
5. 返回用户信息和权限列表
6. 加载用户菜单

### 8.3 租户开通流程
1. 用户提交租户注册申请
2. 系统管理员审核
3. 审核通过，创建租户
4. 初始化租户数据（默认角色、权限、管理员账号）
5. 发送租户开通通知
6. 租户管理员登录配置系统

### 8.4 组织调整流程
1. 发起组织调整申请
2. 审批流程
3. 审批通过，调整组织架构
4. 同步更新相关用户组织归属
5. 记录调整日志
