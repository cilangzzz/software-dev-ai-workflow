## 1. 模块概述

### 1.1 模块定位

设备资产管理(EAM - Enterprise Asset Management)模块是汽车制造业ERP系统的核心运维模块，负责管理企业全生命周期内的设备资产，实现设备从采购、安装、运行、维护、维修到报废的全过程管理，并与IoT平台深度集成，实现设备状态的实时监控和智能预警。

### 1.2 业务价值

| 价值维度 | 具体价值 |
|----------|----------|
| **降本增效** | 通过预防性维护降低设备故障率30%，减少非计划停机时间40% |
| **资产保值** | 延长设备使用寿命15%，提高设备综合效率(OEE)20% |
| **成本控制** | 备件库存周转率提升25%，维修成本降低20% |
| **合规管理** | 确保特种设备合规检验率100%，满足ISO体系要求 |
| **数据驱动** | 设备运行数据积累，支撑精益生产和智能决策 |

### 1.3 核心功能架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     设备资产管理(EAM)模块                        │
├─────────┬─────────┬─────────┬─────────┬─────────┬──────────────┤
│ 设备台账 │ 设备维护 │ 设备维修 │ 备件管理 │ 设备监控 │  IoT集成     │
├─────────┴─────────┴─────────┴─────────┴─────────┴──────────────┤
│                        基础数据层                                │
│  设备分类 | 供应商管理 | 人员管理 | 库存管理 | 工作流引擎       │
└─────────────────────────────────────────────────────────────────┘
```

| 功能编号 | 功能名称 | 功能描述 | 优先级 |

| F09-02-02 | 维护计划管理 | 周期性维护计划制定、审批、调整 | P1 |
| F09-02-03 | 维护工单管理 | 工单创建、派工、执行、验收、关闭 | P1 |
| F09-02-04 | 维护任务执行 | 移动端维护任务执行、记录填写、拍照上传 | P1 |
| F09-02-05 | 维护日历管理 | 维护计划日历视图、资源冲突检测 | P2 |
| F09-02-06 | 润滑管理 | 润滑点管理、润滑计划、润滑记录 | P2 |
| F09-02-07 | 点检管理 | 设备点检标准、点检计划、点检记录 | P1 |
| F09-02-08 | 巡检管理 | 巡检路线、巡检任务、巡检记录 | P2 |
| F09-02-09 | 特检管理 | 特种设备定期检验计划与记录管理 | P2 |
| F09-02-10 | 维护统计分析 | MTBF、MTTR、维护完成率等指标分析 | P2 |

#### 3.2.3 维护策略类型

| 策略类型 | 触发方式 | 适用场景 |
|----------|----------|----------|
| **时间周期维护** | 固定时间间隔(日/周/月/季/年) | 常规保养、定期检查 |
| **运行时长维护** | 累计运行时间达到阈值 | 设备定期保养 |
| **产量触发维护** | 生产产量达到设定值 | 生产设备维护 |
| **条件触发维护** | IoT监测参数超过阈值 | 预测性维护 |
| **事件触发维护** | 故障告警、质量异常 | 事后维护 |

#### 3.2.4 维护工单状态流转

```
待审核 → 待派工 → 进行中 → 待验收 → 已完成 → 已关闭
   ↓        ↓        ↓        ↓
  退回    暂停     退回    驳回
```

#### 3.4.3 备件ABC分类管理

| 分类 | 定义 | 库存策略 |
|------|------|----------|
| A类 | 高价值、关键备件，占金额70% | 安全库存+定期订货，重点管控 |
| B类 | 中等价值备件，占金额20% | 定量订货，适度管控 |
| C类 | 低价值、通用备件，占金额10% | 批量订货，简化管理 |

| 预警类型 | 触发条件 | 预警动作 |

| 效期预警 | 距效期≤30天 | 提示优先使用 |
| 滞销预警 | 无出库记录≥180天 | 提示呆滞风险 |

| 监控数据刷新频率 | ≤5秒 | 设备状态实时更新 |

| 工单创建响应时间 | ≤3秒 | 包含业务规则校验 |
| 并发用户数 | ≥500 | 高峰期同时在线用户 |
| 报表生成时间 | ≤30秒 | 复杂统计报表 |

### 4.2 安全需求

| 安全项 | 具体要求 |
|--------|----------|
| 数据隔离 | 多租户数据严格隔离，tenant_id强制校验 |

| 数据加密 | 敏感数据传输加密(HTTPS)，存储加密 |
| 接口安全 | API鉴权、防重放攻击、请求限流 |
