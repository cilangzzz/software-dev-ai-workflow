## 1. 概述

### 1.1 模块定位

采购管理模块是汽车制造业ERP系统的核心模块之一，负责管理企业从原材料、零部件到辅料的全流程采购业务。本模块深度结合汽车制造业JIT/JIS采购模式、供应商分级管理、长协合同管理等行业特点，实现采购业务的数字化转型与智能化管理。

### 1.2 业务背景

汽车制造业采购管理面临以下挑战：

1. **供应链复杂度高**：涉及上千家供应商，零部件种类繁多
2. **JIT/JIS供货要求**：需要精确的到货时间和顺序管理
3. **质量追溯要求严格**：需要完整的采购追溯链条
4. **价格波动频繁**：原材料价格波动需要灵活的价格管理机制
5. **进口件管理复杂**：涉及海关、汇率、长周期等特殊管理

### 1.3 目标用户

| 用户角色 | 职责描述 | 使用场景 |
|----------|----------|----------|
| 采购经理 | 采购策略制定、供应商管理、采购审批 | 供应商评估、采购审批、报表分析 |
| 采购员 | 采购申请处理、订单执行、到货跟踪 | 日常采购操作、订单管理 |
| 供应商 | 接收订单、发货通知、对账 | 供应商门户操作 |
| 仓库管理员 | 收货入库、质检协调 | 收货确认、入库操作 |
| 财务人员 | 采购结算、付款管理 | 发票校验、付款审批 |

## 3. 详细功能需求

### 3.1 采购申请管理

#### 3.1.1 功能描述

采购申请是采购流程的起点，支持多种来源的采购需求，包括MRP运算结果、手工录入、生产计划自动生成等。

#### 3.1.2 功能清单

| 功能项 | 功能描述 | 优先级 |
|--------|----------|--------|
| PR-001 | 采购申请单创建 | P0 |
| PR-002 | 采购申请审批流程 | P0 |
| PR-003 | 申请单合并/拆分 | P1 |
| PR-004 | MRP采购需求导入 | P0 |
| PR-005 | 生产计划联动生成 | P1 |
| PR-006 | 申请执行状态跟踪 | P0 |
| PR-007 | 紧急采购申请 | P1 |
| PR-008 | 进口件采购申请 | P1 |

#### 3.1.3 业务规则

1. **申请编号规则**：PR + 年月 + 流水号，如 PR2024030001
2. **审批规则**：
   - 金额 < 5万：采购主管审批
   - 金额 >= 5万且 < 50万：采购经理审批
   - 金额 >= 50万：采购总监审批
3. **合并规则**：相同供应商、相同交货日期的申请可合并
4. **紧急申请**：标记紧急的申请自动提升审批优先级

#### 3.1.4 数据字段

| 字段名 | 字段类型 | 必填 | 说明 |
|--------|----------|------|------|
| pr_no | varchar(20) | 是 | 申请单号 |
| pr_type | varchar(10) | 是 | 申请类型(普通/JIT/进口) |
| pr_status | varchar(10) | 是 | 状态(草稿/审批中/已审批/已转单/已关闭) |
| applicant_id | bigint | 是 | 申请人ID |
| department_id | bigint | 是 | 申请部门 |
| required_date | date | 是 | 需求日期 |
| reason | varchar(500) | 否 | 申请原因 |
| urgent_flag | tinyint | 否 | 是否紧急(0/1) |
| total_amount | decimal(18,2) | 是 | 申请总金额 |
| tenant_id | bigint | 是 | 租户ID |

### 3.2 采购订单管理

#### 3.2.1 功能描述

采购订单是采购执行的核心单据，记录与供应商的采购合同明细，支持普通订单、JIT订单、JIS订单等多种类型。

#### 3.2.2 功能清单

| 功能项 | 功能描述 | 优先级 |
|--------|----------|--------|
| PO-001 | 采购订单创建(手动/转单) | P0 |
| PO-002 | 采购订单审批流程 | P0 |
| PO-003 | 订单变更管理 | P0 |
| PO-004 | JIT订单管理 | P0 |
| PO-005 | JIS顺序供货订单 | P0 |
| PO-006 | 订单执行跟踪 | P0 |
| PO-007 | 订单打印/导出 | P1 |
| PO-008 | 批量下单 | P1 |
| PO-009 | 订单关闭 | P1 |

#### 3.2.3 业务规则

1. **订单编号规则**：PO + 年月 + 流水号，如 PO2024030001
2. **审批规则**：
   - 单笔订单 < 10万：采购主管审批
   - 单笔订单 >= 10万且 < 100万：采购经理审批
   - 单笔订单 >= 100万：采购总监审批
3. **JIT订单规则**：
   - 必须指定精确到小时的交货时间窗
   - 支持分批次交货计划
4. **JIS订单规则**：
   - 必须关联生产序列号
   - 支持顺序送货标识

#### 3.2.4 JIT/JIS订单特殊要求

| 订单类型 | 特殊字段 | 说明 |
|----------|----------|------|
| JIT订单 | delivery_window | 交货时间窗(精确到小时) |
| JIT订单 | kanban_no | 看板编号 |
| JIT订单 | batch_plan | 分批交货计划 |
| JIS订单 | sequence_no | 顺序号 |
| JIS订单 | production_order | 关联生产订单号 |
| JIS订单 | assembly_point | 装配点 |

### 3.3 供应商管理

供应商管理模块负责供应商全生命周期管理，包括供应商档案维护、分级管理、绩效评估、准入淘汰等。

#### 3.3.2 功能清单

#### 3.3.3 供应商分级标准

| 等级 | 定义 | 采购份额 | 管理策略 |
|------|------|----------|----------|
| A级 | 战略供应商 | 60%-80% | 深度合作，优先开发 |
| B级 | 合格供应商 | 20%-40% | 持续合作，定期评估 |
| C级 | 培育/考察供应商 | <=10% | 限制采购，重点帮扶 |
| D级 | 不合格供应商 | 禁止采购 | 淘汰或整改 |

#### 3.3.4 供应商评估维度

| 评估维度 | 权重 | 评估指标 |
|----------|------|----------|
| 质量水平 | 30% | 来料合格率、PPM值、质量事故次数 |
| 交付能力 | 25% | 准时交货率、紧急响应能力、JIT支持度 |
| 价格水平 | 20% | 价格竞争力、降本贡献、付款条件 |
| 技术能力 | 15% | 研发能力、工艺水平、质量体系 |
| 服务水平 | 10% | 响应速度、问题解决、配合度 |
