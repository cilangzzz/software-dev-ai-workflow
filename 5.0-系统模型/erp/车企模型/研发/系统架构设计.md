## 一、架构概述

### 1.1 系统定位

本系统是基于芋道框架(yudao-aisk-mes)SaaS平台开发的汽车制造业ERP系统，定位于企业计划层，负责企业资源的统筹规划和协调管理。

**系统定位：多租户SaaS模式**，支持集团多工厂、多组织数据隔离与共享。

```
┌─────────────────────────────────────────────────────────────┐
│                      ERP计划层                              │
│           (销售、采购、库存、生产计划、财务)                 │
│                    【多租户SaaS平台】                        │
└─────────────────────────────┬───────────────────────────────┘
                              │ 接口对接
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      MES执行层                              │
│              (工单管理、作业执行、质量追溯)                  │
└─────────────────────────────┬───────────────────────────────┘
                              │ 设备集成
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      设备控制层                             │
│                 (PLC、扭矩枪、扫描枪)                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 模块化 | 功能模块独立，低耦合高内聚 |
| 可扩展 | 支持功能扩展和二次开发 |
| 高可用 | 关键服务冗余，故障自动恢复 |
| 多租户 | 支持多租户数据隔离 |
| 标准化 | 遵循芋道框架规范 |

## 三、技术选型

### 3.1 后端技术栈

| 技术领域 | 技术选型 | 版本 | 说明 |
|----------|----------|------|------|
| 核心框架 | Spring Boot | 2.7.x | 单体应用框架 |
| ORM框架 | MyBatis Plus | 3.5.x | 数据持久化、多租户插件 |
| 数据库 | MySQL | 8.x | 主数据库 |
| 缓存 | Redis | 6.x | 缓存、会话、分布式锁 |
| 消息队列 | RabbitMQ | 3.x | 异步消息处理 |
| 工作流 | Flowable | 6.8.x | 业务流程引擎 |
| 搜索引擎 | Elasticsearch | 7.x | 全文检索、日志分析（可选） |
| 安全框架 | Spring Security | 5.8.x | 认证授权 |
| API文档 | Knife4j | 4.x | 接口文档 |

### 3.2 前端技术栈

| 技术领域 | 技术选型 | 说明 |
|----------|----------|------|
| 管理后台 | Vue3 + Element Plus | PC端管理界面 |
| 移动端 | UniApp | 微信小程序/H5/APP |
| 报表工具 | JimuReport | 报表设计器 |

### 3.3 单体应用架构优势

| 优势 | 说明 |
|------|------|
| 部署简单 | 一个WAR/JAR包部署，无需复杂的微服务治理 |
| 开发效率高 | 模块间调用直接方法调用，调试方便 |
| 运维成本低 | 单一应用监控，日志集中管理 |
| 事务简单 | 本地事务即可满足需求，无需分布式事务 |
| 性能可控 | 无网络开销，性能优化相对简单 |
| 适合规模 | 日活用户<10万，数据量<千万级 |

### 5.2 多租户核心设计要点

| 设计要点 | 实现方式 | 说明 |
|----------|----------|------|
| 数据隔离 | tenant_id字段 | 所有业务表增加tenant_id，MyBatis拦截器自动过滤 |
| 租户上下文 | TenantContextHolder | 请求进入时解析并设置租户上下文 |
| 租户识别 | 域名/请求头/Token | 支持多方式识别租户 |
| 租户配置 | 租户级参数配置 | 每个租户独立的系统参数 |
| 权限隔离 | 租户级角色权限 | 每个租户独立的用户角色体系 |
| 数据库策略 | 共享数据库 | 成本可控，运维简单 |

## 十、技术难点与解决方案

| 技术难点 | 解决方案 |
|----------|----------|
| 多租户数据隔离 | MyBatis Plus多租户插件，自动注入tenant_id |
| 高并发库存操作 | Redis分布式锁 + 数据库乐观锁 |
| 大数据量报表 | 异步生成 + 缓存 + 分页加载 |
| ERP与MES数据同步 | 消息队列 + 定时任务双模式 |
| 分布式事务 | 本地消息表 + 最终一致性 |
