## 1. 设计概述

### 1.1 设计原则

- 遵循数据库范式设计，减少数据冗余
- 支持多租户架构，所有表包含tenant_id字段
- 采用逻辑删除，保留历史数据
- 优化查询性能，合理建立索引
- 支持数据审计，记录创建和修改信息

### 1.2 命名规范

| 对象类型 | 命名规范 | 示例 |
|---------|---------|------|
| 表名 | co_模块名_实体名 | co_cost_element |
| 主键 | id | id |
| 外键 | 关联表_id | cost_center_id |
| 索引 | idx_表名_字段名 | idx_cost_element_code |
| 唯一索引 | uk_表名_字段名 | uk_cost_element_code |

### 1.3 公共字段

所有表包含以下公共字段：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | BIGINT | 主键ID |
| tenant_id | BIGINT | 租户ID |
| create_by | BIGINT | 创建人ID |
| create_time | DATETIME | 创建时间 |
| update_by | BIGINT | 更新人ID |
| update_time | DATETIME | 更新时间 |
| deleted | TINYINT | 删除标志(0-未删除,1-已删除) |
| remark | VARCHAR(500) | 备注 |

| 05 | 销售费用 | 销售部门发生的费用 |

| 编码 | 名称 | 说明 |
|-----|------|------|
| 01 | 生产成本中心 | 直接生产车间 |
| 02 | 辅助成本中心 | 辅助生产车间 |
| 03 | 管理成本中心 | 管理部门 |
| 04 | 销售成本中心 | 销售部门 |

### 4.3 核算状态

| 编码 | 名称 | 说明 |
|-----|------|------|
| 10 | 草稿 | 核算单创建 |
| 20 | 核算中 | 正在核算 |
| 30 | 待审核 | 核算完成待审核 |
| 40 | 已审核 | 审核通过 |
| 50 | 已结转 | 已完成成本结转 |
| 90 | 作废 | 核算单作废 |

### 4.4 差异类型

| 编码 | 名称 | 计算公式 |
|-----|------|---------|
| 01 | 材料价格差异 | (实际价格-标准价格) x 实际数量 |
| 02 | 材料数量差异 | (实际数量-标准数量) x 标准价格 |
| 03 | 人工工资率差异 | (实际工资率-标准工资率) x 实际工时 |
| 04 | 人工效率差异 | (实际工时-标准工时) x 标准工资率 |
| 05 | 变动制造费用差异 | 实际变动制造费用-标准变动制造费用 |
| 06 | 固定制造费用差异 | 实际固定制造费用-标准固定制造费用 |

## 5. 数据迁移策略

### 5.1 历史数据迁移

- 成本要素、成本中心等主数据需要优先迁移
- 历史成本核算数据需要按会计期间迁移
- 标准成本版本数据需要完整迁移
- 迁移数据需要校验数据的完整性和准确性

### 5.2 数据初始化

| 数据类型 | 初始化要求 |
|---------|-----------|
| 成本要素 | 初始化汽车制造业常用成本要素 |
| 成本中心 | 按组织架构初始化成本中心 |
| 标准成本 | 需要手工设置初始标准成本 |
| 标准费率 | 按车间初始化标准费率 |

## 6. 数据安全

### 6.1 数据权限

- 按租户隔离数据，tenant_id字段参与所有查询
- 成本数据属于敏感数据，需要严格控制访问权限
- 支持按部门、成本中心等维度控制数据访问

### 6.2 数据备份

- 每日进行数据库全量备份
- 成本核算数据需要额外归档备份
- 备份数据保留期限：在线数据3年，归档数据10年
