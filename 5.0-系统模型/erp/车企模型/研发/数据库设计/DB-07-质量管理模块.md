# 数据库设计文档 - 质量管理模块(QM)

## 文档信息
- **模块名称**: 质量管理模块 (Quality Management)
- **模块代码**: QM
- **文档版本**: v1.0
- **创建日期**: 2026-03-24
- **技术栈**: Spring Boot 2.7 + MyBatis Plus + MySQL 8.x
- **数据库版本**: MySQL 8.0+

---

## 1. 模块概述

### 1.1 业务背景
质量管理模块是汽车制造业ERP系统的核心模块之一，负责企业全流程质量管控，包括来料检验(IQC)、过程检验(IPQC)、成品检验(FQC)、出货检验(OQC)、不合格处理、质量追溯等功能，确保产品质量符合汽车行业标准。

### 1.2 设计原则
- 支持多租户架构，所有表包含tenant_id字段
- 遵循数据库设计三范式
- 支持软删除机制
- 完善的审计字段设计
- 支持数据追溯和版本控制

### 1.3 核心功能
| 功能模块 | 功能描述 |
|---------|---------|
| 检验标准管理 | 检验标准定义、检验项目维护、检验方案配置 |
| 来料检验(IQC) | 供应商来料质量检验、抽检方案执行 |
| 过程检验(IPQC) | 生产过程质量检验、首件检验、巡检 |
| 成品检验(FQC) | 成品最终检验、入库检验 |
| 出货检验(OQC) | 出货前质量检验、客户特殊要求检验 |
| 不合格处理 | 不合格品管理、返工返修、让步接收 |
| 质量追溯 | 全流程质量追溯、批次追溯、序列号追溯 |
| 质量报表 | 质量统计分析、SPC统计、质量KPI |

## 2. 数据库表设计

### 2.1 检验标准表 (qm_inspection_standard)

**表名**: `qm_inspection_standard`
**描述**: 存储检验标准主数据，定义产品检验的基本标准
**引擎**: InnoDB
**字符集**: utf8mb4

```sql
CREATE TABLE `qm_inspection_standard` (
  `id` bigint(20) NOT NULL COMMENT '主键ID',
  `tenant_id` bigint(20) NOT NULL COMMENT '租户ID',
  `standard_code` varchar(50) NOT NULL COMMENT '标准编码',
  `standard_name` varchar(200) NOT NULL COMMENT '标准名称',
  `product_id` bigint(20) DEFAULT NULL COMMENT '产品ID',
  `product_code` varchar(50) DEFAULT NULL COMMENT '产品编码',
  `product_name` varchar(200) DEFAULT NULL COMMENT '产品名称',
  `standard_type` varchar(20) NOT NULL COMMENT '标准类型:IQC-来料检验,IPQC-过程检验,FQC-成品检验,OQC-出货检验',
  `inspection_level` varchar(20) DEFAULT 'NORMAL' COMMENT '检验等级:NORMAL-正常,STRICT-加严,REDUCED-放宽',
  `aql_level` varchar(20) DEFAULT 'II' COMMENT 'AQL检验水平:I,II,III',
  `aql_value` decimal(10,4) DEFAULT NULL COMMENT 'AQL值(可接受质量限)',
  `sample_scheme` varchar(20) DEFAULT 'SINGLE' COMMENT '抽样方案:SINGLE-单次抽样,DOUBLE-双次抽样,MULTIPLE-多次抽样',
  `effective_date` date DEFAULT NULL COMMENT '生效日期',
  `expiry_date` date DEFAULT NULL COMMENT '失效日期',
  `version` int(11) DEFAULT 1 COMMENT '版本号',
  `status` varchar(20) DEFAULT 'DRAFT' COMMENT '状态:DRAFT-草稿,ACTIVE-生效,INACTIVE-失效',
  `description` text COMMENT '标准描述',
  `attachment_url` varchar(500) DEFAULT NULL COMMENT '附件URL',
  `created_by` bigint(20) DEFAULT NULL COMMENT '创建人',
  `created_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_by` bigint(20) DEFAULT NULL COMMENT '更新人',
  `updated_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` tinyint(1) DEFAULT 0 COMMENT '删除标记:0-未删除,1-已删除',
  `remark` varchar(500) DEFAULT NULL COMMENT '备注',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_standard_code` (`tenant_id`, `standard_code`, `deleted`),
  KEY `idx_tenant_product` (`tenant_id`, `product_id`),
  KEY `idx_tenant_type` (`tenant_id`, `standard_type`),
  KEY `idx_tenant_status` (`tenant_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='检验标准表';
```

## 3. 索引设计说明

### 3.1 索引设计原则
1. 所有表必须包含租户ID(tenant_id)字段，并作为联合索引的第一列
2. 业务主键(如单号、编码)设计唯一索引
3. 外键关联字段设计普通索引
4. 常用查询条件字段设计索引
5. 时间范围查询字段设计索引

### 3.2 索引命名规范
- 主键索引: `PRIMARY KEY`
- 唯一索引: `uk_字段名`
- 普通索引: `idx_字段名`

## 5. 多租户设计

### 5.1 租户隔离策略
- 所有表包含 `tenant_id` 字段，作为租户标识
- 所有唯一索引包含 `tenant_id` 字段
- 数据访问层自动注入租户条件
- 支持租户数据完全隔离

### 5.2 租户数据访问控制
```sql
-- 查询时自动添加租户条件
SELECT * FROM qm_inspection_order
WHERE tenant_id = ? AND deleted = 0;

-- 插入时自动设置租户ID
INSERT INTO qm_inspection_order (id, tenant_id, inspection_no, ...)
VALUES (?, ?, ?, ...);
```
