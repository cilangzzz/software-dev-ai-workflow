## 1. 概述

### 1.1 模块简介

工艺管理模块(Engineering Management)是汽车制造业ERP系统的核心模块之一，负责管理产品结构、工艺路线、工序定义等工程数据，为生产制造提供基础数据支撑。

### 1.2 设计原则

- **多租户架构**：所有表必须包含`tenant_id`字段，支持多租户数据隔离
- **软删除机制**：核心业务表采用逻辑删除，保留历史数据追溯能力
- **审计字段**：统一包含创建人、创建时间、更新人、更新时间字段
- **数据完整性**：通过外键约束和索引保证数据一致性
- **扩展性设计**：预留扩展字段，支持业务定制需求

### 1.3 数据库命名规范

| 对象类型 | 命名规则 | 示例 |
|---------|---------|------|
| 表名 | `eng_`前缀 + 下划线分隔 | eng_product |
| 主键 | `id` | id |
| 外键 | 关联表名 + `_id` | product_id |
| 索引 | `idx_` + 表名 + 字段名 | idx_eng_product_code |
| 唯一索引 | `uk_` + 表名 + 字段名 | uk_eng_product_code |

## 3. 表结构设计

### 3.1 产品主表 (eng_product)

**表说明**：管理产品基础信息，包括成品、半成品、零部件等。

```sql
CREATE TABLE `eng_product` (
  `id` bigint(20) NOT NULL COMMENT '主键ID',
  `tenant_id` bigint(20) NOT NULL COMMENT '租户ID',
  `product_code` varchar(50) NOT NULL COMMENT '产品编码',
  `product_name` varchar(200) NOT NULL COMMENT '产品名称',
  `product_name_en` varchar(200) DEFAULT NULL COMMENT '产品英文名称',
  `product_type` varchar(20) NOT NULL COMMENT '产品类型(FINISHED-成品,SEMI-半成品,PART-零部件,RAW-原材料)',
  `category_id` bigint(20) DEFAULT NULL COMMENT '产品分类ID',
  `brand_id` bigint(20) DEFAULT NULL COMMENT '品牌ID',
  `model_code` varchar(50) DEFAULT NULL COMMENT '车型代码',
  `specification` varchar(500) DEFAULT NULL COMMENT '规格型号',
  `unit_id` bigint(20) DEFAULT NULL COMMENT '基本单位ID',
  `weight` decimal(18,4) DEFAULT NULL COMMENT '重量(kg)',
  `volume` decimal(18,4) DEFAULT NULL COMMENT '体积(m³)',
  `status` varchar(20) NOT NULL DEFAULT 'DRAFT' COMMENT '状态(DRAFT-草稿,ACTIVE-生效,OBSOLETE-作废)',
  `version` int(11) NOT NULL DEFAULT 1 COMMENT '版本号',
  `eff_date` date DEFAULT NULL COMMENT '生效日期',
  `exp_date` date DEFAULT NULL COMMENT '失效日期',
  `safety_stock` decimal(18,4) DEFAULT NULL COMMENT '安全库存',
  `min_order_qty` decimal(18,4) DEFAULT NULL COMMENT '最小订货量',
  `lead_time` int(11) DEFAULT NULL COMMENT '提前期(天)',
  `is_phantom` tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否虚拟件(0-否,1-是)',
  `is_critical` tinyint(1) NOT NULL DEFAULT 0 COMMENT '是否关键件(0-否,1-是)',
  `remark` varchar(1000) DEFAULT NULL COMMENT '备注',
  `creator` varchar(50) NOT NULL COMMENT '创建人',
  `create_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updater` varchar(50) NOT NULL COMMENT '更新人',
  `update_time` datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` tinyint(1) NOT NULL DEFAULT 0 COMMENT '删除标记(0-正常,1-删除)',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_eng_product_code` (`tenant_id`, `product_code`, `deleted`),
  KEY `idx_eng_product_category` (`tenant_id`, `category_id`),
  KEY `idx_eng_product_type` (`tenant_id`, `product_type`),
  KEY `idx_eng_product_status` (`tenant_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='产品主表';
```

## 7. 性能优化建议

### 7.1 分区策略

对于数据量较大的表，建议按租户ID进行分区：

```sql
-- 按租户ID范围分区示例
ALTER TABLE eng_product PARTITION BY RANGE (tenant_id) (
    PARTITION p1 VALUES LESS THAN (1000),
    PARTITION p2 VALUES LESS THAN (2000),
    PARTITION p3 VALUES LESS THAN (3000),
    PARTITION pmax VALUES LESS THAN MAXVALUE
);
```

### 7.2 查询优化

1. **分页查询**：使用游标分页替代offset分页
2. **批量操作**：使用批量插入和更新减少数据库交互
3. **索引覆盖**：设计覆盖索引减少回表查询
4. **查询缓存**：对热点数据使用Redis缓存

### 7.3 数据归档

建议对历史数据进行归档处理：
- 作废状态的产品数据归档
- 超过3年的历史版本数据归档
- 已完成的变更单数据归档
