# 财务管理模块(FI/CO) 数据库设计文档

**文档版本**: V1.0
**创建日期**: 2026-03-24
**文档编号**: DB-05
**关联PRD**: PRD-05-财务管理模块
**数据库版本**: MySQL 8.x

---

## 1. 文档概述

### 1.1 文档目的
本文档详细描述财务管理模块的数据库设计，包括表结构、字段定义、索引设计、约束关系等。

### 1.2 设计原则
- **多租户支持**: 所有业务表必须包含tenant_id字段
- **审计追踪**: 核心表包含create_by、create_time、update_by、update_time字段
- **软删除**: 业务数据使用逻辑删除，保留deleted字段
- **数据完整性**: 通过外键约束和索引保证数据完整性
- **性能优化**: 合理设计索引，支持高并发查询

### 1.3 命名规范
- 表名: 小写字母+下划线，如fi_voucher
- 字段名: 小写字母+下划线，如voucher_no
- 主键: id (BIGINT)
- 索引: idx_表名_字段名
- 唯一索引: uk_表名_字段名

| 字段名 | 数据类型 | 长度 | 允许空 | 默认值 | 说明 |
|--------|----------|------|--------|--------|------|

| account_code | VARCHAR | 50 | N | - | 科目编码 |
| account_name | VARCHAR | 200 | N | - | 科目名称 |
| parent_id | BIGINT | 20 | Y | NULL | 父科目ID |

| account_type | TINYINT | 4 | N | - | 科目类型(1资产2负债3权益4成本5损益) |
| balance_direction | TINYINT | 4 | N | - | 余额方向(1借2贷) |
| is_leaf | TINYINT | 1 | N | 0 | 是否末级科目(0否1是) |
| is_enabled | TINYINT | 1 | N | 1 | 是否启用(0否1是) |
| aux_customer | TINYINT | 1 | N | 0 | 客户辅助核算(0否1是) |
| aux_supplier | TINYINT | 1 | N | 0 | 供应商辅助核算(0否1是) |

| aux_project | TINYINT | 1 | N | 0 | 项目辅助核算(0否1是) |
| aux_employee | TINYINT | 1 | N | 0 | 人员辅助核算(0否1是) |
| aux_item | TINYINT | 1 | N | 0 | 自定义项辅助核算(0否1是) |
| remark | VARCHAR | 500 | Y | NULL | 备注 |
| deleted | TINYINT | 1 | N | 0 | 删除标记(0否1是) |
| create_by | BIGINT | 20 | N | - | 创建人 |

**索引设计**:
```sql
PRIMARY KEY (id)
UNIQUE KEY uk_account_code (tenant_id, account_code, deleted)
KEY idx_parent_id (parent_id)
KEY idx_account_type (account_type)
KEY idx_tenant_id (tenant_id)
```

**建表语句**:
```sql
CREATE TABLE `fi_account` (
  `id` BIGINT(20) NOT NULL COMMENT '主键ID',
  `tenant_id` BIGINT(20) NOT NULL COMMENT '租户ID',
  `account_code` VARCHAR(50) NOT NULL COMMENT '科目编码',
  `account_name` VARCHAR(200) NOT NULL COMMENT '科目名称',
  `parent_id` BIGINT(20) DEFAULT NULL COMMENT '父科目ID',
  `account_level` TINYINT(4) NOT NULL DEFAULT 1 COMMENT '科目级次',
  `account_type` TINYINT(4) NOT NULL COMMENT '科目类型(1资产2负债3权益4成本5损益)',
  `balance_direction` TINYINT(4) NOT NULL COMMENT '余额方向(1借2贷)',
  `is_leaf` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '是否末级科目(0否1是)',
  `is_enabled` TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用(0否1是)',
  `aux_customer` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '客户辅助核算(0否1是)',
  `aux_supplier` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '供应商辅助核算(0否1是)',
  `aux_department` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '部门辅助核算(0否1是)',
  `aux_project` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '项目辅助核算(0否1是)',
  `aux_employee` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '人员辅助核算(0否1是)',
  `aux_item` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '自定义项辅助核算(0否1是)',
  `remark` VARCHAR(500) DEFAULT NULL COMMENT '备注',
  `deleted` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '删除标记(0否1是)',
  `create_by` BIGINT(20) NOT NULL COMMENT '创建人',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_by` BIGINT(20) DEFAULT NULL COMMENT '更新人',
  `update_time` DATETIME DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_account_code` (`tenant_id`, `account_code`, `deleted`),
  KEY `idx_parent_id` (`parent_id`),
  KEY `idx_account_type` (`account_type`),
  KEY `idx_tenant_id` (`tenant_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='会计科目表';
```
