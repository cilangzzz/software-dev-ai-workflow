# 数据库设计文档 - 采购管理模块(MM)

## 文档信息
- **文档编号**: DB-02
- **模块名称**: 采购管理模块(Material Management)
- **版本号**: V1.0.0
- **创建日期**: 2026-03-24
- **技术栈**: Spring Boot 2.7 + MyBatis Plus + MySQL 8.x

---

## 1. 概述

### 1.1 模块说明
采购管理模块(MM)是汽车制造业ERP系统的核心模块之一，负责管理供应商资源、采购申请、采购订单、采购入库等业务流程，实现采购业务的数字化和规范化管理。

### 1.2 设计原则
- 所有表必须包含`tenant_id`字段支持多租户架构
- 采用逻辑删除机制，所有表包含`deleted`字段
- 所有表包含审计字段：`create_by`, `create_time`, `update_by`, `update_time`
- 主键采用雪花算法生成的BIGINT类型
- 状态字段使用TINYINT类型
- 金额字段使用DECIMAL(18,4)类型保证精度

### 1.3 数据库命名规范
- 表名前缀：`mm_` (Material Management)
- 索引命名：`idx_表名_字段名`
- 唯一索引：`uk_表名_字段名`

## 4. 存储过程设计

### 4.1 生成采购申请单号

```sql
DELIMITER //
CREATE PROCEDURE sp_generate_purchase_request_code(
    IN p_tenant_id BIGINT,
    OUT p_request_code VARCHAR(50)
)
BEGIN
    DECLARE v_date_str VARCHAR(8);
    DECLARE v_seq INT;
    DECLARE v_prefix VARCHAR(20);

    SET v_date_str = DATE_FORMAT(NOW(), '%Y%m%d');
    SET v_prefix = CONCAT('PR', v_date_str);

    SELECT IFNULL(MAX(CAST(SUBSTRING(request_code, 11) AS UNSIGNED)), 0) + 1
    INTO v_seq
    FROM mm_purchase_request
    WHERE tenant_id = p_tenant_id
      AND request_code LIKE CONCAT(v_prefix, '%')
      AND deleted = 0;

    SET p_request_code = CONCAT(v_prefix, LPAD(v_seq, 4, '0'));
END //
DELIMITER ;
```

### 4.3 采购订单状态更新

```sql
DELIMITER //
CREATE PROCEDURE sp_update_purchase_order_status(
    IN p_order_id BIGINT,
    IN p_tenant_id BIGINT
)
BEGIN
    DECLARE v_total_qty DECIMAL(18,4);
    DECLARE v_received_qty DECIMAL(18,4);
    DECLARE v_current_status TINYINT;

    SELECT total_qty, status
    INTO v_total_qty, v_current_status
    FROM mm_purchase_order
    WHERE id = p_order_id AND tenant_id = p_tenant_id;

    SELECT IFNULL(SUM(received_qty), 0)
    INTO v_received_qty
    FROM mm_purchase_order_detail
    WHERE order_id = p_order_id AND tenant_id = p_tenant_id AND deleted = 0;

    IF v_received_qty = 0 AND v_current_status = 4 THEN
        -- 状态保持为已发布
        -- 不做更新
    ELSEIF v_received_qty > 0 AND v_received_qty < v_total_qty THEN
        -- 部分入库
        UPDATE mm_purchase_order
        SET status = 5, update_time = NOW()
        WHERE id = p_order_id AND tenant_id = p_tenant_id;
    ELSEIF v_received_qty >= v_total_qty THEN
        -- 完全入库
        UPDATE mm_purchase_order
        SET status = 6, update_time = NOW()
        WHERE id = p_order_id AND tenant_id = p_tenant_id;
    END IF;
END //
DELIMITER ;
```

## 8. 性能优化建议

### 8.1 索引优化
- 为高频查询字段建立组合索引
- 定期分析索引使用率，移除冗余索引
- 大表考虑分区策略

### 8.2 查询优化
- 避免使用SELECT *，只查询需要的字段
- 合理使用分页查询
- 使用EXPLAIN分析慢查询

### 8.3 数据归档
- 历史订单数据定期归档
- 归档数据保留查询接口
- 建立归档数据表

### 9.1 多租户数据隔离
