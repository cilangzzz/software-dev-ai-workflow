# 生产计划模块(PP)数据库设计文档

## 文档信息
- 文档编号: DB-04
- 模块名称: 生产计划模块 (Production Planning - PP)
- 版本: V1.0
- 创建日期: 2026-03-24
- 数据库: MySQL 8.x

---

## 1. 概述

### 1.1 模块说明
生产计划模块(PP)是汽车制造业ERP系统的核心模块，负责生产计划的制定、物料需求计算、生产排程等功能。该模块与MES系统深度集成，实现计划与执行的闭环管理。

### 1.2 设计原则
- 支持多租户架构，所有表必须包含tenant_id字段
- 采用逻辑删除，保留历史数据
- 支持多工厂、多车间管理
- 与MES系统集成，支持计划下发和执行反馈

### 1.3 数据库命名规范
- 表名前缀: pp_ (Production Planning)
- 主键: id (BIGINT, 雪花算法生成)
- 租户ID: tenant_id (BIGINT)
- 创建时间: create_time
- 更新时间: update_time
- 创建人: create_by
- 更新人: update_by
- 删除标记: del_flag (0-未删除, 1-已删除)

## 2. 表结构设计

### 2.1 主生产计划表 (pp_master_production_schedule)

主生产计划(MPS)是生产计划体系的核心，确定最终产品的生产数量和时间。

```sql
CREATE TABLE pp_master_production_schedule (
    id                  BIGINT          NOT NULL COMMENT '主键ID',
    tenant_id           BIGINT          NOT NULL COMMENT '租户ID',
    schedule_no         VARCHAR(50)     NOT NULL COMMENT '计划编号',
    schedule_name       VARCHAR(200)    NOT NULL COMMENT '计划名称',
    plan_type           VARCHAR(20)     NOT NULL COMMENT '计划类型(DAILY-日计划,WEEKLY-周计划,MONTHLY-月计划)',
    factory_id          BIGINT          NOT NULL COMMENT '工厂ID',
    workshop_id         BIGINT          NULL COMMENT '车间ID',
    plan_start_date     DATE            NOT NULL COMMENT '计划开始日期',
    plan_end_date       DATE            NOT NULL COMMENT '计划结束日期',
    status              VARCHAR(20)     NOT NULL DEFAULT 'DRAFT' COMMENT '状态(DRAFT-草稿,SUBMITTED-已提交,APPROVED-已审批,RELEASED-已发布,CLOSED-已关闭)',
    version             INT             NOT NULL DEFAULT 1 COMMENT '版本号',
    total_quantity      DECIMAL(18,4)   NOT NULL DEFAULT 0 COMMENT '计划总数量',
    total_amount         DECIMAL(18,2)   NOT NULL DEFAULT 0 COMMENT '计划总金额',
    remark              VARCHAR(500)    NULL COMMENT '备注',
    approve_by          BIGINT          NULL COMMENT '审批人ID',
    approve_time        DATETIME        NULL COMMENT '审批时间',
    release_by          BIGINT          NULL COMMENT '发布人ID',
    release_time        DATETIME        NULL COMMENT '发布时间',
    create_time         DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time         DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    create_by           BIGINT          NOT NULL COMMENT '创建人',
    update_by           BIGINT          NOT NULL COMMENT '更新人',
    del_flag            TINYINT         NOT NULL DEFAULT 0 COMMENT '删除标记(0-未删除,1-已删除)',
    PRIMARY KEY (id),
    INDEX idx_tenant_id (tenant_id),
    INDEX idx_schedule_no (tenant_id, schedule_no),
    INDEX idx_factory_date (tenant_id, factory_id, plan_start_date),
    INDEX idx_status (tenant_id, status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='主生产计划表';
```

## 3. 索引设计说明

### 3.1 索引设计原则
1. 所有表必须包含tenant_id字段作为索引的第一列，支持多租户查询
