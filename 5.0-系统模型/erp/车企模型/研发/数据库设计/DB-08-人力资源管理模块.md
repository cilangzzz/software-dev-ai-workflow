## 1. 设计概述

### 1.1 设计原则

- 遵循第三范式(3NF)，减少数据冗余
- 所有表包含tenant_id字段支持多租户
- 主键采用雪花算法生成的BIGINT类型
- 敏感数据(薪资、身份证号)加密存储
- 软删除设计，deleted字段标记删除状态
- 完善的索引设计，支撑高性能查询

### 1.2 命名规范

| 类型 | 命名规范 | 示例 |
|------|----------|------|
| 表名 | hr_模块_表名，小写下划线分隔 | hr_org_department |
| 字段名 | 小写下划线分隔 | employee_name |
| 主键 | id | id |
| 外键 | 关联表名_id | department_id |
| 索引 | idx_表名_字段名 | idx_employee_name |

### 1.3 公共字段

所有表包含以下公共字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 主键，雪花算法生成 |
| tenant_id | BIGINT | 租户ID |
| created_by | BIGINT | 创建人ID |
| created_time | DATETIME | 创建时间 |
| updated_by | BIGINT | 更新人ID |
| updated_time | DATETIME | 更新时间 |
| deleted | TINYINT | 删除标记(0:未删除,1:已删除) |
| version | INT | 乐观锁版本号 |

## 2. 组织管理表设计

### 2.1 组织单元表 (hr_org_department)

```sql
CREATE TABLE `hr_org_department` (
  `id` BIGINT NOT NULL COMMENT '主键ID',
  `tenant_id` BIGINT NOT NULL COMMENT '租户ID',
  `dept_code` VARCHAR(50) NOT NULL COMMENT '组织编码',

  `dept_type` VARCHAR(20) NOT NULL COMMENT '组织类型(company:公司,division:事业部,department:部门,group:班组)',

  `sort_order` INT DEFAULT 0 COMMENT '排序号',
  `status` TINYINT DEFAULT 1 COMMENT '状态(0:禁用,1:启用)',
  `remark` VARCHAR(500) DEFAULT NULL COMMENT '备注',
  `created_by` BIGINT DEFAULT NULL COMMENT '创建人',
  `created_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_by` BIGINT DEFAULT NULL COMMENT '更新人',
  `updated_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` TINYINT DEFAULT 0 COMMENT '删除标记(0:未删除,1:已删除)',
```
