# 数据库设计文档 - 设备资产管理模块(EAM)

## 文档信息
- **模块名称**: 设备资产管理模块 (Equipment Asset Management)
- **模块编码**: EAM
- **文档版本**: V1.0
- **创建日期**: 2024-03-24
- **技术栈**: Spring Boot 2.7 + MyBatis Plus + MySQL 8.x
- **数据库版本**: MySQL 8.0+

---

## 1. 模块概述

### 1.1 功能范围
设备资产管理模块(EAM)是汽车制造业ERP系统的核心模块之一，负责管理企业全生命周期的设备资产，包括：
- 设备台账管理
- 设备维护计划管理
- 设备维修记录管理
- 备品备件管理
- 设备巡检管理
- 设备状态监控
- 设备折旧管理

### 1.2 设计原则
- 支持多租户架构，所有表必须包含 `tenant_id` 字段
- 遵循第三范式(3NF)设计，适度反范式优化查询性能
- 采用逻辑删除，保留历史数据追溯能力
- 支持多工厂、多车间层级管理
- 预留扩展字段，支持自定义属性

## 3. 详细表结构设计

### 3.1 设备分类表 (eam_equipment_category)

```sql
CREATE TABLE `eam_equipment_category` (
  `id` bigint(20) NOT NULL COMMENT '主键ID',
  `tenant_id` bigint(20) NOT NULL COMMENT '租户ID',
  `category_code` varchar(50) NOT NULL COMMENT '分类编码',
  `category_name` varchar(100) NOT NULL COMMENT '分类名称',
  `parent_id` bigint(20) DEFAULT 0 COMMENT '父分类ID',
  `category_level` tinyint(4) DEFAULT 1 COMMENT '分类层级(1-一级,2-二级,3-三级)',
  `category_path` varchar(500) DEFAULT NULL COMMENT '分类路径(逗号分隔ID)',
  `sort_order` int(11) DEFAULT 0 COMMENT '排序号',
  `status` tinyint(4) DEFAULT 1 COMMENT '状态(1-启用,0-禁用)',
  `remark` varchar(500) DEFAULT NULL COMMENT '备注',
  `create_by` varchar(64) DEFAULT NULL COMMENT '创建人',
  `create_time` datetime DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_by` varchar(64) DEFAULT NULL COMMENT '更新人',
  `update_time` datetime DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted` tinyint(4) DEFAULT 0 COMMENT '删除标志(0-正常,1-删除)',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_tenant_code` (`tenant_id`, `category_code`, `deleted`),
  KEY `idx_tenant_parent` (`tenant_id`, `parent_id`),
  KEY `idx_tenant_status` (`tenant_id`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='设备分类表';
```

## 4. 索引设计说明

### 4.1 索引设计原则
1. **租户索引**: 所有表都必须包含 `tenant_id` 作为联合索引的第一列
2. **唯一索引**: 业务编码字段配置租户级唯一索引，包含 `deleted` 字段
3. **外键索引**: 关联字段建立索引，提高JOIN查询性能
4. **时间索引**: 时间范围查询字段建立索引
5. **状态索引**: 状态查询字段建立索引

### 4.2 索引命名规范
- 主键索引: `PRIMARY KEY`
- 唯一索引: `uk_字段名`
- 普通索引: `idx_字段名`
