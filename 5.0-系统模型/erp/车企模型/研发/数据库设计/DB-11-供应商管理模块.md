## 1. 设计概述

### 1.1 设计原则

1. **多租户支持**：所有业务表必须包含`tenant_id`字段，支持SaaS多租户架构
2. **软删除机制**：核心业务表采用逻辑删除，保留数据可追溯性
3. **审计字段**：统一包含创建人、创建时间、修改人、修改时间字段
4. **数据一致性**：采用外键约束和索引保证数据完整性
5. **性能优化**：针对高频查询场景设计合理的索引策略

### 1.2 数据库命名规范

| 对象类型 | 命名规则 | 示例 |
|----------|----------|------|
| 表名 | srm_模块_业务名称 | srm_supplier_master |
| 主键 | pk_表名 | pk_srm_supplier_master |
| 外键 | fk_表名_关联表 | fk_srm_supplier_master_org |
| 索引 | idx_表名_字段名 | idx_srm_supplier_master_code |
| 唯一索引 | uk_表名_字段名 | uk_srm_supplier_master_code |

## 2. 表结构设计

### 2.1 供应商档案主表 (srm_supplier_master)

**功能描述**：存储供应商基本信息，是供应商管理的核心主表。

```sql
CREATE TABLE srm_supplier_master (
    id                  BIGINT          NOT NULL COMMENT '主键ID',
    tenant_id           BIGINT          NOT NULL COMMENT '租户ID',
    supplier_code       VARCHAR(50)     NOT NULL COMMENT '供应商编码',
    supplier_name       VARCHAR(200)    NOT NULL COMMENT '供应商名称',
    supplier_short_name VARCHAR(100)    DEFAULT NULL COMMENT '供应商简称',
    supplier_type       VARCHAR(20)     NOT NULL COMMENT '供应商类型(01-原材料,02-零部件,03-辅料,04-设备,05-服务)',
    supplier_level      VARCHAR(10)     DEFAULT 'C' COMMENT '供应商等级(A/B/C/D)',
    status              VARCHAR(10)     DEFAULT '01' COMMENT '状态(01-潜在,02-认证中,03-合格,04-暂停,05-淘汰)',
    company_type        VARCHAR(20)     DEFAULT NULL COMMENT '企业性质(01-国企,02-民企,03-外资,04-合资)',
    registered_capital  DECIMAL(18,2)   DEFAULT NULL COMMENT '注册资本(万元)',
    established_date    DATE            DEFAULT NULL COMMENT '成立日期',
    legal_person        VARCHAR(50)     DEFAULT NULL COMMENT '法人代表',
    business_scope      VARCHAR(1000)   DEFAULT NULL COMMENT '经营范围',
    province            VARCHAR(50)     DEFAULT NULL COMMENT '省份',
    city                VARCHAR(50)     DEFAULT NULL COMMENT '城市',
    district            VARCHAR(50)     DEFAULT NULL COMMENT '区县',
    address             VARCHAR(500)    DEFAULT NULL COMMENT '详细地址',
    postal_code         VARCHAR(20)     DEFAULT NULL COMMENT '邮政编码',
    website             VARCHAR(200)    DEFAULT NULL COMMENT '企业网址',
    credit_code         VARCHAR(50)     DEFAULT NULL COMMENT '统一社会信用代码',
    tax_no              VARCHAR(50)     DEFAULT NULL COMMENT '纳税人识别号',
    bank_name           VARCHAR(100)    DEFAULT NULL COMMENT '开户银行',
    bank_account        VARCHAR(50)     DEFAULT NULL COMMENT '银行账号',
    contact_person      VARCHAR(50)     DEFAULT NULL COMMENT '联系人',
    contact_phone       VARCHAR(30)     DEFAULT NULL COMMENT '联系电话',
    contact_mobile      VARCHAR(20)     DEFAULT NULL COMMENT '移动电话',
    contact_email       VARCHAR(100)    DEFAULT NULL COMMENT '联系邮箱',
    contact_fax         VARCHAR(30)     DEFAULT NULL COMMENT '传真号码',
    org_id              BIGINT          DEFAULT NULL COMMENT '所属组织ID',
    purchaser_id        BIGINT          DEFAULT NULL COMMENT '采购员ID',
    payment_terms       VARCHAR(50)     DEFAULT NULL COMMENT '付款条款',
    delivery_cycle      INT             DEFAULT NULL COMMENT '交货周期(天)',
    min_order_qty       INT             DEFAULT NULL COMMENT '最小订货量',
    currency            VARCHAR(10)      DEFAULT 'CNY' COMMENT '结算币种',
    tax_rate            DECIMAL(5,2)    DEFAULT NULL COMMENT '税率(%)',
    is_strategic        TINYINT(1)      DEFAULT 0 COMMENT '是否战略供应商(0-否,1-是)',
    is_active           TINYINT(1)      DEFAULT 1 COMMENT '是否有效(0-否,1-是)',
    remark              VARCHAR(500)    DEFAULT NULL COMMENT '备注',
    create_by           VARCHAR(50)      NOT NULL COMMENT '创建人',
    create_time         DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by           VARCHAR(50)      DEFAULT NULL COMMENT '修改人',
    update_time         DATETIME        DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    delete_flag         TINYINT(1)      DEFAULT 0 COMMENT '删除标志(0-正常,1-已删除)',
    version             INT             DEFAULT 0 COMMENT '版本号',
    PRIMARY KEY (id),
    UNIQUE KEY uk_srm_supplier_master_code (tenant_id, supplier_code, delete_flag),
    KEY idx_srm_supplier_master_tenant (tenant_id),
    KEY idx_srm_supplier_master_name (supplier_name),
    KEY idx_srm_supplier_master_status (status),
    KEY idx_srm_supplier_master_type (supplier_type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='供应商档案主表';
```

### 2.2 供应商资质信息表 (srm_supplier_qualification)

    valid_end_date          DATE            DEFAULT NULL COMMENT '有效期结束',
    qual_status             VARCHAR(10)     DEFAULT '01' COMMENT '资质状态(01-有效,02-即将过期,03-已过期,04-已作废)',

    attachment_id           BIGINT          DEFAULT NULL COMMENT '附件ID',
    attachment_url          VARCHAR(500)    DEFAULT NULL COMMENT '附件URL',
    remind_days             INT             DEFAULT 30 COMMENT '提前提醒天数',

    create_time             DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_by               VARCHAR(50)     DEFAULT NULL COMMENT '修改人',
    update_time             DATETIME        DEFAULT NULL ON UPDATE CURRENT_TIMESTAMP COMMENT '修改时间',
    delete_flag             TINYINT(1)      DEFAULT 0 COMMENT '删除标志(0-正常,1-已删除)',
    PRIMARY KEY (id),
    KEY idx_srm_supplier_qual_tenant (tenant_id),
    KEY idx_srm_supplier_qual_supplier (supplier_id),
    KEY idx_srm_supplier_qual_type (qual_type),
    KEY idx_srm_supplier_qual_status (qual_status),

    cost_score              DECIMAL(5,2)    DEFAULT 0 COMMENT '成本得分(权重15%)',
    service_score           DECIMAL(5,2)    DEFAULT 0 COMMENT '服务得分(权重10%)',

    assessor_name           VARCHAR(50)     DEFAULT NULL COMMENT '考核人姓名',

    UNIQUE KEY uk_srm_perf_master_period (tenant_id, supplier_id, assess_period, delete_flag),

## 4. 索引设计

### 4.1 索引设计原则

1. **主键索引**：所有表使用`id`作为主键，采用BIGINT类型
2. **租户索引**：所有业务表必须包含`tenant_id`索引
3. **外键索引**：外键字段必须建立索引
4. **组合索引**：针对高频查询场景设计组合索引
5. **唯一索引**：业务唯一键建立唯一索引

### 4.2 关键索引列表

| 表名 | 索引名 | 索引字段 | 索引类型 | 说明 |
|------|--------|----------|----------|------|
| srm_supplier_master | uk_srm_supplier_master_code | tenant_id, supplier_code, delete_flag | UNIQUE | 租户内编码唯一 |

## 6. 存储过程设计

### 6.1 供应商绩效自动评分存储过程

```sql
DELIMITER //
CREATE PROCEDURE sp_calculate_supplier_score(
    IN p_tenant_id BIGINT,
    IN p_supplier_id BIGINT,
    IN p_assess_period VARCHAR(20)
)
BEGIN
    DECLARE v_quality_score DECIMAL(5,2) DEFAULT 0;
    DECLARE v_delivery_score DECIMAL(5,2) DEFAULT 0;
    DECLARE v_cost_score DECIMAL(5,2) DEFAULT 0;
    DECLARE v_service_score DECIMAL(5,2) DEFAULT 0;
    DECLARE v_tech_score DECIMAL(5,2) DEFAULT 0;
    DECLARE v_total_score DECIMAL(5,2) DEFAULT 0;

    -- 计算质量得分(示例)
    SELECT IFNULL(AVG(weighted_score), 0) INTO v_quality_score
    FROM srm_performance_detail
    WHERE tenant_id = p_tenant_id
      AND supplier_id = p_supplier_id
      AND assess_period = p_assess_period
      AND indicator_category = '01';

    -- 计算交付得分(示例)
    SELECT IFNULL(AVG(weighted_score), 0) INTO v_delivery_score
    FROM srm_performance_detail
    WHERE tenant_id = p_tenant_id
      AND supplier_id = p_supplier_id
      AND assess_period = p_assess_period
      AND indicator_category = '02';

    -- 计算总得分
    SET v_total_score = v_quality_score + v_delivery_score + v_cost_score
                      + v_service_score + v_tech_score;

    -- 更新主表得分
    UPDATE srm_performance_master
    SET quality_score = v_quality_score,
        delivery_score = v_delivery_score,
        cost_score = v_cost_score,
        service_score = v_service_score,
        tech_score = v_tech_score,
        total_score = v_total_score,
        update_time = NOW()
    WHERE tenant_id = p_tenant_id
      AND supplier_id = p_supplier_id
      AND assess_period = p_assess_period;
END //
DELIMITER ;
```

### 6.2 资质到期提醒存储过程

```sql
DELIMITER //
CREATE PROCEDURE sp_qualification_expire_remind(
    IN p_tenant_id BIGINT,
    IN p_remind_days INT
)
BEGIN
    -- 更新即将过期的资质状态
    UPDATE srm_supplier_qualification
    SET qual_status = '02',
        update_time = NOW()
    WHERE tenant_id = p_tenant_id
      AND delete_flag = 0
      AND qual_status = '01'
      AND valid_end_date BETWEEN CURDATE() AND DATE_ADD(CURDATE(), INTERVAL p_remind_days DAY);

    -- 更新已过期的资质状态
    UPDATE srm_supplier_qualification
    SET qual_status = '03',
        update_time = NOW()
    WHERE tenant_id = p_tenant_id
      AND delete_flag = 0
      AND qual_status IN ('01', '02')
      AND valid_end_date < CURDATE();
END //
DELIMITER ;
```
