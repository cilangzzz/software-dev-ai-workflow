## 1. 设计概述

### 1.1 设计原则

- 遵循第三范式，减少数据冗余
- 支持多租户架构，所有表包含tenant_id字段
- 主键采用雪花算法生成的BIGINT类型
- 包含创建时间、更新时间、创建人、更新人等审计字段
- 采用逻辑删除，使用is_deleted字段标识

### 1.2 命名规范

- 表名：小写字母，单词间用下划线分隔，格式为 wm_xxx
- 字段名：小写字母，单词间用下划线分隔
- 主键：id
- 索引：idx_表名_字段名
- 唯一索引：uk_表名_字段名

### 1.3 公共字段

所有表包含以下公共字段：

| 字段名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| id | BIGINT | - | 主键，雪花算法生成 |
| tenant_id | BIGINT | - | 租户ID |
| create_time | DATETIME | CURRENT_TIMESTAMP | 创建时间 |
| create_by | VARCHAR(64) | - | 创建人 |
| update_time | DATETIME | CURRENT_TIMESTAMP | 更新时间 |
| update_by | VARCHAR(64) | - | 更新人 |
| is_deleted | TINYINT | 0 | 删除标识：0-未删除，1-已删除 |
| remark | VARCHAR(500) | - | 备注 |
