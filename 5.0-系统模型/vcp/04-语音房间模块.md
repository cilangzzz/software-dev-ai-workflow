# 语音房间模块 (VCR)

## 功能概述

语音房间是VCP系统的核心功能，用户进入语音频道后自动创建/加入语音房间，实现多人实时语音通话。基于WebRTC的SFU架构实现低延迟音频传输。

## 功能清单

| 序号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 1 | 多人语音通话 | P0 | 基于WebRTC的实时音频 |
| 2 | 麦克风/扬声器控制 | P0 | 静音、调节音量 |
| 3 | 语音房间进入/退出 | P0 | 自由进出 |
| 4 | 发言人高亮显示 | P0 | 当前说话者视觉标识 |
| 5 | 语音质量调节 | P1 | 采样率、降噪 |
| 6 | 屏幕共享 | P2 | 后续排期 |
| 7 | 视频聊天 | P2 | 后续排期 |

## 详细需求

### 4.1 多人语音通话

**功能描述：**
用户连接语音频道后进入语音房间，支持最多20人同时在线语音通话。

**技术架构：**
```
客户端A ─┐
客户端B ─┤
客户端C ─┼──→ SFU服务器 ──→ 音频路由转发
客户端D ─┤
客户端E ─┘
```

**业务规则：**
- 单房间最大20人（后续优化扩大）
- 支持Opus音频编码
- 支持48kHz采样率
- 支持自动增益控制（AGC）
- 支持回声消除（AEC）
- 支持噪声抑制（ANS）
- 音频延迟 < 200ms

**数据表：voice_room**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| channel_id | BIGINT | 是 | 关联的语音频道ID |
| server_id | BIGINT | 是 | 服务器ID |
| max_users | INT | 是 | 最大用户数（默认20） |
| current_users | INT | 是 | 当前用户数 |
| status | TINYINT | 是 | 状态：0-空闲 1-活跃 |
| created_at | DATETIME | 是 | 创建时间 |
| ended_at | DATETIME | 否 | 结束时间 |

### 4.2 麦克风/扬声器控制

**功能描述：**
用户可控制自己的麦克风和扬声器，管理员可静音其他成员。

**用户操作：**
- **自我静音** - 关闭自己的麦克风
- **自我闭麦** - 关闭扬声器（不听）
- **调节音量** - 调整自己的收听音量
- **调节单人音量** - 调整某一个成员的音量

**管理员操作：**
- **静音他人** - 管理员静音某个成员
- **屏蔽他人音频** - 管理员屏蔽某个成员的音频

**数据表：voice_room_member**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| room_id | BIGINT | 是 | 语音房间ID |
| user_id | BIGINT | 是 | 用户ID |
| is_muted | TINYINT | 是 | 是否自我静音 |
| is_deafened | TINYINT | 是 | 是否自我闭麦 |
| is_server_muted | TINYINT | 是 | 是否被管理员静音 |
| is_speaking | TINYINT | 是 | 是否正在说话 |
| volume | INT | 是 | 音量（0-100） |
| joined_at | DATETIME | 是 | 加入时间 |

### 4.3 语音房间进入/退出

**功能描述：**
用户自由进出语音房间，系统实时同步房间状态。

**进入流程：**
```
用户点击语音频道
    ↓
检查权限（是否有CONNECT权限）
    ↓
检查房间容量（是否已满）
    ↓
请求麦克风权限
    ↓
WebRTC连接建立
    ↓
加入语音房间
    ↓
通知其他成员
```

**退出方式：**
- 主动断开连接
- 关闭客户端（自动断开）
- 被管理员踢出语音
- 切换到其他语音房间（自动离开当前）

### 4.4 发言人高亮显示

**功能描述：**
正在说话的用户在界面上有明显的视觉标识。

**显示方式：**
- 用户头像外圈高亮边框
- 用户名颜色变化
- 发言人排在列表前列
- 实时音量指示条

### 4.5 语音质量调节

**功能描述：**
用户可根据网络状况调节语音质量。

**可调节项：**
- 音频采样率（16kHz / 48kHz）
- 降噪等级（关闭 / 低 / 中 / 高）
- 自动增益控制开关
- 回声消除开关

## 接口设计（概要）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/voice/room/{channelId}/join | POST | 加入语音房间 |
| /api/voice/room/{channelId}/leave | POST | 离开语音房间 |
| /api/voice/room/{channelId}/members | GET | 获取房间成员 |
| /api/voice/room/{channelId}/mute | POST | 自我静音 |
| /api/voice/room/{channelId}/unmute | POST | 取消自我静音 |
| /api/voice/room/{channelId}/deafen | POST | 自我闭麦 |
| /api/voice/room/{channelId}/undeafen | POST | 取消自我闭麦 |
| /api/voice/room/{channelId}/server-mute/{userId} | POST | 管理员静音某人 |
| /api/voice/room/{channelId}/server-unmute/{userId} | POST | 管理员取消静音 |
| /api/voice/quality | PUT | 设置语音质量参数 |

### 信令接口（WebSocket）

| 事件 | 方向 | 说明 |
|------|------|------|
| voice:join | Client → Server | 用户加入房间 |
| voice:leave | Client → Server | 用户离开房间 |
| voice:offer | Client ↔ Server | WebRTC SDP Offer |
| voice:answer | Client ↔ Server | WebRTC SDP Answer |
| voice:ice-candidate | Client ↔ Server | ICE候选交换 |
| voice:mute | Client → Server | 静音状态变更 |
| voice:speaking | Server → Client | 发言状态通知 |

## 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 麦克风权限被拒绝 | 提示"需要麦克风权限才能语音" |
| 房间已满 | 提示"语音房间已满" |
| 网络断开 | 30秒内自动重连，超时提示重试 |
| WebRTC连接失败 | 尝试TURN服务器中继 |
| 音频设备异常 | 提示检查音频设备 |

## 验收标准

- [ ] 用户可连接语音频道进入语音房间
- [ ] 支持最多20人同时语音通话
- [ ] 音频延迟 < 200ms
- [ ] 用户可自我静音/闭麦
- [ ] 管理员可静音其他成员
- [ ] 发言人有高亮视觉标识
- [ ] 切换语音房间时自动离开前一个
- [ ] 关闭客户端时自动断开语音
- [ ] 网络波动时有自动重连机制
