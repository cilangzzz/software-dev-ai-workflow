# 服务器/社区模块 (SVC)

## 功能概述

服务器（Server/社区）是VCP系统的核心组织单元，用户创建服务器后可在其下创建频道、管理成员和角色权限。

## 功能清单

| 序号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 1 | 创建服务器 | P0 | 用户创建自己的社区 |
| 2 | 服务器设置 | P0 | 名称、图标、描述、区域设置 |
| 3 | 邀请机制 | P0 | 生成邀请链接/邀请码 |
| 4 | 成员管理 | P0 | 成员加入/退出/踢出 |
| 5 | 角色与权限管理 | P0 | 创建角色、分配权限、层级管理 |

## 详细需求

### 2.1 创建服务器

**功能描述：**
任何注册用户都可以创建服务器，服务器创建者自动成为服务器拥有者（Owner）。

**业务规则：**
- 服务器名称：2-100字符
- 服务器图标：支持上传，最大2MB
- 服务器描述：最多500字符
- 区域选择：用于选择最优的语音服务器节点
- 每个用户最多创建10个服务器

**数据表：server**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| name | VARCHAR(100) | 是 | 服务器名称 |
| icon | VARCHAR(500) | 否 | 服务器图标URL |
| description | VARCHAR(500) | 否 | 服务器描述 |
| region | VARCHAR(50) | 是 | 服务器区域（用于语音节点选择） |
| owner_id | BIGINT | 是 | 创建者/拥有者ID |
| max_members | INT | 是 | 最大成员数（默认500） |
| invite_code | VARCHAR(20) | 否 | 默认邀请码 |
| status | TINYINT | 是 | 状态：0-禁用 1-正常 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

### 2.2 服务器设置

**功能描述：**
服务器拥有者和管理员可修改服务器的基本设置。

**可设置项：**
- 服务器名称
- 服务器图标
- 服务器描述
- 区域（影响语音服务器节点选择）
- 验证等级（新成员需要验证的等级）
- 默认通知设置（全体静音/仅@提及）

**数据表：server_setting**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| verification_level | TINYINT | 是 | 验证等级：0-无 1-低 2-中 3-高 |
| default_notification | TINYINT | 是 | 默认通知：0-所有 1-仅@提及 |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

### 2.3 邀请机制

**功能描述：**
服务器管理员可生成邀请链接或邀请码，新用户通过邀请加入服务器。

**邀请方式：**
- **邀请链接** - 可设置有效期和使用次数
- **邀请码** - 固定短码，可随时更换

**业务规则：**
- 邀请链接默认7天有效
- 可设置永久有效
- 可设置最大使用次数
- 邀请码由服务器管理员手动更换

**数据表：server_invite**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| code | VARCHAR(20) | 是 | 邀请码/链接标识 |
| inviter_id | BIGINT | 是 | 创建者ID |
| max_uses | INT | 否 | 最大使用次数（null=无限） |
| uses | INT | 是 | 已使用次数 |
| expires_at | DATETIME | 否 | 过期时间（null=永不过期） |
| created_at | DATETIME | 是 | 创建时间 |

### 2.4 成员管理

**功能描述：**
管理服务器内的成员，包括加入、退出、踢出、封禁等操作。

**功能点：**
- 通过邀请链接/码加入服务器
- 主动退出服务器
- 管理员踢出成员（可选择是否封禁）
- 成员列表查看（按角色分组）
- 成员在线状态显示

**数据表：server_member**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| user_id | BIGINT | 是 | 用户ID |
| nickname | VARCHAR(50) | 否 | 服务器内昵称（覆盖全局昵称） |
| joined_at | DATETIME | 是 | 加入时间 |

### 2.5 角色与权限管理

**功能描述：**
服务器管理员可创建角色并分配权限，角色有层级关系，权限覆盖到频道级别。

**默认角色：**
- **Owner（拥有者）** - 拥有所有权限，不可被修改
- **Admin（管理员）** - 拥有大部分管理权限
- **Member（成员）** - 基本使用权限

**权限项：**
| 权限 | 说明 |
|------|------|
| MANAGE_SERVER | 管理服务器设置 |
| MANAGE_CHANNELS | 创建/编辑/删除频道 |
| MANAGE_ROLES | 创建/编辑/删除角色 |
| MANAGE_MEMBERS | 踢出/封禁成员 |
| SEND_MESSAGES | 发送文字消息 |
| CONNECT_VOICE | 连接语音频道 |
| SPEAK | 在语音频道中说话 |
| MUTE_MEMBERS | 静音其他成员 |
| DEAFEN_MEMBERS | 屏蔽其他成员音频 |
| MANAGE_MESSAGES | 删除/编辑他人消息 |
| VIEW_CHANNEL | 查看频道 |

**数据表：server_role**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| name | VARCHAR(50) | 是 | 角色名称 |
| color | VARCHAR(7) | 否 | 角色颜色（十六进制） |
| position | INT | 是 | 排序位置（越高越优先） |
| permissions | BIGINT | 是 | 权限位集合 |
| is_default | TINYINT | 是 | 是否默认角色 |
| created_at | DATETIME | 是 | 创建时间 |

## 接口设计（概要）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/server/create | POST | 创建服务器 |
| /api/server/list | GET | 获取我的服务器列表 |
| /api/server/{id} | GET | 获取服务器详情 |
| /api/server/{id} | PUT | 更新服务器设置 |
| /api/server/{id} | DELETE | 删除服务器 |
| /api/server/{id}/members | GET | 获取成员列表 |
| /api/server/{id}/members/{userId}/kick | POST | 踢出成员 |
| /api/server/{id}/members/{userId}/ban | POST | 封禁成员 |
| /api/server/{id}/invite/create | POST | 创建邀请链接 |
| /api/server/{id}/invite/list | GET | 获取邀请列表 |
| /api/server/{id}/invite/{code}/delete | DELETE | 删除邀请链接 |
| /api/server/join/{code} | POST | 通过邀请加入服务器 |
| /api/server/{id}/leave | POST | 退出服务器 |
| /api/server/{id}/roles | GET | 获取角色列表 |
| /api/server/{id}/roles | POST | 创建角色 |
| /api/server/{id}/roles/{roleId} | PUT | 编辑角色 |
| /api/server/{id}/roles/{roleId} | DELETE | 删除角色 |
| /api/server/{id}/members/{userId}/roles | PUT | 分配角色给成员 |

## 异常处理

| 异常场景 | 处理方式 |
|----------|----------|
| 服务器名称重复 | 允许重复（不同用户） |
| 达到创建上限 | 提示"最多创建10个服务器" |
| 邀请码已过期 | 提示"邀请链接已过期" |
| 邀请码使用次数耗尽 | 提示"邀请链接已失效" |
| 服务器已达最大成员数 | 提示"服务器已满" |
| 无权限执行操作 | 提示"权限不足" |
| 尝试踢出拥有者 | 不允许操作 |

## 验收标准

- [ ] 用户可创建服务器并设置基本信息
- [ ] 服务器拥有者可修改服务器设置
- [ ] 可生成邀请链接和邀请码
- [ ] 用户可通过邀请链接/码加入服务器
- [ ] 成员可主动退出服务器
- [ ] 管理员可踢出/封禁成员
- [ ] 可创建自定义角色并分配权限
- [ ] 角色有层级关系，高层级可管理低层级
