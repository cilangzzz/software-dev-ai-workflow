# 管理审核模块 (MOD)

## 功能概述

管理审核模块提供服务器内的成员管理和内容审核能力，包括踢出/封禁成员、消息审核、举报机制、敏感词过滤等功能。

## 功能清单

| 序号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 1 | 踢出成员 | P0 | 将成员移出服务器 |
| 2 | 封禁成员 | P0 | 禁止成员加入服务器 |
| 3 | 消息审核与删除 | P0 | 管理员删除违规消息 |
| 4 | 举报机制 | P1 | 成员举报违规内容/用户 |
| 5 | 管理员操作日志 | P1 | 记录所有管理操作 |
| 6 | 内容过滤 | P1 | 敏感词/敏感图过滤 |

## 详细需求

### 7.1 踢出成员

**功能描述：**
管理员将成员移出服务器，可选择是否允许重新加入。

**业务规则：**
- 需要 MANAGE_MEMBERS 权限
- 不能踢出拥有者（Owner）
- 不能踢出角色层级高于自己的成员
- 可选择：踢出 + 禁止重新加入（7天/永久）

**接口：**
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/server/{id}/members/{userId}/kick | POST | 踢出成员 |
| 参数 | reason | 踢出原因 |
| 参数 | ban_duration | 封禁时长（0=不封禁，-1=永久） |

### 7.2 封禁成员

**功能描述：**
管理员封禁用户，禁止其加入服务器。

**业务规则：**
- 需要 MANAGE_MEMBERS 权限
- 封禁后自动踢出服务器
- 被封禁用户无法通过任何邀请加入
- 管理员可解除封禁
- 封禁记录持久化

**封禁状态：**
| 状态 | 说明 |
|------|------|
| 临时封禁 | 指定天数后自动解除 |
| 永久封禁 | 需手动解除 |
| 已解除 | 管理员手动解除 |

**数据表（扩展server_ban）：**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| user_id | BIGINT | 是 | 被封禁用户ID |
| moderator_id | BIGINT | 是 | 操作管理员ID |
| reason | VARCHAR(500) | 否 | 封禁原因 |
| expires_at | DATETIME | 否 | 解封时间（null=永久） |
| is_active | TINYINT | 是 | 是否生效中 |
| created_at | DATETIME | 是 | 创建时间 |

### 7.3 消息审核与删除

**功能描述：**
管理员可删除违规消息，清理频道内容。

**业务规则：**
- 需要 MANAGE_MESSAGES 权限
- 可删除任意成员的消息
- 删除后其他成员可见"消息已被管理员删除"
- 支持批量删除（清理刷屏）
- 删除操作记录到审核日志

### 7.4 举报机制

**功能描述：**
成员可举报违规消息或用户，管理员在审核面板处理。

**举报类型：**
| 类型 | 说明 |
|------|------|
| SPAM | 垃圾信息/刷屏 |
| HARASSMENT | 骚扰/辱骂 |
| INAPPROPRIATE | 不当内容 |
| IMPERSONATION | 冒充他人 |
| OTHER | 其他 |

**举报流程：**
```
用户点击举报
    → 选择举报类型
    → 填写描述（可选）
    → 提交举报
    → 管理员审核面板收到
    → 管理员处理（删除/警告/封禁/忽略）
    → 举报人收到处理结果通知
```

**数据表：report**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| reporter_id | BIGINT | 是 | 举报人ID |
| target_type | TINYINT | 是 | 举报对象：1-用户 2-消息 |
| target_id | BIGINT | 是 | 目标ID |
| reason_type | VARCHAR(50) | 是 | 举报类型 |
| description | VARCHAR(1000) | 否 | 举报描述 |
| status | TINYINT | 是 | 状态：0-待处理 1-已处理 2-已忽略 |
| handler_id | BIGINT | 否 | 处理人ID |
| handle_result | VARCHAR(500) | 否 | 处理结果 |
| created_at | DATETIME | 是 | 创建时间 |
| handled_at | DATETIME | 否 | 处理时间 |

### 7.5 管理员操作日志

**功能描述：**
记录所有管理员操作，用于追溯和审计。

**记录的操作类型：**
- 踢出成员
- 封禁成员/解除封禁
- 删除消息
- 修改角色权限
- 创建/删除频道
- 修改服务器设置
- 处理举报

**数据表：moderation_log**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| server_id | BIGINT | 是 | 服务器ID |
| moderator_id | BIGINT | 是 | 操作管理员ID |
| action | VARCHAR(50) | 是 | 操作类型 |
| target_type | VARCHAR(50) | 否 | 目标类型 |
| target_id | BIGINT | 否 | 目标ID |
| detail | TEXT | 否 | 操作详情（JSON） |
| ip_address | VARCHAR(50) | 否 | IP地址 |
| created_at | DATETIME | 是 | 创建时间 |

### 7.6 内容过滤

**功能描述：**
自动检测和过滤违规内容。

**过滤方式：**
- **敏感词过滤** - 基于关键词匹配
- **正则过滤** - 基于正则表达式
- **自动操作** - 删除 / 替换为* / 警告

**业务规则：**
- 支持自定义敏感词库
- 支持按服务器配置
- 匹配到敏感词时可自动删除或标记待审核

## 接口设计（概要）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/server/{id}/members/{userId}/kick | POST | 踢出成员 |
| /api/server/{id}/members/{userId}/ban | POST | 封禁成员 |
| /api/server/{id}/bans | GET | 获取封禁列表 |
| /api/server/{id}/bans/{userId}/revoke | POST | 解除封禁 |
| /api/server/{id}/messages/{msgId}/delete | DELETE | 管理员删除消息 |
| /api/server/{id}/messages/bulk-delete | POST | 批量删除消息 |
| /api/report | POST | 提交举报 |
| /api/server/{id}/reports | GET | 获取举报列表 |
| /api/report/{id}/handle | POST | 处理举报 |
| /api/server/{id}/moderation-logs | GET | 获取操作日志 |
| /api/server/{id}/word-filter | GET | 获取敏感词列表 |
| /api/server/{id}/word-filter | PUT | 更新敏感词列表 |

## 验收标准

- [ ] 管理员可踢出普通成员
- [ ] 被踢出成员可重新通过邀请加入（未封禁时）
- [ ] 封禁后成员无法重新加入
- [ ] 管理员可解除封禁
- [ ] 管理员可删除任意成员的消息
- [ ] 成员可提交举报
- [ ] 管理员在审核面板查看和处理举报
- [ ] 所有管理操作记录到日志
- [ ] 敏感词过滤自动拦截违规内容
