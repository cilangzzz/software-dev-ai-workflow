# 文字消息模块 (MSG)

## 功能概述

文字消息模块提供频道内的实时文字交流能力，支持文本、图片、文件等多种消息类型，以及消息历史搜索、@提及、引用回复等交互功能。

## 功能清单

| 序号 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 1 | 消息发送/接收 | P0 | 实时文字消息收发 |
| 2 | 消息历史与搜索 | P0 | 历史消息加载、关键词搜索 |
| 3 | @提及 | P0 | @成员/角色/@全体 |
| 4 | 引用回复 | P1 | 引用某条消息进行回复 |
| 5 | 图片/文件发送 | P0 | 附件消息 |
| 6 | 消息编辑/删除 | P1 | 修改和撤回自己的消息 |
| 7 | Emoji表情反应 | P1 | 对消息添加表情反应 |

## 详细需求

### 5.1 消息发送/接收

**功能描述：**
用户在文字频道中发送和接收消息，消息实时推送。

**消息类型：**
| 类型 | 说明 |
|------|------|
| TEXT | 纯文本消息 |
| IMAGE | 图片消息 |
| FILE | 文件消息 |
| SYSTEM | 系统消息（自动） |
| VOICE | 语音通话通知（自动） |

**业务规则：**
- 消息内容最大长度：2000字符
- 发送频率限制：慢速模式下按频道设置间隔
- 消息实时通过WebSocket推送
- 消息持久化存储

**数据表：message**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| channel_id | BIGINT | 是 | 频道ID |
| server_id | BIGINT | 是 | 服务器ID |
| author_id | BIGINT | 是 | 发送者ID |
| content | TEXT | 否 | 消息内容 |
| type | TINYINT | 是 | 消息类型 |
| reply_to_id | BIGINT | 否 | 引用的消息ID |
| is_pinned | TINYINT | 是否置顶 |
| is_edited | TINYINT | 是 | 是否已编辑 |
| is_deleted | TINYINT | 是 | 是否已删除（软删除） |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

### 5.2 消息历史与搜索

**功能描述：**
用户可查看历史消息，支持关键词搜索。

**功能点：**
- 频道内消息按时间倒序加载
- 向上滚动加载更早消息（分页，每次50条）
- 消息搜索（按关键词、发送者、时间范围）
- 跳转到特定日期的消息

**接口：**
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/channel/{chId}/messages | GET | 获取频道消息列表（分页） |
| /api/channel/{chId}/messages/search | GET | 搜索消息 |
| /api/channel/{chId}/messages/{msgId} | GET | 获取特定消息 |

### 5.3 @提及

**功能描述：**
用户可在消息中@其他成员、角色或全体成员。

**提及类型：**
- `@用户名` - 提及单个成员
- `@角色名` - 提及某个角色的所有成员
- `@全体成员` - 提及频道内所有人（需权限）

**业务规则：**
- 被@的用户会收到通知
- `@全体成员`需要 MANAGE_MESSAGES 权限
- 提及信息在消息内容中用特殊标记表示（如 `<@userId>`）

### 5.4 引用回复

**功能描述：**
用户可引用一条已有消息进行回复，形成消息引用链。

**业务规则：**
- 只能引用当前频道内的消息
- 引用回复显示原消息摘要（发送者、内容前100字符）
- 点击引用可跳转到原消息

### 5.5 图片/文件发送

**功能描述：**
用户可在消息中附带图片或文件。

**限制：**
- 图片最大10MB，支持 jpg/png/gif/webp
- 文件最大50MB
- 图片在聊天中预览
- 文件显示文件名、大小、类型

**数据表：message_attachment**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| message_id | BIGINT | 是 | 消息ID |
| file_name | VARCHAR(255) | 是 | 文件名 |
| file_url | VARCHAR(500) | 是 | 文件存储URL |
| file_size | BIGINT | 是 | 文件大小（字节） |
| file_type | VARCHAR(50) | 是 | MIME类型 |
| width | INT | 否 | 图片宽度（仅图片） |
| height | INT | 否 | 图片高度（仅图片） |
| created_at | DATETIME | 是 | 创建时间 |

### 5.6 消息编辑/删除

**功能描述：**
用户可编辑或删除自己发送的消息。

**业务规则：**
- 只能编辑/删除自己发送的消息
- 编辑后显示"已编辑"标识
- 删除采用软删除（标记is_deleted=1）
- 管理员可删除任意成员的消息（需MANAGE_MESSAGES权限）
- 消息发送后5分钟内可编辑

### 5.7 Emoji表情反应

**功能描述：**
用户可对任意消息添加Emoji表情反应。

**业务规则：**
- 每条消息每个用户最多添加20个反应
- 相同反应可被多人添加，显示数量
- 点击反应切换（已添加则取消）

**数据表：message_reaction**

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | BIGINT | 是 | 主键 |
| message_id | BIGINT | 是 | 消息ID |
| user_id | BIGINT | 是 | 用户ID |
| emoji | VARCHAR(50) | 是 | Emoji标识 |
| created_at | DATETIME | 是 | 创建时间 |

## WebSocket事件

| 事件 | 方向 | 说明 |
|------|------|------|
| message:create | Server → Client | 新消息 |
| message:update | Server → Client | 消息编辑 |
| message:delete | Server → Client | 消息删除 |
| message:reaction:add | Server → Client | 添加反应 |
| message:reaction:remove | Server → Client | 移除反应 |
| typing:start | Client → Server | 开始输入 |
| typing:stop | Client → Server | 停止输入 |
| typing:indicator | Server → Client | 输入中指示 |
