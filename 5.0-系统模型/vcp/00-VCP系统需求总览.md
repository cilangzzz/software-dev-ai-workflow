# VCP语音社区系统 - 需求总览

## 项目背景

开发一套语音社区系统（Voice Chat Platform, VCP），参考YY语音、Discord等产品，支持用户创建社区、频道分类、多人语音房间等核心功能。

| 属性 | 值 |
|------|-----|
| **产品定位** | 语音社区平台（类Discord/YY语音） |
| **目标平台** | 桌面端优先（Windows/Mac），Web和移动端后续排期 |
| **语音规模** | 单房间20人，后续优化扩大 |
| **国际化** | 支持多语言 |
| **扩展性** | 支持Bot扩展、第三方集成（后续排期） |

## 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                    VCP语音社区系统                           │
├──────────┬──────────┬──────────┬──────────┬────────────────┤
│  用户模块  │ 服务器模块 │  频道模块  │ 语音房间  │  文字消息模块   │
├──────────┴──────────┴──────────┴──────────┴────────────────┤
│  通知模块  │  管理审核模块  │  系统设置模块                   │
├─────────────────────────────────────────────────────────────┤
│  基础服务层（RTC引擎、信令服务、存储服务、认证服务）           │
├─────────────────────────────────────────────────────────────┤
│  数据层（MySQL/PostgreSQL + Redis + 对象存储）                │
└─────────────────────────────────────────────────────────────┘
```

## 核心业务流程

```
用户注册登录 → 创建/加入服务器 → 浏览频道列表 → 进入语音房间 → 多人语音聊天
                                      ↓
                               文字频道实时交流
                                      ↓
                               通知与消息推送
```

## 实施路线图

| 阶段 | 周期 | 内容 | 交付物 |
|------|------|------|--------|
| **Phase 1** | 4周 | 基础搭建 | 用户模块、系统设置模块 |
| **Phase 2** | 6周 | 社区核心 | 服务器模块、频道模块 |
| **Phase 3** | 6周 | 语音核心 | 语音房间模块（WebRTC集成） |
| **Phase 4** | 4周 | 消息系统 | 文字消息模块、通知模块 |
| **Phase 5** | 4周 | 管理与优化 | 管理审核模块、性能优化 |
| **Phase 6** | 后续 | 扩展功能 | 屏幕共享、视频、Bot、第三方集成 |

## 技术选型方向

| 模块 | 技术方案 |
|------|----------|
| **桌面客户端** | Electron 或 Tauri |
| **音频传输** | WebRTC（SFU架构） |
| **信令服务** | WebSocket |
| **后端服务** | 待定（Java/Go/Node.js） |
| **数据库** | MySQL/PostgreSQL |
| **缓存** | Redis |
| **对象存储** | MinIO / 云OSS |
