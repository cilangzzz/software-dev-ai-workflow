# 移动终端模块需求文档

## 模块概述

| 属性 | 值 |
|------|-----|
| 模块名称 | 移动终端 |
| 英文名称 | Mobile Terminal |
| 优先级 | P0 |
| 所属系统 | 汽车整车装配MES系统 |
| 技术栈 | UniApp（微信小程序/H5/APP） |

## 功能清单

| 功能项 | 功能描述 | 优先级 | 状态 |
|--------|----------|--------|------|
| 登录认证 | 用户登录和权限验证 | P0 | 待开发 |
| 扫码作业 | 扫描VIN/物料码进行作业 | P0 | 待开发 |
| 作业指引 | 显示当前工序作业指导 | P0 | 待开发 |
| 关键件扫描 | 扫描绑定关键零部件 | P0 | 待开发 |
| 作业记录 | 查看个人作业记录 | P0 | 待开发 |
| 消息通知 | 接收异常/任务通知 | P1 | 待开发 |
| 离线作业 | 断网时缓存数据后续同步 | P2 | 待开发 |

## 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| MT-001 | 工位权限 | 作业员只能操作分配给自己的工位 |
| MT-002 | 扫码范围 | 扫码需在规定工位范围内有效 |
| MT-003 | 关键件必扫 | 关键工序必须扫描所有关键件才能完成报工 |
| MT-004 | 异常原因 | 异常上报需填写异常原因 |

## 接口设计

| 接口名称 | 请求方式 | 接口路径 | 说明 |
|----------|----------|----------|------|
| 用户登录 | POST | /app/auth/login | 用户登录认证 |
| 扫码解析 | POST | /app/operation/scan | 扫码解析并返回作业信息 |
| 开始作业 | POST | /app/operation/start | 开始工序作业 |
| 完成作业 | PUT | /app/operation/complete | 完成工序作业 |
| 绑定关键件 | POST | /app/operation/bind-part | 扫描绑定关键零部件 |
| 工单列表 | GET | /app/work-order/list | 查询工单列表 |
| 作业记录 | GET | /app/operation/records | 查询个人作业记录 |
| 异常上报 | POST | /app/exception/report | 上报作业异常 |

## 离线作业设计

### 数据缓存策略

| 数据类型 | 缓存策略 |
|----------|----------|
| 工单信息 | 登录时缓存当前工单 |
| 工艺路线 | 登录时缓存当前工艺路线 |
| 作业记录 | 本地存储，联网后上传 |
| 关键件绑定 | 本地存储，联网后上传 |

### 同步机制

1. 检测网络状态
2. 网络恢复后自动上传本地数据
3. 服务端校验数据一致性
4. 返回同步结果
