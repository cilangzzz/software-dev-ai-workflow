# 设备集成模块需求文档

## 模块概述

| 属性 | 值 |
|------|-----|
| 模块名称 | 设备集成 |
| 英文名称 | Device Integration |
| 优先级 | P1 |
| 所属系统 | 汽车整车装配MES系统 |

## 功能清单

| 功能项 | 功能描述 | 优先级 | 状态 |
|--------|----------|--------|------|
| 扭矩数据采集 | 接收扭矩枪/拧紧机数据 | P1 | 待开发 |
| PLC数据采集 | 采集PLC设备状态和IO信号 | P1 | 待开发 |
| 设备状态监控 | 监控设备运行状态 | P1 | 待开发 |
| 设备告警处理 | 处理设备异常告警 | P1 | 待开发 |

## 设备类型与协议

| 设备类型 | 连接协议 | 数据内容 | 集成方式 |
|----------|----------|----------|----------|
| 扭矩枪/拧紧机 | MQTT/串口/以太网 | 扭矩值、角度值、合格判定 | IoT网关转发 |
| PLC设备 | Modbus TCP/RTU | 设备状态、IO信号 | IoT Modbus适配器 |
| 条码扫描枪 | USB/串口/蓝牙 | VIN码、物料码 | 终端直接读取 |

## 扭矩数据集成

### 数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| deviceCode | String | 扭矩枪设备编号 |
| vin | String | 当前作业车辆VIN |
| torqueValue | Decimal | 实际扭矩值(N·m) |
| angleValue | Decimal | 拧紧角度(度) |
| result | String | 合格判定: OK/NG |
| timestamp | DateTime | 采集时间 |

### 业务处理流程

1. IoT网关接收扭矩数据
2. 根据VIN关联作业记录
3. 判断扭矩是否在标准范围内
4. 更新作业记录扭矩结果
5. 不合格时触发预警

## PLC集成

### 采集内容

| 采集项 | 说明 |
|--------|------|
| 设备运行状态 | 运行/停止/故障 |
| 关键IO信号 | 到位信号、传感器信号 |
| 计数器值 | 产量计数 |

### 业务场景

| 场景 | 触发条件 | 业务动作 |
|------|----------|----------|
| 工位到位 | IO信号触发 | 自动开始作业计时 |
| 设备故障 | 故障信号 | 触发停线报警 |
| 产量计数 | 计数器变化 | 更新产量统计 |

## 接口设计

### 扭矩数据接收接口

**接口：** POST /mes/device/torque

**请求示例：**
```json
{
  "deviceCode": "TQ-GUN-001",
  "vin": "LSVAU2180N2123456",
  "torqueValue": 122.5,
  "angleValue": 180,
  "result": "OK",
  "timestamp": "2026-03-24T08:43:00"
}
```

**响应示例：**
```json
{
  "code": 0,
  "message": "success"
}
```

### 设备状态上报接口

**接口：** POST /mes/device/status

**请求示例：**
```json
{
  "deviceCode": "PLC-001",
  "deviceName": "主线PLC",
  "status": 1,
  "statusText": "运行",
  "ioSignals": [
    {"signalCode": "IO001", "signalName": "工位到位", "value": 1}
  ],
  "timestamp": "2026-03-24T08:43:00"
}
```
