## 一、架构概述

### 1.1 系统定位

本系统是基于芋道框架(yudao-aisk-mes)SaaS平台开发的汽车整车装配MES系统，定位于生产执行层，连接ERP计划层和设备控制层。

```
┌─────────────────────────────────────────────────────────────┐
│                        ERP计划层                            │
│                    (生产计划、物料管理)                      │
└─────────────────────────────┬───────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      MES执行层                              │
│              (工单管理、作业执行、质量追溯)                  │
└─────────────────────────────┬───────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      设备控制层                             │
│                 (PLC、扭矩枪、扫描枪)                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| 模块化 | 功能模块独立，低耦合高内聚 |
| 可扩展 | 支持功能扩展和二次开发 |
| 高可用 | 关键服务冗余，故障自动恢复 |
| 多租户 | 支持多租户数据隔离 |
| 标准化 | 遵循芋道框架规范 |


## 十、技术难点与解决方案

| 技术难点 | 解决方案 |
|----------|----------|
| 多设备协议适配 | 复用IoT模块协议框架，插件化扩展 |
| 实时数据采集 | MQTT + 消息队列架构 |
| 高并发扫码 | Redis缓存工单状态，异步写入数据库 |
| 生产追溯性能 | 追溯数据索引优化，Elasticsearch支持 |
| ERP数据同步 | 定时任务 + 事件驱动双模式 |
| 移动端离线作业 | 本地缓存 + 后台同步机制 |
