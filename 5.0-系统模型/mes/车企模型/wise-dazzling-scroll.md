# 汽车整车装配MES系统 - 功能模块需求文档

## 项目背景

基于现有芋道框架(yudao-aisk-mes)SaaS平台，开发汽车整车装配MES系统。
- **技术栈**：Spring Boot 2.7 + MyBatis Plus + Redis + MySQL + Flowable
- **已有模块**：ERP、IoT、工作流、系统管理
- **运行模式**：MES独立运行，后续对接ERP
- **移动终端**：UniApp多端支持
- **设备集成**：PLC、扭矩枪/拧紧机、条码扫描枪

---

## 一、生产工单管理模块

### 1.1 模块概述

生产工单是MES系统的核心驱动单元，用于管理整车装配的生产任务。工单从创建到完成的全生命周期管理。

### 1.2 功能清单

| 功能项 | 功能描述 | 优先级 |
|--------|----------|--------|
| 工单创建 | 手动创建或导入生产工单 | P0 |
| 工单编辑 | 修改未下发的工单信息 | P0 |
| 工单下发 | 将工单下发到产线开始执行 | P0 |
| 工单进度查看 | 查看工单执行进度和详情 | P0 |
| 工单关闭 | 关闭未完成的工单 | P0 |
| 工单查询 | 多条件查询工单列表 | P0 |
| 工单统计 | 按状态、产线统计工单数量 | P1 |

### 1.3 工单状态流转

```
待下发 → 已下发 → 生产中 → 已完成
    ↓         ↓        ↓
  已关闭    已关闭    已关闭
```

| 状态 | 说明 | 允许操作 |
|------|------|----------|
| 待下发 | 工单已创建，等待下发 | 编辑、下发、关闭 |
| 已下发 | 工单已下发到产线，等待开始生产 | 开始生产、关闭 |
| 生产中 | 工单正在生产执行中 | 暂停、完成、关闭 |
| 已完成 | 工单已完成生产 | 查看、归档 |
| 已关闭 | 工单被强制关闭 | 查看 |

### 1.4 工单数据项

| 字段名 | 说明 | 类型 | 必填 |
|--------|------|------|------|
| 工单编号 | 系统自动生成 | string | 是 |
| 产品编码 | 关联产品信息 | string | 是 |
| 产品名称 | 产品名称 | string | 是 |
| 计划数量 | 计划生产数量 | int | 是 |
| 已产数量 | 已完成数量 | int | 自动 |
| 工艺路线 | 关联工艺路线 | reference | 是 |
| 计划开始时间 | 计划开始生产时间 | datetime | 是 |
| 计划结束时间 | 计划完成时间 | datetime | 是 |
| 产线 | 生产产线 | reference | 是 |
| 优先级 | 工单优先级(1-10) | int | 否 |
| 状态 | 工单状态 | enum | 自动 |
| 备注 | 备注信息 | string | 否 |

### 1.5 业务规则

1. 工单编号规则：WO + 年月日 + 4位流水号，如 WO202603240001
2. 同一产线同一时间只能有一个"生产中"状态的工单
3. 工单下发后不可修改产品信息和数量
4. 工单完成后自动归档，不可修改

## 二、工艺路线管理模块

| 工艺路线编辑 | 修改工艺路线及工序 | P0 |
| 工艺路线版本管理 | 管理工艺路线的多个版本 | P1 |
| 工艺路线复制 | 复制已有工艺路线 | P0 |
| 工艺路线生效/失效 | 控制工艺路线的使用状态 | P0 |
| 工序配置 | 配置工序详细信息 | P0 |

### 2.3 工艺路线数据项

| 字段名 | 说明 | 类型 | 必填 |
|--------|------|------|------|
| 路线编码 | 工艺路线唯一编码 | string | 是 |
| 路线名称 | 工艺路线名称 | string | 是 |
| 产品编码 | 关联产品 | string | 是 |
| 版本号 | 版本号(如V1.0) | string | 是 |
| 状态 | 草稿/生效/失效 | enum | 自动 |
| 描述 | 路线描述 | string | 否 |

### 2.4 工序配置数据项

| 字段名 | 说明 | 类型 | 必填 |
|--------|------|------|------|
| 工序编码 | 工序唯一编码 | string | 是 |
| 工序名称 | 工序名称 | string | 是 |
| 工序顺序 | 在路线中的顺序 | int | 是 |
| 工作站 | 默认工作站 | reference | 是 |
| 标准工时 | 标准作业时间(分钟) | decimal | 否 |
| 是否关键工序 | 关键工序标识 | boolean | 否 |
| 是否需要质检 | 质检标识 | boolean | 否 |
| 作业指导书 | 作业指导文件 | file | 否 |

### 2.5 业务规则

1. 工艺路线生效后不可修改，需要创建新版本
2. 工序顺序必须连续，不能有断号
3. 同一产品只能有一个生效的工艺路线版本
4. 被工单引用的工艺路线不可失效

## 四、装配作业执行模块

### 4.1 模块概述

装配作业执行是MES的核心操作模块，管理现场作业人员的扫码作业、工序报工、关键件绑定等操作。

### 4.2 功能清单

| 功能项 | 功能描述 | 优先级 |
|--------|----------|--------|
| 扫码识别 | 识别VIN码/工单码/物料码 | P0 |
| 作业指引 | 显示当前工序作业指导 | P0 |
| 工序报工 | 完成工序作业并报工 | P0 |
| 关键件绑定 | 扫描绑定关键零部件 | P0 |
| 作业记录查询 | 查询作业历史记录 | P0 |
| 异常上报 | 上报作业异常 | P1 |

### 4.3 扫码作业流程

```
扫描VIN码 → 系统识别车辆 → 显示当前工序 → 执行作业 → 扫描关键件 → 完成报工
     ↓
  新VIN     → 创建新作业记录
  已有VIN   → 显示作业进度，继续未完成工序
```

### 4.4 作业记录数据项

| 字段名 | 说明 | 类型 | 必填 |
|--------|------|------|------|
| VIN码 | 车辆识别码 | string | 是 |
| 工单编号 | 关联工单 | reference | 是 |
| 工序编码 | 当前工序 | reference | 是 |
| 工作站 | 作业工作站 | reference | 是 |
| 操作员 | 作业人员 | reference | 是 |
| 开始时间 | 作业开始时间 | datetime | 自动 |
| 结束时间 | 作业结束时间 | datetime | 自动 |
| 作业时长 | 作业耗时(秒) | int | 自动 |
| 作业结果 | 合格/不合格 | enum | 是 |
| 备注 | 作业备注 | string | 否 |

### 4.5 关键件绑定数据项

| 字段名 | 说明 | 类型 | 必填 |
|--------|------|------|------|
| VIN码 | 车辆识别码 | string | 是 |
| 零部件编码 | 关键件编码 | string | 是 |
| 零部件名称 | 关键件名称 | string | 是 |
| 零部件序列号 | 关键件唯一SN | string | 是 |
| 供应商编码 | 供应商编码 | string | 否 |
| 绑定时间 | 绑定时间 | datetime | 自动 |
| 操作员 | 绑定人员 | reference | 是 |

### 4.6 业务规则

1. VIN码为17位，需校验格式正确性
2. 同一VIN同一工序只能有一条作业记录
3. 关键件序列号全局唯一，不可重复绑定
4. 前置工序未完成不可执行后续工序
5. 关键工序必须绑定所有指定的关键件

## 七、生产追溯模块

### 7.1 模块概述

提供车辆全生命周期的生产追溯查询，支持VIN追溯和关键件追溯。

### 7.2 功能清单

| 功能项 | 功能描述 | 优先级 |
|--------|----------|--------|
| VIN追溯查询 | 按VIN查询车辆生产信息 | P0 |
| 关键件追溯 | 按关键件SN反向追溯车辆 | P0 |
| 工序追溯 | 查询各工序作业详情 | P0 |
| 操作员追溯 | 查询操作员作业记录 | P1 |
| 设备数据追溯 | 查询设备采集数据 | P1 |

### 7.3 追溯查询内容

**VIN正向追溯：**
- 车辆基本信息（车型、颜色、配置）
- 工单信息
- 各工序作业记录（时间、操作员、工位）
- 关键件绑定记录
- 质量检验记录
- 设备采集数据（扭矩值等）

**关键件反向追溯：**
- 关键件基本信息
- 绑定车辆VIN
- 绑定时间、工位、操作员

### 7.4 业务规则

1. 追溯数据永久保存，不可删除
2. 追溯查询需有相应权限
3. 敏感数据脱敏显示

## 九、设备集成模块

### 9.1 模块概述

管理MES与现场设备的数据交互，实现设备数据采集和控制指令下发。

### 9.2 设备类型与协议

| 设备类型 | 连接协议 | 数据内容 | 集成方式 |
|----------|----------|----------|----------|
| 扭矩枪/拧紧机 | MQTT/串口 | 扭矩值、角度值、合格判定 | IoT网关转发 |
| PLC设备 | Modbus TCP/RTU | 设备状态、IO信号 | IoT Modbus适配器 |
| 条码扫描枪 | USB/串口/蓝牙 | VIN码、物料码 | 终端直接读取 |

### 9.3 设备数据采集流程

```
设备产生数据 → IoT网关接收 → 协议解析 → 消息队列 → MES处理 → 数据存储
                                                              ↓
                                                        业务关联(VIN/工单)
```

### 9.4 扭矩数据集成说明

**数据字段：**
| 字段 | 说明 |
|------|------|
| 设备编码 | 扭矩枪设备编号 |
| VIN码 | 当前作业车辆VIN |
| 扭矩值 | 实际扭矩值(N·m) |
| 角度值 | 拧紧角度(度) |
| 合格判定 | OK/NG |
| 时间戳 | 采集时间 |

**业务处理：**
1. 接收扭矩数据
2. 根据VIN关联作业记录
3. 判断扭矩是否在标准范围内
4. 更新作业记录结果
5. 不合格时触发预警

### 9.5 PLC集成说明

**采集内容：**
- 设备运行状态（运行/停止/故障）
- 关键IO信号（到位信号、传感器信号）
- 计数器值（产量计数）

**业务场景：**
- 工位到位信号触发作业开始
- 设备故障触发停线报警
- 自动计数更新产量

### 10.4 业务规则

1. 作业员只能操作分配给自己的工位
2. 扫码需在规定工位范围内有效
3. 关键工序必须扫描所有关键件才能完成报工
4. 异常上报需填写异常原因
