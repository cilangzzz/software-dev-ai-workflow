## 一、系统概述

### 1.1 系统定位

汽车整车装配MES系统是基于芋道框架(yudao-aisk-mes)SaaS平台开发的制造执行系统，定位于生产执行层，连接ERP计划层和设备控制层。

## 三、服务管理

### 3.1 应用服务管理

#### 启动服务
```bash
# 方式一：jar包启动
cd /opt/mes/app
java -jar yudao-server.jar --spring.profiles.active=prod

# 方式二：使用启动脚本
./bin/startup.sh

# 方式三：使用systemd服务
systemctl start mes-server
```

#### 停止服务
```bash
# 方式一：使用shutdown端点
curl -X POST http://localhost:8080/actuator/shutdown

# 方式二：使用脚本
./bin/shutdown.sh

# 方式三：使用systemd服务
systemctl stop mes-server
```

#### 重启服务
```bash
./bin/restart.sh
# 或
systemctl restart mes-server
```

#### 查看服务状态
```bash
# 查看进程
ps -ef | grep yudao-server

# 查看端口
netstat -tlnp | grep 8080

# 查看日志
tail -f logs/mes-server.log

# 健康检查
curl http://localhost:8080/actuator/health
```

| 配置文件 | 路径 | 说明 |

| Redis配置 | config/application-prod.yml | 缓存配置 |

### 4.2 关键配置参数

#### 数据库连接池配置
```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://localhost:3306/mes_prod?useSSL=true
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    druid:
      initial-size: 5
      min-idle: 5
      max-active: 20
      max-wait: 60000
```

#### Redis配置
```yaml
spring:
  redis:
    host: ${REDIS_HOST}
    port: 6379
    password: ${REDIS_PASSWORD}
    database: 0
    lettuce:
      pool:
        max-active: 10
        max-idle: 5
        min-idle: 2
```

#### RabbitMQ配置
```yaml
spring:
  rabbitmq:
    host: ${RABBITMQ_HOST}
    port: 5672
    username: ${RABBITMQ_USERNAME}
    password: ${RABBITMQ_PASSWORD}
    virtual-host: /mes
```

### 4.3 Nginx配置

```nginx
upstream mes_backend {
    server 192.168.1.101:8080 weight=1;
    server 192.168.1.102:8080 weight=1;
    keepalive 32;
}

server {
    listen 80;
    server_name mes.example.com;

    # HTTPS重定向
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name mes.example.com;

    ssl_certificate /etc/nginx/ssl/mes.crt;
    ssl_certificate_key /etc/nginx/ssl/mes.key;

    location / {
        proxy_pass http://mes_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_connect_timeout 60s;
        proxy_read_timeout 60s;
    }
}
```

## 六、备份与恢复

### 6.1 数据库备份

#### 自动备份脚本
```bash
#!/bin/bash
# 文件：/opt/scripts/backup_mysql.sh

BACKUP_DIR="/data/backup/mysql"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="mes_prod"
DB_USER="backup"
DB_PASS="******"

mkdir -p $BACKUP_DIR

mysqldump -u$DB_USER -p$DB_PASS --single-transaction --routines --triggers $DB_NAME | gzip > $BACKUP_DIR/${DB_NAME}_${DATE}.sql.gz

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: ${DB_NAME}_${DATE}.sql.gz"
```

#### 配置定时备份
```bash
# 添加到crontab
# 每天凌晨2点执行备份
0 2 * * * /opt/scripts/backup_mysql.sh >> /var/log/mysql_backup.log 2>&1
```

### 6.2 数据库恢复

```bash
# 解压备份文件
gunzip mes_prod_20260327.sql.gz

# 恢复数据库
mysql -u root -p mes_prod < mes_prod_20260327.sql
```

### 6.3 Redis备份

```bash
# 手动触发RDB快照
redis-cli BGSAVE

# 复制RDB文件
cp /var/lib/redis/dump.rdb /data/backup/redis/dump_$(date +%Y%m%d).rdb
```

### 6.4 配置文件备份

```bash
# 备份应用配置
tar -czvf config_backup_$(date +%Y%m%d).tar.gz /opt/mes/config/

# 备份Nginx配置
cp /etc/nginx/nginx.conf /data/backup/nginx/nginx_$(date +%Y%m%d).conf
```

## 七、回滚操作

### 7.1 应用回滚

```bash
# 1. 停止当前服务
./bin/shutdown.sh

# 2. 备份当前版本
cp yudao-server.jar yudao-server.jar.bak

# 3. 恢复历史版本
cp /data/releases/yudao-server-v1.0.0.jar yudao-server.jar

# 4. 启动服务
./bin/startup.sh

# 5. 验证服务
curl http://localhost:8080/actuator/health
```

### 7.2 数据库回滚

```bash
# 1. 停止应用服务，防止数据写入
./bin/shutdown.sh

# 2. 恢复数据库
mysql -u root -p mes_prod < /data/backup/mysql/mes_prod_20260327.sql

# 3. 启动服务
./bin/startup.sh
```

### 7.3 配置回滚

```bash
# 恢复配置文件
tar -xzvf /data/backup/config_backup_20260327.tar.gz -C /opt/mes/config/

# 重启服务使配置生效
./bin/restart.sh
```
