## 一、项目概述

### 1.1 项目背景

本系统是基于现有芋道框架(yudao-aisk-mes)SaaS平台开发的汽车整车装配MES（制造执行系统）。系统定位于生产车间的执行层，管理从工单下发到成品入库的生产全过程。

### 1.2 项目目标

| 目标类型 | 具体目标 |
|----------|----------|
| **生产透明化** | 实时掌握生产进度、产量、质量等关键指标 |
| **质量可追溯** | 实现车辆VIN追溯和关键零部件追溯 |
| **作业标准化** | 通过工艺路线和作业指导规范化作业流程 |
| **数据实时化** | 设备数据实时采集，异常实时预警 |

### 1.3 项目范围

| 范围 | 说明 |
|------|------|
| **业务范围** | 整车装配车间（总装环节） |
| **功能范围** | 工单管理、工艺路线、作业执行、物料管理、质量管理、生产追溯、看板 |
| **集成范围** | IoT设备集成（PLC、扭矩枪、扫描枪），后续对接ERP |

### 1.4 技术架构

| 属性 | 值 |
|------|-----|
| **技术栈** | Spring Boot 2.7 + MyBatis Plus + Redis + MySQL + Flowable |
| **架构模式** | 多租户SaaS架构 |
| **前端技术** | Vue3 + Element Plus / UniApp |
| **移动端** | UniApp多端（微信小程序/H5/APP） |

## 五、业务流程

### 5.1 生产工单流程

```
创建工单 → 下发工单 → 开始生产 → 扫码作业 → 完成生产 → 工单归档
    ↓          ↓          ↓          ↓
  已关闭    已关闭      已关闭    异常处理
```

### 5.2 扫码作业流程

```
扫描VIN码 → 系统识别 → 显示工序 → 执行作业 → 扫描关键件 → 完成报工
     ↓
  新VIN → 创建作业记录
  已有VIN → 显示进度，继续作业
```

### 5.3 质量检验流程

```
执行检验 → 记录结果 → 判定合格
              ↓
           不合格 → 记录原因 → 处理判定 → 执行处理 → 验证闭环
                              ↓
                    返修/报废/让步接收
```
