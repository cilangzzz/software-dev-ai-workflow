# 汽车制造业ERP系统 - 功能需求与架构设计

## 一、项目背景

### 1.1 系统定位

ERP（企业资源计划）系统定位于企业计划层，是汽车制造企业的核心业务管理平台，负责企业资源的统筹规划和协调管理。

**系统定位：多租户SaaS模式**，支持集团多工厂、多组织数据隔离与共享。

```
┌─────────────────────────────────────────────────────────────┐
│                      ERP计划层                              │
│           (销售、采购、库存、生产计划、财务)                 │
│                    【多租户SaaS平台】                        │
└─────────────────────────────┬───────────────────────────────┘
                              │ 接口对接
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      MES执行层                              │
│              (工单管理、作业执行、质量追溯)                  │
└─────────────────────────────┬───────────────────────────────┘
                              │ 设备集成
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      设备控制层                             │
│                 (PLC、扭矩枪、扫描枪)                       │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| 业务闭环 | 实现从订单到交付的完整业务流程管理 |
| 资源统筹 | 整合企业人、财、物、产、供、销资源 |
| 数据贯通 | 打通ERP与MES、CRM、SRM等系统数据壁垒 |
| 决策支持 | 提供实时经营数据分析与决策支持 |
| 多租户隔离 | 租户数据逻辑隔离，支持集团化部署 |

### 3.2 技术架构选型

**架构模式：单体应用架构**（基于芋道框架），适合中小规模应用，部署运维简单。

| 技术领域 | 技术选型 | 版本 | 说明 |
|----------|----------|------|------|
| **后端框架** | Spring Boot | 2.7.x | 单体应用框架 |
| **ORM框架** | MyBatis Plus | 3.5.x | 数据持久化、多租户插件 |
| **数据库** | MySQL | 8.x | 主数据库 |
| **缓存** | Redis | 6.x | 缓存、会话、分布式锁 |
| **消息队列** | RabbitMQ | 3.x | 异步消息处理 |
| **工作流** | Flowable | 6.8.x | 业务流程引擎 |
| **搜索引擎** | Elasticsearch | 7.x | 全文检索、日志分析（可选） |
| **安全框架** | Spring Security | 5.8.x | 认证授权 |
| **API文档** | Knife4j | 4.x | 接口文档 |

| **前端框架** | Vue3 + Element Plus | 3.x | PC端管理界面 |
| **移动端** | UniApp | 3.x | 微信小程序/H5/APP |
| **报表工具** | JimuReport | 1.x | 报表设计器 |

### 3.3 单体应用架构优势

| 优势 | 说明 |
|------|------|
| 部署简单 | 一个WAR/JAR包部署，无需复杂的微服务治理 |
| 开发效率高 | 模块间调用直接方法调用，调试方便 |
| 运维成本低 | 单一应用监控，日志集中管理 |
| 事务简单 | 本地事务即可满足需求，无需分布式事务 |
| 性能可控 | 无网络开销，性能优化相对简单 |
| 适合规模 | 日活用户<10万，数据量<千万级 |

### 4.2 数据交互矩阵

| ERP → MES | 方向 | MES → ERP | 说明 |
|-----------|------|-----------|------|
| 生产计划/工单 | → | 工单完成反馈 | 生产计划下达与进度反馈 |
| 物料需求计划 | → | 物料消耗记录 | 物料需求与消耗闭环 |
| 产品/BOM数据 | → | - | 主数据同步 |
| 工艺路线数据 | → | - | 工艺数据同步 |
| - | ← | 作业执行记录 | 生产执行数据回传 |
| - | ← | 质量检验记录 | 质量数据回传 |
| - | ← | 设备状态数据 | 设备运行数据回传 |

### 4.3 核心接口定义

| 接口名称 | 方向 | 触发时机 | 数据内容 |
|----------|------|----------|----------|
| 生产计划下达 | ERP→MES | 计划发布 | 订单号、产品、数量、计划日期 |
| 工单状态同步 | MES→ERP | 状态变更 | 工单号、状态、完成数量 |
| 物料消耗上报 | MES→ERP | 作业完成 | VIN、物料、消耗数量 |
| 库存变动同步 | 双向 | 实时/定时 | 仓库、物料、数量、批次 |
| 质量数据回传 | MES→ERP | 检验完成 | VIN、检验项、结果 |

### 6.4 多租户核心设计要点

| 设计要点 | 实现方式 | 说明 |

| 数据隔离 | tenant_id字段 | 所有业务表增加tenant_id，MyBatis拦截器自动过滤 |

| 租户配置 | 租户级参数配置 | 每个租户独立的系统参数 |
| 权限隔离 | 租户级角色权限 | 每个租户独立的用户角色体系 |
| 数据库策略 | 共享数据库 | 成本可控，运维简单 |

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 需求变更频繁 | 延期、成本增加 | 敏捷开发、分阶段交付、需求冻结机制 |
| 数据迁移困难 | 上线延期 | 提前规划、分批迁移、数据清洗 |
| 系统集成复杂 | 接口问题频发 | 标准接口规范、联调测试、异常处理 |
| 用户接受度低 | 推广困难 | 用户培训、操作手册、现场支持 |
| 性能瓶颈 | 用户体验差 | 架构优化、缓存策略、数据库优化 |
