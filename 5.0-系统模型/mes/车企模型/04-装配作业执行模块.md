# 装配作业执行模块需求文档

## 模块概述

| 属性 | 值 |
|------|-----|
| 模块名称 | 装配作业执行 |
| 英文名称 | Assembly Operation Execution |
| 优先级 | P0 |
| 所属系统 | 汽车整车装配MES系统 |

|--------|----------|--------|------|

| 作业指引 | 显示当前工序作业指导 | P0 | 待开发 |
| 工序报工 | 完成工序作业并报工 | P0 | 待开发 |
| 关键件绑定 | 扫描绑定关键零部件 | P0 | 待开发 |
| 作业记录查询 | 查询作业历史记录 | P0 | 待开发 |
| 异常上报 | 上报作业异常 | P1 | 待开发 |
| 扭矩数据关联 | 关联扭矩枪采集的扭矩数据 | P1 | 待开发 |

### 扫码类型识别规则

| 码类型 | 格式特征 | 长度 | 处理方式 |
|--------|----------|------|----------|
| VIN码 | 字母数字组合 | 17位 | 创建/查询车辆作业记录 |
| 工单码 | WO+日期+流水 | 14位 | 切换当前工单 |
| 物料码 | 物料编码规则 | 可变 | 关键件绑定/物料消耗 |

## 业务规则

| 规则编号 | 规则名称 | 规则描述 |
|----------|----------|----------|
| OP-001 | VIN校验 | VIN码为17位，需校验格式正确性 |
| OP-002 | 工序唯一 | 同一VIN同一工序只能有一条作业记录 |
| OP-003 | 关键件唯一 | 关键件序列号全局唯一，不可重复绑定 |
| OP-004 | 前置工序 | 前置工序未完成不可执行后续工序 |
| OP-005 | 关键件必扫 | 关键工序必须绑定所有指定的关键件 |
| OP-006 | 工单状态 | 工单必须为"生产中"状态才能扫码作业 |

## 用户故事

### US-OP-001: 扫描VIN开始作业

**作为** 作业员
**我想要** 扫描车辆VIN码开始装配作业
**以便于** 系统记录我的作业信息并显示作业指引

**验收标准：**
- Given 工单状态为"生产中"且工位已绑定
- When 作业员扫描VIN码
- Then 系统创建作业记录，显示当前工序和作业指引

### US-OP-002: 绑定关键零部件

**作为** 作业员
**我想要** 扫描关键零部件序列号绑定到车辆
**以便于** 实现零部件追溯

**验收标准：**
- Given 车辆正在进行关键工序作业
- When 作业员扫描关键件序列号
- Then 系统记录绑定关系，序列号不可重复使用

### US-OP-003: 完成工序报工

**作为** 作业员
**我想要** 完成当前工序并报工
**以便于** 系统记录作业结果并进入下一工序

**验收标准：**
- Given 所有关键件已绑定且作业已完成
- When 作业员点击完成报工
- Then 系统更新作业状态，车辆进入下一工序
