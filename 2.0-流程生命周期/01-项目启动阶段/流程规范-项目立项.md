# 流程规范：项目立项

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 项目立项 |
| 所属阶段 | 项目启动阶段 |
| 流程负责人 | 项目经理 |
| 前置流程 | 业务需求确认 |
| 后置流程 | 团队组建 |

## 流程目的

建立项目正式立项的规范流程，确保项目目标明确、资源到位、干系人认可。

## 触发条件

- 业务需求已明确
- 有初步的预算和时间预期
- 发起人提交立项申请

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 业务需求说明 | 业务方 | 必须 |
| 预算估算 | 财务 | 必须 |
| 时间预期 | 业务方 | 建议 |

## 执行步骤

### Step 1: 需求确认

- [ ] 与业务方确认需求范围
- [ ] 识别关键干系人
- [ ] 明确项目目标（SMART原则）

### Step 2: 可行性评估

- [ ] 技术可行性评估
- [ ] 资源可行性评估
- [ ] 风险初步评估

### Step 3: 项目章程编制

- [ ] 编制项目章程
- [ ] 定义项目范围
- [ ] 确定里程碑
- [ ] 明确预算和资源

### Step 4: 立项评审

- [ ] 组织立项评审会
- [ ] 获取干系人签字确认
- [ ] 确认资源分配

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 项目章程 | 文档 | 项目文档库 |
| 干系人登记表 | 表格 | 项目文档库 |
| 立项评审记录 | 文档 | 项目文档库 |

## 质量标准

- 项目目标符合SMART原则
- 干系人识别完整率 ≥ 95%
- 资源承诺已书面确认

## 相关Skill

| Skill | 用途 |
|-------|------|
| project-status | 生成项目状态报告 |
| gate-check | 立项阶段门控检查 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 需求不明确 | 返回需求确认阶段 |
| 资源不满足 | 升级至管理层协调 |
| 立项评审不通过 | 根据反馈修改后重新评审 |
