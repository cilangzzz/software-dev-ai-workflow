# 流程规范：部署执行

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 部署执行 |
| 所属阶段 | 部署阶段 |
| 流程负责人 | 运维工程师 |
| 前置流程 | 发布规划 |
| 后置流程 | 发布验证 |

## 流程目的

按照发布计划执行部署操作，确保部署过程顺利。

## 触发条件

- 发布计划已批准
- 发布时间窗口到达

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 发布计划 | 发布规划 | 必须 |
| 部署包 | 开发阶段 | 必须 |
| 配置文件 | 配置管理 | 必须 |

## 执行步骤

### Step 1: 部署前检查

- [ ] 检查部署包完整性
- [ ] 检查配置文件正确性
- [ ] 检查环境依赖
- [ ] 备份当前版本

### Step 2: 停服公告（如需要）

- [ ] 发布停服公告
- [ ] 执行停服操作
- [ ] 确认服务已停止

### Step 3: 执行部署

- [ ] 上传部署包
- [ ] 执行部署脚本
- [ ] 更新配置文件
- [ ] 执行数据库迁移

### Step 4: 服务启动

- [ ] 启动应用服务
- [ ] 检查启动日志
- [ ] 确认服务正常

### Step 5: 部署记录

- [ ] 记录部署时间
- [ ] 记录部署版本
- [ ] 记录操作人员

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 部署记录 | 文档 | 运维日志 |
| 部署日志 | 日志 | 日志系统 |

## 质量标准

- 部署步骤按计划执行
- 部署时间在窗口内
- 无部署错误

## 相关Skill

| Skill | 用途 |
|-------|------|
| ci-pipeline-assistant | CI/CD配置辅助 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 部署失败 | 按回滚方案执行 |
| 部署超时 | 评估后决定继续或回滚 |
