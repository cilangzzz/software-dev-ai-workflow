# 流程规范：需求变更

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 需求变更 |
| 所属阶段 | 需求分析阶段（跨阶段） |
| 流程负责人 | 产品经理 |
| 前置流程 | 需求评审 |
| 后置流程 | 变更实施 |

## 流程目的

规范需求变更流程，控制需求蔓延，确保变更可控。

## 触发条件

- 干系人提出需求变更请求
- 业务环境发生变化
- 技术条件发生变化

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 变更请求 | 干系人 | 必须 |
| 变更原因说明 | 申请人 | 必须 |

## 执行步骤

### Step 1: 变更申请

- [ ] 提交变更申请单
- [ ] 说明变更原因和期望
- [ ] 评估变更紧迫性

### Step 2: 影响评估

- [ ] 评估对进度的影响
- [ ] 评估对成本的影响
- [ ] 评估对质量的影响
- [ ] 评估对其他需求的影响

### Step 3: 变更评审

- [ ] 组织变更评审会议
- [ ] 讨论变更必要性
- [ ] 决策是否接受变更

### Step 4: 变更实施

- [ ] 更新PRD文档
- [ ] 更新用户故事
- [ ] 更新追溯矩阵
- [ ] 通知相关干系人

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 变更申请单 | 文档 | 项目文档库 |
| 影响评估报告 | 文档 | 项目文档库 |
| 变更评审记录 | 文档 | 项目文档库 |

## 质量标准

- 变更影响评估完整
- 变更决策有记录
- 文档同步更新

## 相关Skill

| Skill | 用途 |
|-------|------|
| requirement-change | 需求变更处理 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 变更影响重大 | 升级至管理层决策 |
| 变更被拒绝 | 说明原因，归档记录 |
