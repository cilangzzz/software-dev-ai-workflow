# 流程规范：架构设计

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 架构设计 |
| 所属阶段 | 设计阶段 |
| 流程负责人 | 技术负责人 |
| 前置流程 | 需求评审 |
| 后置流程 | 技术选型 |

## 流程目的

设计系统整体架构，明确组件划分、技术分层、部署架构等关键决策。

## 触发条件

- 需求评审已通过
- 非功能需求已明确

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| PRD文档 | 需求分析 | 必须 |
| 非功能需求 | 需求分析 | 必须 |

## 执行步骤

### Step 1: 业务架构设计

- [ ] 分析业务领域
- [ ] 识别业务边界
- [ ] 定义业务能力模型

### Step 2: 应用架构设计

- [ ] 划分系统模块
- [ ] 定义模块职责
- [ ] 设计模块间接口

### Step 3: 技术架构设计

- [ ] 选择架构风格（微服务/单体等）
- [ ] 设计技术分层
- [ ] 定义技术组件

### Step 4: 部署架构设计

- [ ] 设计部署拓扑
- [ ] 规划网络架构
- [ ] 设计容灾方案

### Step 5: 架构文档输出

- [ ] 绘制架构图
- [ ] 编写架构说明文档
- [ ] 记录架构决策（ADR）

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 系统架构图 | 图表 | 项目文档库 |
| 架构设计文档 | 文档 | 项目文档库 |
| 架构决策记录(ADR) | 文档 | 项目文档库 |

## 质量标准

- 架构满足非功能需求
- 组件职责单一
- 接口定义清晰

## 相关Skill

| Skill | 用途 |
|-------|------|
| architect | 架构设计辅助 |
| adr | 架构决策记录编写 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 架构方案争议 | 组织技术评审会议 |
| 技术风险高 | 制定技术预研计划 |
