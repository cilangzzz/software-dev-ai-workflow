# 流程治理规范

## 概述

本文档定义流程的创建、审批、发布、变更、废弃的生命周期管理规范。

## 流程生命周期

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  流程创建 │──▶│  流程审批 │──▶│  流程发布 │──▶│  流程执行 │──▶│  流程废弃 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
                   │                              │
                   │         ┌──────────┐         │
                   └────────▶│  流程变更 │◀────────┘
                             └──────────┘
```

## 流程定义规范

### 流程文档结构

每个流程文档应包含以下章节：

| 章节 | 必要性 | 说明 |
|------|--------|------|
| 基本信息 | 必须 | 流程名称、所属阶段、负责人、前置后置流程 |
| 流程目的 | 必须 | 流程的目标和价值 |
| 触发条件 | 必须 | 什么情况下启动该流程 |
| 输入物 | 必须 | 流程执行所需的输入材料 |
| 执行步骤 | 必须 | 详细的执行步骤和检查点 |
| 输出物 | 必须 | 流程执行完成后的交付物 |
| 质量标准 | 建议 | 流程执行的质量要求 |
| 相关Skill | 建议 | 可辅助执行的AI Skill |
| 异常处理 | 建议 | 常见异常场景的处理方式 |

### 命名规范

| 类型 | 命名格式 | 示例 |
|------|---------|------|
| 流程规范 | 流程规范-{流程名称}.md | 流程规范-项目立项.md |
| 检查清单 | 检查清单-{阶段}就绪.md | 检查清单-启动就绪.md |
| 模板文件 | {文档名称}模板.md | 项目章程模板.md |

## 流程审批机制

### 审批流程

1. **流程创建者** 提交流程文档
2. **流程负责人** 审核流程合理性
3. **相关部门负责人** 确认协作接口
4. **质量委员会** 最终批准

### 审批标准

| 审批项 | 标准 |
|--------|------|
| 完整性 | 文档包含所有必要章节 |
| 合理性 | 流程步骤逻辑清晰、可执行 |
| 兼容性 | 与现有流程无冲突 |
| 可度量 | 包含明确的质量标准或检查点 |

## 流程变更管理

### 变更类型

| 类型 | 说明 | 审批级别 |
|------|------|---------|
| 新增 | 新增流程文档 | 流程负责人审批 |
| 修订 | 修改流程内容 | 流程负责人审批 |
| 废弃 | 废弃不再使用的流程 | 质量委员会审批 |

### 变更流程

1. 提出变更申请，说明变更原因和影响范围
2. 评估变更影响，识别受影响的流程和角色
3. 修订流程文档，更新版本号
4. 审批变更申请
5. 发布变更通知，通知相关执行者
6. 更新培训材料

### 版本管理

| 版本号 | 说明 |
|--------|------|
| v1.0 | 初始发布版本 |
| v1.1 | 小幅修订（如调整步骤顺序） |
| v2.0 | 重大变更（如增加新环节） |

## 流程废弃标准

满足以下条件之一的流程可以废弃：

- 流程目标已不存在
- 有新流程替代
- 长期未执行（超过6个月）

废弃流程需保留归档，并在流程索引中标注"已废弃"。
