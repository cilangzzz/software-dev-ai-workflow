# 流程工具集成

## 概述

本文档说明如何将AI Skill与流程执行集成，实现自动化流程驱动。


## 集成方式

### 1. 命令触发

通过命令直接调用Skill执行流程活动：

| 阶段 | Skill命令 | 用途 |
|------|----------|------|
| 需求阶段 | `/requirement-analyzer` | 生成PRD框架 |
| 需求阶段 | `/user-story-generator` | 生成用户故事 |
| 设计阶段 | `/architect` | 生成架构设计 |
| 开发阶段 | `/code-review` | 代码审查 |
| 测试阶段 | `/test-case-generator` | 生成测试用例 |
| 项目管理 | `/gate-check` | 质量门控检查 |
| 项目管理 | `/project-status` | 项目状态报告 |

### 2. 流程规范集成

在流程规范文档中标注相关Skill：

```markdown
## 相关Skill

| Skill | 用途 | 触发方式 |
|-------|------|---------|
| requirement-analyzer | 生成PRD框架 | /requirement-analyzer |
| gate-check | 需求就绪检查 | /gate-check stage=requirement |
```

### 3. 检查清单自动化

检查清单中的检查项可配置自动化检查：

```markdown
## 自动检查项

- [ ] PRD文档完整性 (AI检查) → 调用 gate-check
- [ ] 用户故事格式 (AI检查) → 调用 user-story-generator
```

## Skill与流程映射

### 需求阶段

| 流程活动 | 推荐Skill | 输入 | 输出 |
|---------|----------|------|------|
| 需求采集 | requirement-analyzer | 用户反馈、业务需求 | PRD框架 |
| 用户故事编写 | user-story-generator | 需求描述 | 用户故事列表 |
| 验收标准编写 | acceptance-criteria-writer | 用户故事 | 验收标准 |
| 需求就绪检查 | gate-check | stage=requirement | 检查报告 |

### 设计阶段

| 流程活动 | 推荐Skill | 输入 | 输出 |
|---------|----------|------|------|
| 架构设计 | architect | PRD、技术需求 | 架构设计文档 |
| 技术决策记录 | adr | 技术选型问题 | ADR文档 |
| 设计就绪检查 | gate-check | stage=design | 检查报告 |

### 开发阶段

| 流程活动 | 推荐Skill | 输入 | 输出 |
|---------|----------|------|------|
| 项目脚手架 | scaffold | 架构设计 | 项目结构 |
| 代码实现 | implement | 详细设计 | 源代码 |
| 代码审查 | code-review | 源代码 | 审查报告 |
| 开发就绪检查 | gate-check | stage=develop | 检查报告 |

### 测试阶段

| 流程活动 | 推荐Skill | 输入 | 输出 |
|---------|----------|------|------|
| 测试用例设计 | test-case-generator | 需求文档 | 测试用例 |
| 测试执行 | test-executor | 测试用例 | 测试报告 |
| 缺陷分析 | bug-analyzer | 缺陷信息 | 缺陷报告 |
| 测试就绪检查 | gate-check | stage=test | 检查报告 |

### 部署阶段

| 流程活动 | 推荐Skill | 输入 | 输出 |
|---------|----------|------|------|
| 部署方案分析 | deploy-analyzer | 系统架构 | 部署方案 |
| CI/CD配置 | ci-pipeline-assistant | 部署需求 | 流水线配置 |
| 发布就绪检查 | gate-check | stage=release | 检查报告 |

## 自动化流程触发

### 事件驱动触发

| 触发事件 | 自动执行 |
|---------|---------|
| 代码提交 | code-review |
| PR创建 | code-review + 安全扫描 |
| 合并到main | gate-check stage=develop |
| 部署完成 | gate-check stage=release |

### 定时触发

| 定时任务 | 执行频率 |
|---------|---------|
| 项目状态报告 | 每周一 |
| 风险检查 | 每日 |
| 质量趋势分析 | 每两周 |
