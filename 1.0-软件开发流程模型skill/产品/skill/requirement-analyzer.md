# Skill: requirement-analyzer

## 基本信息
- **名称**: requirement-analyzer
- **版本**: 1.0.0
- **所属部门**: 产品部
- **优先级**: P0

## 功能描述
解析模糊的用户需求描述，自动提取关键信息，生成结构化的PRD框架。帮助产品经理快速将零散的需求信息整理成规范的文档。

## 触发条件
- 命令触发: `/requirement-analyzer`
- 自然语言触发:
  - "分析这个需求"
  - "帮我整理需求"
  - "生成PRD框架"
  - "解析用户反馈"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| requirement_text | string | 是 | 原始需求描述文本 |
| business_context | string | 否 | 业务背景信息 |
| target_users | string | 否 | 目标用户群体描述 |

## 执行流程
1. **需求解析** - 分析输入文本，识别核心需求点
2. **实体提取** - 提取关键实体、角色、动作
3. **关系识别** - 识别需求之间的依赖和冲突
4. **结构化输出** - 生成PRD框架文档
5. **澄清问题** - 生成待确认的问题清单

```markdown

- [功能点1]
- [功能点2]

### 边界条件
- [边界条件1]
- [边界条件2]

## 非功能需求
- 性能要求：
- 安全要求：
- 兼容性要求：

## 待澄清问题
```

## 质量标准
- 需求覆盖率 ≥ 95%（所有输入需求点都被处理）
- 歧义检测准确率 ≥ 80%
- 澄清问题相关性 ≥ 90%
- 输出文档结构完整性 100%

## 使用示例

### 示例1：用户反馈分析
**输入**:
```
用户反馈：希望能在手机上查看订单状态，有时候在外面也需要确认订单走到哪一步了，
还有就是支付的时候老是报错，特别是用微信支付的时候。
```

**输出**:
```markdown
# 产品需求文档

## 需求概述
用户需要在移动端查看订单状态，并解决微信支付报错问题。

## 用户故事
1. 作为外出用户，我希望能在手机上查看订单状态，以便随时了解订单进度
2. 作为支付用户，我希望微信支付能正常工作，以便顺利完成订单

## 功能需求
### 核心功能
- 移动端订单状态查询页面
- 订单进度追踪功能
- 微信支付异常修复

### 边界条件
- 支持iOS和Android
- 订单状态实时更新（延迟<3秒）
- 支持断网情况下查看缓存订单

## 待澄清问题
1. 订单状态具体包含哪些节点？
2. 微信支付报错的具体错误信息是什么？
3. 是否需要推送通知功能？

## 建议优先级
- P0: 微信支付异常修复（影响交易）
- P1: 移动端订单查询
- P2: 推送通知功能
```

## 依赖工具
- Read - 读取已有需求文档
- Write - 输出PRD文档
- Grep - 搜索相关需求
- WebSearch - 查询行业最佳实践

## 注意事项
- AI生成的PRD框架需要人工审核和补充
- 澄清问题需要与业务方确认
- 优先级建议仅供参考，需要结合业务实际情况调整
- 注意检测需求冲突和遗漏场景
