# Skill: data-quality-check

## 基本信息
- **名称**: data-quality-check
- **版本**: 1.0.0
- **所属部门**: 数据部
- **优先级**: P0

## 功能描述
执行数据质量检查，包括完整性、准确性、一致性、及时性等多维度检测。发现数据异常并提供修复建议。

## 触发条件
- 命令触发: `/data-quality-check`
- 自然语言触发:
  - "检查数据质量"
  - "数据质量分析"
  - "数据异常检测"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| data_source | string | 是 | 数据源路径或连接信息 |
| check_types | string | 否 | 检查类型：completeness / accuracy / consistency / timeliness / all |
| threshold | string | 否 | 质量阈值，低于此值报警 |

## 执行流程
1. **数据采样** - 采集数据样本
2. **完整性检查** - 检查缺失值、空值
3. **准确性检查** - 检查数据格式、范围
4. **一致性检查** - 检查跨表一致性
5. **异常检测** - AI智能异常识别
6. **报告生成** - 生成质量报告

## 数据质量维度
- **完整性**: 数据是否存在缺失
- **准确性**: 数据是否正确
- **一致性**: 数据是否一致
- **及时性**: 数据是否新鲜
- **唯一性**: 数据是否重复

## 使用示例

### 示例：订单数据质量检查
**输入**:
```
data_source: orders_table
check_types: all
threshold: 80
```

**输出**:
```markdown
# 数据质量报告

## 概览
- 数据源: orders_table
- 检查时间: 2024-01-15 09:00:00
- 记录数: 1,234,567
- 整体质量评分: 78/100 ⚠️

## 质量维度评分
| 维度 | 评分 | 状态 | 问题数 |
|------|------|------|--------|
| 完整性 | 92 | ✅ | 3 |
| 准确性 | 85 | ✅ | 12 |
| 一致性 | 65 | ❌ | 45 |
| 及时性 | 70 | ⚠️ | 8 |
| 唯一性 | 78 | ⚠️ | 156 |

## 问题详情

### 🔴 严重问题
| 问题类型 | 位置 | 影响 | 建议 |
|----------|------|------|------|
| 跨表不一致 | order.user_id vs users.id | 订单关联用户不存在 | 修复外键约束，清理孤立订单 |
| 时间戳异常 | created_at字段 | 234条记录时间在未来 | 检查时区配置，修正错误数据 |

### 🟡 中等问题
| 问题类型 | 位置 | 建议 |
|----------|------|------|
| 字段缺失 | delivery_address | 12%订单缺少收货地址（可能是虚拟商品） |
| 重复记录 | order_id | 发现156条重复订单号 |

## 异常数据样本
| 字段 | 异常值 | 出现次数 | 类型 |
|------|--------|----------|------|
| amount | -99.00 | 23 | 负值异常 |
| status | 'UNKNOWN' | 45 | 枚举异常 |
| created_at | 2099-01-01 | 12 | 时间异常 |

## 数据分布
- 订单金额分布: 正常，均值¥156，中位数¥89
- 订单状态分布: 待支付15%, 已支付65%, 已发货18%, 已取消2%
- 时间分布: 存在周末数据缺失，需检查ETL调度

## 改进建议
1. 建立外键约束，防止孤立数据产生
2. 添加数据校验规则，拒绝负金额订单
3. 检查ETL任务调度，确保周末数据同步
4. 实施数据去重策略
```

## 质量标准
- 异常检出率 ≥ 85%
- 误报率 ≤ 20%
- 检查响应时间 < 5分钟/百万行

## 依赖工具
- Read - 读取数据文件
- Bash - 执行数据查询
- Write - 输出报告

## 注意事项
- 大数据量建议采样检查
- 注意数据隐私保护
- 敏感数据需要脱敏处理
