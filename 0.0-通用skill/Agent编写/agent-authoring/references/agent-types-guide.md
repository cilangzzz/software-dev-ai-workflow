## 二、按角色特征分类

### 研发类 Agent 特征

#### 后端开发
- **核心能力**: API设计、数据处理、业务逻辑实现
- **技术栈**: Java/Spring Boot, Python/FastAPI, Go/Gin, Node.js/Express
- **产出物**: API接口、业务代码、数据库设计、技术文档
- **协作角色**: 前端开发、DBA、测试工程师

#### 前端开发
- **核心能力**: 界面开发、组件设计、用户体验实现
- **技术栈**: Vue/React/Angular, Qt/Flutter, Tailwind/SCSS
- **产出物**: 前端组件、页面代码、样式文件、交互设计
- **协作角色**: 后端开发、UI设计师、产品经理

#### DBA
- **核心能力**: 数据库设计、性能优化、数据安全
- **技术栈**: MySQL/PostgreSQL/Oracle, Redis/MongoDB
- **产出物**: 表结构、索引策略、优化报告、备份方案
- **协作角色**: 后端开发、架构师、运维工程师

### 产品类 Agent 特征

#### B端产品经理
- **核心能力**: 业务需求分析、功能规划、流程设计
- **关注点**: 业务效率、功能完整性、系统稳定性
- **产出物**: PRD、业务流程图、功能列表、验收标准
- **协作角色**: 研发、设计、测试、运营

#### C端产品经理
- **核心能力**: 用户需求洞察、增长策略、体验优化
- **关注点**: 用户增长、转化率、用户满意度
- **产出物**: 产品规划、增长方案、体验优化方案
- **协作角色**: 设计、研发、运营、数据分析

#### 数据产品经理
- **核心能力**: 数据需求分析、指标设计、数据产品规划
- **关注点**: 数据价值、指标体系、数据应用
- **产出物**: 数据产品规划、指标体系、数据需求文档
- **协作角色**: 数据分析师、数据工程师、研发

### 设计类 Agent 特征

#### UI设计师
- **核心能力**: 界面设计、视觉规范、设计系统
- **工具**: Figma/Sketch/Adobe XD
- **产出物**: UI设计稿、设计规范、图标库、样式指南
- **协作角色**: 前端开发、产品经理、UX设计师

#### UX设计师
- **核心能力**: 用户研究、交互设计、体验优化
- **方法**: 用户访谈、可用性测试、原型设计
- **产出物**: 交互设计稿、用户研究报告、原型文件
- **协作角色**: 产品经理、UI设计师、前端开发

### 运维类 Agent 特征

#### DevOps工程师
- **核心能力**: CI/CD、容器化、自动化运维
- **技术栈**: Docker/K8s, Jenkins/GitLab CI, Terraform/Ansible
- **产出物**: 部署脚本、CI配置、监控配置、运维文档
- **协作角色**: 研发、测试、安全工程师

#### SRE工程师
- **核心能力**: 可靠性保障、监控告警、故障处理
- **技术栈**: Prometheus/Grafana, ELK, 云平台
- **产出物**: SLO定义、监控方案、故障预案
- **协作角色**: 研发、运维、测试

## 三、Agent 设计参考模板

### 设计流程

1. **明确角色定位**
   - 定义角色所属部门和类别
   - 明确角色的核心职责
   - 确定角色的能力边界

2. **定义核心能力**
   - 列出角色的核心技能（3-5项）
   - 为每个技能设定能力等级
   - 定义技能的具体能力项

3. **确定技术栈**
   - 根据角色类型确定技术栈
   - 明确技术栈版本要求
   - 考虑技术栈的关联工具

4. **配置 Skill 加载**
   - 确定技术特定 Skill
   - 确定公共 Skill
   - 排除无关 Skill

5. **定义工作流**
   - 设计典型工作流程
   - 明确每个步骤的 Skill
   - 定义协作关系

6. **制定规范**
   - 定义命名规范
   - 定义代码风格
   - 定义注意事项

### 能力等级定义

| 等级 | 描述 | 能力特征 |
|------|------|---------|
| **expert** | 专家级 | 能够独立解决复杂问题，指导他人，制定最佳实践 |
| **advanced** | 高级 | 能够独立完成复杂任务，处理异常情况，优化方案 |
| **intermediate** | 中级 | 能够独立完成常规任务，需要指导处理复杂问题 |
| **beginner** | 初级 | 能够完成基础任务，需要较多指导 |

## 五、设计参考建议

1. **先定义大类，再细化子类**
   - 先确定角色的部门和大类
   - 再根据具体场景细化子类

2. **能力等级要合理**
   - 不要所有技能都设为 expert
   - 核心技能设为 expert，相关技能设为 advanced/intermediate

3. **Skill 加载要精准**
   - 技术特定 Skill 加载核心技能
   - 公共 Skill 加载通用能力
   - 排除 Skill 明确排除原因

4. **工作流要完整**
   - 从需求理解开始
   - 包含设计、开发、测试、上线
   - 定义每个步骤的 Skill 和协作关系

5. **规范要具体**
   - 命名规范要有示例
   - 代码风格要有模板
   - 注意事项要覆盖常见问题
