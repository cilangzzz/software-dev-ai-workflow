# Word 文档生成规则

## 概述

将接口文档转换为 Microsoft Word (.docx) 格式，便于打印、分享和离线查看。

## 生成方式

使用程序化方式生成 DOCX 文件（如 python-docx 或 Node.js docx 库）。

## 文档结构

```
{项目名称} API 文档

1. 文档信息
   - 项目名称
   - 版本号
   - 生成日期
   - 服务器地址

2. 接口列表
   2.1 {模块名称}
       2.1.1 {接口名称}
             - 请求方法和路径
             - 请求参数表
             - 请求体示例
             - 响应参数表
             - 响应示例
```

## 格式规范

### 标题层级

| 层级 | 用途 | 格式 |
|------|------|------|
| Heading 1 | 项目名称 / 大章节 | 文档信息、接口列表 |
| Heading 2 | 模块/Tag名称 | 登录模块、用户模块 |
| Heading 3 | 接口名称 | 登录接口、自动登录 |

### 参数表格格式

**请求参数表：**

| 参数名 | 位置 | 类型 | 必填 | 描述 | 示例 |
|--------|------|------|------|------|------|
| Authorization | header | string | 是 | Bearer Token | {{ACCESS_TOKEN}} |
| username | body | string | 是 | 用户名 | admin |

**响应参数表：**

| 参数名 | 类型 | 描述 |
|--------|------|------|
| code | integer | 状态码 |
| data | object | 响应数据 |
| data.userId | integer | 用户ID |
| data.accessToken | string | 访问令牌 |
| msg | string | 提示信息 |

### 代码示例格式

使用等宽字体的代码块展示 JSON 请求体和响应体。

### 页面设置

- 纸张大小：A4
- 页边距：普通（2.54cm）
- 正文字体：宋体/微软雅黑，小四号
- 代码字体：Consolas/Courier New，小五号
- 表格边框：1px 实线

## 生成步骤

1. 从 OpenAPI 规范或中间数据模型读取接口信息
2. 按模块/Tag分组接口
3. 为每个接口生成参数表格和代码示例
4. 使用 docx 库组装文档
5. 输出 .docx 文件
