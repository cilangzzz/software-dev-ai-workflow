# Skill: test-case-generator

## 基本信息
- **名称**: test-case-generator
- **版本**: 1.0.0
- **所属部门**: 测试部
- **优先级**: P0

## 功能描述
根据需求文档、用户故事或代码自动生成测试用例。支持多种测试类型，包括功能测试、边界测试、异常测试等。

## 触发条件
- 命令触发: `/test-case-generator`
- 自然语言触发:
  - "生成测试用例"
  - "创建测试案例"
  - "生成测试场景"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| source | string | 是 | 需求文档/用户故事/代码路径 |
| source_type | string | 否 | 来源类型：requirement / code / story |
| test_type | string | 否 | 测试类型：functional / integration / e2e |

## 执行流程
1. **内容解析** - 解析需求/代码内容
2. **功能点提取** - 识别测试对象的功能点
3. **场景分析** - 分析正常、异常、边界场景
4. **用例生成** - 生成结构化测试用例
5. **优先级排序** - 按风险和重要性排序

## 输出格式

### 测试用例文档
```markdown
# 测试用例集

## 概述
- 测试对象: {对象名称}
- 用例总数: {count}
- 优先级分布: P0={n1}, P1={n2}, P2={n3}

## 测试用例列表

### TC-001: {用例标题}
| 属性 | 内容 |
|------|------|
| 用例ID | TC-001 |
| 优先级 | P0 |
| 前置条件 | {条件} |
| 测试步骤 | 1. {步骤1}<br>2. {步骤2} |
| 预期结果 | {结果} |
| 测试数据 | {数据} |

## 测试场景覆盖
| 场景类型 | 用例数 | 覆盖率 |
|----------|--------|--------|
| 正常流程 | {n} | {p}% |
| 异常流程 | {n} | {p}% |
| 边界条件 | {n} | {p}% |
```

## 使用示例

### 示例：登录功能测试用例
**输入**:
```
用户故事：作为注册用户，我希望使用手机号和密码登录系统。
验收标准：
- 正确凭据登录成功
- 错误密码提示错误
- 手机号格式校验
```

**输出**:
```markdown
# 测试用例集 - 用户登录功能

## 概述
- 测试对象: 用户登录模块
- 用例总数: 12
- 优先级分布: P0=4, P1=5, P2=3

## 测试用例列表

### TC-001: 正确凭据登录成功
| 属性 | 内容 |
|------|------|
| 用例ID | TC-001 |
| 优先级 | P0 |
| 前置条件 | 用户已注册手机号13800138000 |
| 测试步骤 | 1. 打开登录页面<br>2. 输入手机号13800138000<br>3. 输入正确密码<br>4. 点击登录按钮 |
| 预期结果 | 1. 登录成功<br>2. 跳转到首页<br>3. 显示用户信息 |
| 测试数据 | 手机号:13800138000, 密码:Test@123 |

### TC-002: 错误密码登录失败
| 属性 | 内容 |
|------|------|
| 用例ID | TC-002 |
| 优先级 | P0 |
| 前置条件 | 用户已注册 |
| 测试步骤 | 1. 打开登录页面<br>2. 输入正确手机号<br>3. 输入错误密码<br>4. 点击登录 |
| 预期结果 | 1. 显示"手机号或密码错误"<br>2. 保留在登录页 |

### TC-003: 手机号格式校验
| 属性 | 内容 |
```

- 功能覆盖率 ≥ 95%
- 边界条件覆盖率 ≥ 80%
- 用例可执行性 100%

## 依赖工具

- 边界值测试容易遗漏，需特别关注
- 考虑兼容性和性能测试场景
