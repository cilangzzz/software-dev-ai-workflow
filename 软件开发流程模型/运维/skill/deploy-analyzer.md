# Skill: deploy-analyzer

## 基本信息
- **名称**: deploy-analyzer
- **版本**: 1.0.0
- **所属部门**: 运维部
- **优先级**: P0

## 功能描述
部署前风险评估和部署计划生成。分析变更内容、依赖关系、环境差异，生成风险评估报告和部署方案。

## 触发条件
- 命令触发: `/deploy-analyzer`
- 自然语言触发:
  - "分析部署风险"
  - "生成部署计划"
  - "部署前检查"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| version | string | 是 | 待部署版本号 |
| environment | string | 是 | 目标环境：dev / staging / prod |
| changes | string | 否 | 变更内容描述或Git diff |

## 执行流程
1. **变更分析** - 分析版本变更内容
2. **依赖检查** - 检查依赖变更和兼容性
3. **环境对比** - 对比源环境和目标环境
4. **风险评估** - 评估部署风险点
5. **计划生成** - 生成部署计划和回滚方案

## 使用示例

### 示例：生产环境部署分析
**输入**:
```
version: v2.3.0
environment: prod
changes: |
  - 新增商品评价功能
  - 修复订单支付超时问题
  - 升级Redis客户端版本
  - 数据库Schema变更：添加评价表
```

**输出**:
```markdown
# 部署分析报告

## 版本信息
- 版本号: v2.3.0
- 目标环境: prod
- 部署时间: 待定

## 变更摘要
| 类型 | 数量 | 详情 |
|------|------|------|
| 新功能 | 1 | 商品评价功能 |
| 修复 | 1 | 订单支付超时 |
| 优化 | 1 | Redis客户端升级 |
| 破坏性变更 | 1 | 数据库Schema变更 |

## 风险评估
### 高风险项 🔴
| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 数据库Schema变更 | 可能导致服务启动失败 | 先执行DDL，再部署服务 |
| Redis客户端升级 | 连接池配置变更 | 提前测试连接参数 |

### 中风险项 🟠
| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 新增评价功能 | 可能影响商品详情页性能 | 添加缓存，监控响应时间 |

## 部署计划
### 前置检查
- [ ] 数据库备份完成
- [ ] Redis配置参数确认
- [ ] 监控告警规则更新

### 部署步骤
1. 00:00 - 执行数据库DDL（添加评价表）
2. 00:30 - 灰度发布10%流量
3. 01:00 - 验证评价功能正常
4. 01:30 - 逐步扩大至100%
5. 02:00 - 全量部署完成

### 验证步骤
1. 检查服务健康状态
2. 验证评价功能可用
3. 监控错误率和响应时间

## 回滚方案
### 回滚条件
- 错误率超过1%
- 核心功能不可用
- 响应时间超过阈值2倍

### 回滚步骤
1. 切换流量到旧版本
2. 回滚数据库变更
3. 恢复Redis配置

## 建议
- 建议在业务低峰期（凌晨）部署
- 提前通知客服团队新功能上线
- 准备好功能开关，必要时可快速关闭
```

## 质量标准
- 风险识别覆盖率 ≥ 90%
- 部署步骤可执行性 100%
- 回滚方案可执行性 100%

## 依赖工具
- Bash - 执行检查命令
- Read - 读取配置文件
- Grep - 搜索相关配置

## 注意事项
- 生产部署必须经过staging验证
- 破坏性变更需要额外关注
- 保持回滚能力是关键
