# Skill: security-threat-model

## 基本信息
- **名称**: security-threat-model
- **版本**: 1.0.0
- **所属部门**: 安全部
- **优先级**: P0

## 功能描述
进行威胁建模，使用STRIDE方法分析系统安全威胁。识别潜在安全风险，提供缓解措施建议。

## 触发条件
- 命令触发: `/security-threat-model`
- 自然语言触发:
  - "威胁建模"
  - "安全威胁分析"
  - "STRIDE分析"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| system_description | string | 是 | 系统描述或架构图 |
| data_flow | string | 否 | 数据流描述 |
| trust_boundaries | string | 否 | 信任边界定义 |

## 执行流程
1. **系统分析** - 解析系统组件和交互
2. **边界识别** - 识别信任边界
3. **威胁识别** - 使用STRIDE分类威胁
4. **风险评估** - 评估威胁风险等级
5. **缓解建议** - 提供安全控制建议

## STRIDE威胁分类
- **S**poofing（欺骗）- 冒充合法用户或系统
- **T**ampering（篡改）- 恶意修改数据
- **R**epudiation（抵赖）- 否认执行操作
- **I**nformation Disclosure（信息泄露）- 未授权访问信息
- **D**enial of Service（拒绝服务）- 破坏服务可用性
- **E**levation of Privilege（权限提升）- 获取未授权权限

## 使用示例

### 示例：API系统威胁建模
**输入**:
```
system_description: |
  用户通过移动App访问后端API，API连接MySQL数据库和Redis缓存。
  支持用户登录、查询订单、修改个人信息等功能。
  使用JWT进行身份认证。
```

**输出**:
```markdown
# 威胁建模报告

## 系统概述
移动App + API后端 + MySQL + Redis，支持用户认证和订单管理。

## 信任边界
- 边界1: 移动App → API服务器（不可信网络）
- 边界2: API服务器 → 数据库（可信内网）
- 边界3: API服务器 → Redis（可信内网）

## 威胁清单

### Spoofing（欺骗）
| ID | 威胁描述 | 风险 | 缓解措施 |
|----|----------|------|---------|
| S-01 | 攻击者伪造JWT访问他人账户 | 高 | 使用强密钥签名，设置合理过期时间 |
| S-02 | 中间人攻击截获凭据 | 高 | 强制HTTPS，证书校验 |
| S-03 | 重放攻击 | 中 | JWT添加jti，Redis记录已使用token |

### Tampering（篡改）
| ID | 威胁描述 | 风险 | 缓解措施 |
|----|----------|------|---------|
| T-01 | 修改他人订单信息 | 高 | 接口权限校验，操作日志记录 |
| T-02 | SQL注入篡改数据库 | 高 | 参数化查询，输入验证 |

### Information Disclosure（信息泄露）
| ID | 威胁描述 | 风险 | 缓解措施 |
|----|----------|------|---------|
| I-01 | 敏感信息未加密存储 | 高 | 密码bcrypt加密，敏感数据AES加密 |
| I-02 | API返回过多用户信息 | 中 | 响应数据脱敏处理 |

## 风险矩阵
| 风险等级 | 数量 | 示例 |
|----------|------|------|
| 高 | 5 | JWT伪造、SQL注入、中间人攻击 |
| 中 | 3 | 重放攻击、信息泄露 |
| 低 | 2 | 日志信息暴露 |

## 安全需求
- 所有通信必须使用TLS 1.2+
- 密码必须使用bcrypt存储
- 敏感操作需要二次验证
- API请求需要限流保护

## 建议优先级
1. 实现HTTPS强制跳转
2. JWT安全配置优化
3. SQL注入防护审计
4. 敏感数据加密存储
```
