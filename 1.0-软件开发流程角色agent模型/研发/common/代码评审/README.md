# 代码评审 Skill 目录

本目录包含代码审查相关的 Skill 定义和配置文件。

## 文件列表

| 文件 | 类型 | 描述 |
|------|------|------|
| [code-review.md](code-review.md) | Skill 定义 | 代码审查 Skill 的完整定义文档 |
| [role-config.yaml](role-config.yaml) | 配置文件 | 代码评审角色配置和**触发条件**定义 |

## ⚠️ 按需加载机制

本 Skill 采用**条件触发加载**机制，只有在满足特定条件时才会被自动加载。

### 自动触发条件

| 触发类型 | 条件 | 说明 |
|----------|------|------|
| Git 事件 | `pre_commit`, `pre_push` | Git 提交/推送前自动触发 |
| PR 事件 | `pr_create`, `pr_update` | PR 创建/更新时自动触发 |
| 文件变更 | `*.java`, `*.py`, `*.js`, `*.ts`, `*.vue`, `*.go` | 代码文件变更时触发 |
| 显式命令 | `/code-review` | 手动触发审查 |

### 跳过审查条件

| 条件 | 说明 |
|------|------|
| `[skip-review]` | 提交消息包含此标记跳过审查 |
| 草稿模式 | 草稿 PR 不触发审查 |
| 仅文档变更 | `.md`, `.txt`, `.json`, `.yaml` 变不触发 |

## 快速使用

### 显式触发命令
```
/code-review
```

### 自然语言触发
- "审查代码"
- "检查代码质量"
- "review代码"
- "代码评审"

### 常用参数
```yaml
target: src/main/java/com/example/  # 审查目标
review_type: full                    # quick / full / security
focus: all                           # quality / security / performance / all
```

## 审查类型说明

| 类型 | 描述 | 适用场景 |
|------|------|----------|
| quick | 快速审查 | 日常开发，检查基础问题 |
| full | 完整审查 | PR合并前，全面检查 |
| security | 安全专项 | 安全审计，漏洞扫描 |

## 输出产物
- 代码审查报告
- 问题列表（按严重等级分类）
- 质量评分表
- 修复建议详情

## 检查维度

### 安全检查
- SQL注入
- XSS漏洞
- 敏感信息泄露
- 硬编码密码
- 不安全的随机数

### 性能检查
- N+1查询
- 内存泄漏
- 不必要的循环
- 过大的对象创建

### 代码质量
- 代码重复
- 过长方法
- 过深嵌套
- 魔法数字
- 缺少注释

## 阻断规则

```yaml
阻断条件:
  critical_issues: ">0"     # 有严重问题必须修复
  major_issues: ">=3"       # 重要问题>=3需警告确认
```
