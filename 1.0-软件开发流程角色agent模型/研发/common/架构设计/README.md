# 架构设计 Skill 目录

本目录包含架构设计相关的 Skill 定义和配置文件，主要用于记录和管理技术决策。

## 文件列表

| 文件 | 类型 | 描述 |
|------|------|------|
| [adr.md](adr.md) | Skill 定义 | 架构决策记录(ADR) Skill 的完整定义文档 |
| [role-config.yaml](role-config.yaml) | 配置文件 | 架构设计角色配置和参数说明 |

## 快速使用

### 触发命令
```
/adr
```

### 自然语言触发
- "创建ADR"
- "记录架构决策"
- "写决策文档"
- "技术决策记录"

### 常用参数
```yaml
title: 决策标题
context: 决策背景和上下文说明
decision: 决策内容（可选）
alternatives: 备选方案列表（可选）
```

## ADR 状态说明

| 状态 | 描述 |
|------|------|
| Proposed | 提议中，待讨论 |
| Accepted | 已接受，开始执行 |
| Deprecated | 已弃用，不再推荐 |
| Superseded | 已取代，被新决策替代 |

## 决策分类

### 技术选型类
- 数据库选型
- 缓存方案选择
- 消息队列选择

### 架构设计类
- 服务拆分决策
- 模块边界划分
- 通信方式选择

### 最佳实践类
- API设计规范
- 代码组织规范
- 错误处理策略

### 安全合规类
- 认证方案选择
- 数据加密策略
- 日志审计要求

## 输出产物
- ADR文档（标准格式）
- 备选方案对比表
- 影响评估报告
- 决策追溯索引

## 文档命名规则
```
ADR-{序号}-{简短标题}.md
示例：
  ADR-001-选择PostgreSQL作为主数据库.md
  ADR-002-使用Redis作为分布式缓存.md
```
