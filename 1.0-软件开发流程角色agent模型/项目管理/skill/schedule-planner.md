## 触发条件

| 触发方式 | 条件 |
|---------|------|
| 关键词 | "排期"、"规划"、"甘特图"、"工时"、"任务分配" |
| 命令 | `/schedule` |
| 事件 | 收到需求清单、项目启动 |

## 输入参数

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|-------|------|------|--------|------|
| feature_list | array | true | - | 功能点清单（编码、名称、优先级、类型） |
| dev_team | array | true | - | 开发人员列表（姓名、技能、可用工时） |
| test_team | array | false | - | 测试人员列表 |
| release_date | string | true | - | 发版日期（YYYY-MM-DD） |
| sprint_length | number | false | 14 | Sprint周期（天），默认2周 |
| output_format | string | false | all | 输出格式：excel/mermaid/msproject/all |

## 执行流程

### Phase 1: 需求收集分析（5分钟）

| 步骤 | 动作 | 输出 |
|------|------|------|
| 1 | 读取功能点清单 | 功能数量、模块分布 |
| 2 | 统计优先级分布 | 高/中/低数量 |
| 3 | 统计类型分布 | 新增/优化/修复数量 |

### Phase 2: 工时估算（10分钟）

| 步骤 | 动作 | 方法 |
|------|------|------|
| 1 | 技术复杂度评估 | 高/中/低分级 |
| 2 | 基础工时估算 | 复杂度→基础工时映射 |
| 3 | 三点估算计算 | (乐观+4×最可能+悲观)/6 |
| 4 | 缓冲系数调整 | 高×1.5, 中×1.3, 低×1.2 |

**复杂度评估标准**:
| 等级 | 基础工时 | 识别特征 |
|------|---------|---------|
| 高 | 3-5天 | 新增数据库表、并发控制、权限体系、第三方集成、短信推送 |
| 中 | 1.5-3天 | 流程调整、字段扩展、组件复用、权限控制 |
| 低 | 0.5-1.5天 | UI调整、校验逻辑、Bug修复、CSS调整、默认值修改 |

### Phase 3: 人员分配（10分钟）

| 步骤 | 动作 | 计算 |
|------|------|------|
| 1 | 技能匹配度评估 | 历史velocity + 技术栈匹配 |
| 2 | 任务分配 | 高优先级→核心人员 |
| 3 | 负载率计算 | 分配工时 / 可用工时 |
| 4 | 缺口识别 | 负载率>70%或未分配任务 |

### Phase 4: 排期设计（15分钟）

| 步骤 | 动作 | 输出 |
|------|------|------|
| 1 | 任务依赖分析 | 前置任务ID列表 |
| 2 | 关键路径识别 | 最长依赖链 |
| 3 | Sprint划分 | 按Sprint周期分组任务 |
| 4 | 里程碑定义 | M1/M2/M3/M4节点 |

### Phase 5: 文档生成（10分钟）

| 输出格式 | 生成内容 | 文件类型 |
|---------|---------|---------|
| excel | 任务清单、工时汇总、里程碑、风险清单 | .xlsx |
| mermaid | 整体进度甘特图、人员泳道甘特图 | .md内嵌 |
| msproject | CSV导入文件 | .csv |
| markdown | 排期文档汇总 | .md |

## 输出产物

### Excel文件结构

| Sheet名称 | 内容 |
|----------|------|
| 任务清单 | 任务ID、名称、开始日期、结束日期、工期、负责人、优先级、前置任务、备注、状态 |
| 工时汇总 | 开发人员、高/中/低优先级工时、总工时、可用工时、负荷率 |
| 里程碑 | 里程碑名称、日期、验收标准、状态 |
| 风险清单 | 风险项、影响等级、应对措施、责任人、状态 |

### Mermaid甘特图模板

```mermaid
gantt
    title [项目名称]项目进度
    dateFormat YYYY-MM-DD

    section Sprint1
    [任务名称]           :[task_id], [开始日期], [工期]
    ...

    section 里程碑
    M1-[里程碑名]        :milestone, m1, [日期], 0d
```

### MS Project CSV格式

| 字段 | 格式 | 示例 |
|------|------|------|
| Task ID | 数字 | 1 |
| Task Name | 文本 | FZ1002-批量受理功能 |
| Start Date | YYYY-MM-DD | 2026-03-27 |
| Finish Date | YYYY-MM-DD | 2026-04-03 |
| Duration | Xd | 8d |
| Resource Names | 姓名 | 蔡尔柘 |
| Priority | 高/中/低 | 高 |
| Predecessors | ID;ID | 3;4 |

## 质量检查清单

### Before 排期设计
- [ ] 功能点清单完整
- [ ] 优先级明确
- [ ] 开发人员可用工时确认

### After 排期输出
- [ ] 高优先级全部安排
- [ ] 关键路径人员稳定
- [ ] 无单人阻塞多任务
- [ ] 缺口已识别并标注
- [ ] 里程碑验收标准明确
- [ ] 风险清单完整

## 注意事项

1. **关键路径**: 识别最长依赖链，安排核心人员
2. **缓冲设计**: 总工时预留20-30%缓冲
3. **缺口预警**: 未分配任务或负载>70%需要标注
4. **里程碑固定**: 发版日期不可延期，低优先级可延期
5. **格式选择**: 建议同时输出Excel+Mermaid

## 示例输出

**输入**: 17个功能点，2名开发，发版日期2026-04-24

**输出摘要**:
- 工时汇总: 陈上康8.5天(42.5%负荷), 蔡尔柘12.5天(62.5%负荷)
- 缺口: FZ1072、FZ1004共6天需协调
- 关键路径: FZ1072(扬言提醒)→测试→修复→发版
- 风险: 人员缺口(高), 发版日期固定(高)
