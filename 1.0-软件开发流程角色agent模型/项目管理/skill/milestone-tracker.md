## 触发条件

| 触发方式 | 条件 |
|---------|------|
| 关键词 | "里程碑"、"验收"、"偏差"、"进度"、"燃尽" |
| 命令 | `/milestone` |
| 事件 | 到达里程碑日期、里程碑验收请求 |

## 输入参数

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|-------|------|------|--------|------|
| milestone_list | array | true | - | 里程碑列表（名称、日期、验收标准） |
| current_tasks | array | true | - | 当前任务状态列表 |
| actual_hours | array | false | - | 实际工时数据（任务ID、实际工时） |

## 执行流程

### Phase 1: 里程碑状态检查

| 步骤 | 动作 | 判断条件 |
|------|------|---------|
| 1 | 获取里程碑列表 | 按日期排序 |
| 2 | 检查当前里程碑 | 当前日期 >= 里程碑日期 |
| 3 | 验收标准检查 | 关联任务状态是否全部完成 |

### Phase 2: 任务完成度统计

| 指标 | 计算 | 公式 |
|------|------|------|
| 完成率 | 已完成数/总数 | percentage |
| 高优先级完成率 | 高优已完成/高优总数 | priority_complete_rate |
| 燃尽进度 | 剩余工时/总工时 | burn_down_rate |

### Phase 3: 偏差分析

| 偏差类型 | 计算 | 预警阈值 |
|---------|------|---------|
| 工时偏差 | (实际-预估)/预估 | >30%预警, >50%严重 |
| 进度偏差 | (计划完成-实际完成)/计划 | >20%预警 |
| 里程碑偏差 | 实际日期-计划日期 | >0天预警 |

### Phase 4: 预警生成

| 预警等级 | 条件 | 动作 |
|---------|------|------|
| 红色 | 偏差>50%或里程碑延期 | 立即通知, 启动应对 |
| 黄色 | 偏差30-50% | 预警通知, 分析原因 |
| 绿色 | 偏差<30% | 正常, 继续跟进 |

## 质量检查清单

### Before 里程碑检查
- [ ] 任务状态更新完整
- [ ] 实际工时数据收集

### After 里程碑验收
- [ ] 验收标准全部检查
- [ ] 偏差原因记录
- [ ] 风险预警更新
- [ ] 下一步计划确认

## 注意事项

1. **偏差阈值**: >30%预警，>50%严重，需复盘分析
2. **里程碑延期**: 延期需要更新排期并通知干系人
3. **阻塞问题**: 立会识别的阻塞需要当日协调解决
4. **燃尽趋势**: 连续3天燃尽停滞需要预警
