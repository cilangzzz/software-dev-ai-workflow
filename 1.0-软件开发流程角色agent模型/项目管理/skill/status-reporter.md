## 触发条件

| 触发方式 | 条件 |
|---------|------|
| 关键词 | "状态报告"、"周报"、"进度汇报"、"项目报告" |
| 命令 | `/status` |
| 事件 | 周期性触发（每周）、里程碑验收后 |

## 输入参数

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|-------|------|------|--------|------|
| report_type | string | false | weekly | 报告类型：daily/weekly/milestone/release |
| project_info | object | true | - | 项目基本信息 |
| task_status | array | true | - | 任务状态列表 |
| milestone_status | array | false | - | 里程碑状态 |
| risk_status | array | false | - | 风险状态 |

## 报告类型与频率

| 类型 | 频率 | 内容范围 | 目标读者 |
|------|------|---------|---------|
| daily | 每日 | 立会摘要、阻塞问题 | 项目组内部 |
| weekly | 每周 | 进度统计、偏差分析、风险更新 | 项目组+干系人 |
| milestone | 里程碑节点 | 验收报告、偏差分析、经验总结 | 项目组+管理层 |
| release | 发版节点 | 发版总结、功能清单、质量数据 | 全员+客户 |

## 执行流程

### Phase 1: 数据收集

| 数据项 | 来源 | 统计方式 |
|-------|------|---------|
| 任务完成数 | 任务状态 | 已完成数量统计 |
| 任务进行数 | 任务状态 | 进行中数量统计 |
| 工时消耗 | 工时记录 | 实际工时累加 |
| 偏差数据 | 对比分析 | 预估vs实际 |

### Phase 2: 进度计算

| 指标 | 公式 | 说明 |
|------|------|------|
| 整体完成率 | 已完成数/总数 | 百分比 |
| 高优先级完成率 | 高优已完成/高优总数 | 关键指标 |
| 工时消耗率 | 实际工时/总预估 | 燃尽进度 |
| 偏差率 | (实际-预估)/预估 | 预警指标 |

### Phase 3: 报告生成

## 质量检查清单

### Before 报告生成
- [ ] 任务状态更新完整
- [ ] 工时数据收集
- [ ] 风险状态更新

### After 报告输出
- [ ] 数据准确性核实
- [ ] 预警内容完整
- [ ] 协调事项明确

## 注意事项

1. **数据准确**: 报告数据必须与实际一致
2. **预警突出**: 风险和阻塞需要明显标注
3. **简洁明了**: 报告控制在2页内，重点突出
4. **及时发送**: 周报固定时间发送，如周五下午
