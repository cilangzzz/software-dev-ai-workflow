## 触发条件

| 触发方式 | 条件 |
|---------|------|
| 关键词 | "变更"、"需求变更"、"调整排期"、"增加功能"、"删减功能" |
| 命令 | `/change` |
| 事件 | 收到变更请求 |

## 输入参数

| 参数名 | 类型 | 必填 | 默认值 | 描述 |
|-------|------|------|--------|------|
| change_request | object | true | - | 变更请求（类型、内容、提出人、日期） |
| current_schedule | object | true | - | 当前排期数据 |
| change_type | string | true | - | 变更类型：add/remove/modify/priority_change |

## 变更类型分类

| 类型 | 描述 | 影响范围 |
|------|------|---------|
| add | 新增功能 | 增加工时，可能延期 |
| remove | 删除功能 | 减少工时，可能提前 |
| modify | 修改功能 | 工时变化，依赖调整 |
| priority_change | 优先级调整 | 排期顺序变化 |

## 执行流程

### Phase 1: 变更登记

```markdown
## 变更请求登记表

| 项目 | 内容 |
|------|------|
| 变更编号 | CR-[YYYYMMDD]-[序号] |
| 变更类型 | [add/remove/modify/priority_change] |
| 变换内容 | [详细描述] |
| 提出人 | [姓名] |
| 提出日期 | [YYYY-MM-DD] |
| 紧急程度 | [紧急/一般] |
```

### Phase 2: 影响分析

| 分析维度 | 评估内容 | 输出 |
|---------|---------|------|
| 工时影响 | 新增/减少工时 | 工时变化值 |
| 进度影响 | 是否延期/提前 | 延期天数 |
| 依赖影响 | 上下游任务调整 | 受影响任务列表 |
| 资源影响 | 人员分配调整 | 人员变化 |
| 风险影响 | 新增风险/风险消除 | 风险变化 |

### Phase 4: 排期调整方案

| 场景 | 调整策略 |
|------|---------|
| 延期风险 | 低优先级功能延期至下批次 |
| 工时增加 | 协调额外人员或外包 |
| 优先级提升 | 调整任务顺序，高优先级优先 |
| 功能删除 | 释放工时，补充低优先级功能 |

### Phase 5: 审批流程

```markdown
## 变更审批流程

### 审批节点
| 节点 | 审批人 | 审批内容 |
|------|--------|---------|
| 技术评估 | 架构师 | 技术可行性 |
| 工时评估 | 项目经理 | 工时和进度影响 |
| 业务评估 | 产品经理 | 业务价值和优先级 |
| 最终审批 | 项目负责人 | 是否批准变更 |

### 审批状态
| 状态 | 说明 |
|------|------|
| 待评估 | 变更登记完成，等待评估 |
| 待审批 | 评估完成，等待审批 |
| 已批准 | 审批通过，执行调整 |
| 已拒绝 | 审批拒绝，变更关闭 |
```

## 注意事项

1. **变更登记**: 所有变更必须登记，有编号追踪
2. **影响分析**: 工时、进度、依赖、风险全面评估
3. **审批流程**: 必须经过审批才能执行
4. **排期更新**: 批准后立即更新排期文档
5. **通知干系人**: 变更批准后通知相关人员
