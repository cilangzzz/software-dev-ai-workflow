## 执行流程

### Phase 1: 风险识别

| 风险类型 | 识别方法 | 预警信号 |
|---------|---------|---------|
| 人员缺口 | 未分配任务、负载>70% | 任务无负责人 |
| 技术复杂度 | 高复杂度任务数量 | 多个高复杂度任务 |
| 依赖阻塞 | 前置任务未完成 | 下游任务等待 |
| 发版固定 | 发版日期约束 | 无延期空间 |
| 资源冲突 | 多人依赖同一资源 | 关键路径单人阻塞 |

### Phase 2: 风险评估

| 评估维度 | 评分标准 | 分值 |
|---------|---------|------|
| 影响程度 | 项目延期/功能缺失/质量下降 | 高/中/低 |
| 发生概率 | 确定/可能/不确定 | 高/中/低 |
| 综合等级 | 影响×概率 | 高/中/低 |

**风险矩阵**:
| 影响\概率 | 高 | 中 | 低 |
|----------|-----|-----|-----|
| 高 | **高** | **高** | 中 |
| 中 | **高** | 中 | 低 |
| 低 | 中 | 低 | 低 |

### Phase 3: 应对措施设计

| 风险类型 | 应对策略 | 具体措施 |
|---------|---------|---------|
| 人员缺口 | 协调/外包/延期 | Sprint前协调人员，低优先级延期 |
| 技术复杂度 | Review/缓冲 | 提前设计方案Review，增加缓冲 |
| 依赖阻塞 | 协调/并行 | 协调上游优先完成，下游并行准备 |
| 发版固定 | 缓冲/延期功能 | 低优先级作为缓冲，延期至下批次 |
| 资源冲突 | 时间错开/共享 | 调整排期错开时间，建立共享机制 |

## 风险识别检查清单

### 人员风险
- [ ] 是否有任务未分配人员？
- [ ] 是否有人员负载超过70%？
- [ ] 是否有核心人员承担关键路径？
- [ ] 是否有人员技能不匹配？

### 技术风险
- [ ] 是否有多个高复杂度任务？
- [ ] 是否有新技术引入？
- [ ] 是否有并发控制需求？
- [ ] 是否有第三方集成？

### 依赖风险
- [ ] 是否有外部依赖（API、第三方）？
- [ ] 是否有跨团队依赖？
- [ ] 是否有资源竞争依赖？
- [ ] 是否有数据依赖？

### 时间风险
- [ ] 发版日期是否固定？
- [ ] 是否有缓冲空间？
- [ ] 是否有并行任务？

## 预警机制

| 预警等级 | 触发条件 | 响应时间 | 响应动作 |
|---------|---------|---------|---------|
| 红色 | 高风险发生/里程碑延期 | 立即 | 通知干系人，启动应对 |
| 黄色 | 中风险发生/偏差预警 | 24小时内 | 分析原因，制定方案 |
| 绿色 | 低风险发生 | 每周立会 | 监控状态 |

## 注意事项

1. **高风险优先**: 高风险必须立即处理，有明确应对方案
2. **持续监控**: 风险清单每周更新，状态变化及时记录
3. **复盘总结**: 已解决风险记录经验教训，用于后续项目
4. **预警联动**: 风险预警与里程碑跟踪联动
