# 里程碑验收模板

## 一、里程碑定义模板

| 项目 | 内容 | 填写说明 |
|------|------|---------|
| 里程碑编号 | M[N] | 如M1、M2、M3 |
| 里程碑名称 | | 如"高优先级开发完成" |
| 计划日期 | YYYY-MM-DD | 计划达成日期 |
| 关联任务 | 任务ID列表 | 该里程碑关联的任务 |
| 验收标准 | | 具体可检验的标准 |
| 负责人 | | 里程碑负责人 |

---

## 二、里程碑验收标准示例

| 里程碑 | 典型验收标准 |
|--------|-------------|
| M1-需求确认 | 需求文档评审通过，功能清单确认 |
| M2-架构设计完成 | 技术方案评审通过，数据库设计完成 |
| M3-高优先级开发完成 | 高优先级功能代码完成，单元测试通过 |
| M4-高优先级测试通过 | 无P0/P1级Bug，测试报告确认 |
| M5-全功能开发完成 | 所有功能代码完成，代码Review通过 |
| M6-回归测试完成 | 全功能回归测试通过，Bug修复完成 |
| M7-发版上线 | 生产环境部署完成，验证通过 |

---

## 三、里程碑验收检查清单

### Before 验收
- [ ] 关联任务状态确认
- [ ] 验收标准准备
- [ ] 验收材料准备（代码、文档、测试报告）

### During 验收
- [ ] 检查任务完成状态
- [ ] 检查代码质量（Review通过）
- [ ] 检查测试结果（Bug状态）
- [ ] 检查文档产出

### After 验收
- [ ] 验收结论确认（通过/延期/阻塞）
- [ ] 偏差记录（如有延期）
- [ ] 风险更新（如有阻塞）
- [ ] 下一里程碑准备

### 状态定义
| 状态 | 定义 | 颜色 |

## 使用说明

1. **定义里程碑**: 使用模板一、二定义里程碑节点
2. **验收检查**: 使用模板三、四进行验收
3. **进度跟踪**: 使用模板五进行日常跟踪
4. **延期处理**: 使用模板六处理延期情况
