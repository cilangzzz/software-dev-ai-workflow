# 风险识别检查清单

## 一、人员风险检查

### 检查项
- [ ] 是否有任务未分配人员？
- [ ] 是否有人员负载超过70%？
- [ ] 是否有人员负载超过90%？
- [ ] 是否有核心人员承担关键路径任务？
- [ ] 是否有人员技能与技术栈不匹配？
- [ ] 是否有人员请假/培训计划？
- [ ] 是否有新入职人员需要磨合期？

### 风险等级判断
| 检查结果 | 风险等级 |
|---------|---------|
| 5项以上存在问题 | 高 |
| 3-4项存在问题 | 中 |
| 1-2项存在问题 | 低 |

---

## 二、技术风险检查

### 检查项
- [ ] 是否有多个高复杂度任务（>3个）？
- [ ] 是否有新技术引入（团队未使用过）？
- [ ] 是否有并发控制需求？
- [ ] 是否有分布式事务需求？
- [ ] 是否有第三方系统集成？
- [ ] 是否有大量数据迁移？
- [ ] 是否有性能要求（高并发/大数据量）？
- [ ] 是否有安全合规要求？

### 风险等级判断
| 检查结果 | 风险等级 |
|---------|---------|
| 6项以上存在问题 | 高 |
| 4-5项存在问题 | 中 |
| 1-3项存在问题 | 低 |

---

## 三、依赖风险检查

### 检查项
- [ ] 是否有外部API依赖？
- [ ] 是否有第三方服务依赖？
- [ ] 是否有跨团队依赖？
- [ ] 是否有跨模块依赖？
- [ ] 是否有资源竞争依赖（同一人员/同一资源）？
- [ ] 是否有数据依赖（等待数据准备）？
- [ ] 是否有环境依赖（等待环境搭建）？

### 风险等级判断
| 检查结果 | 风险等级 |
|---------|---------|
| 5项以上存在问题 | 高 |
| 3-4项存在问题 | 中 |
| 1-2项存在问题 | 低 |

---

## 四、时间风险检查

### 检查项
- [ ] 发版日期是否固定不可延期？
- [ ] 是否有外部时间约束（客户/监管）？
- [ ] 是否有节假日影响排期？
- [ ] 是否有缓冲时间不足（<20%）？
- [ ] 是否有并行任务过多？
- [ ] 是否有长周期任务（>Sprint周期）？

### 风险等级判断
| 检查结果 | 风险等级 |
|---------|---------|
| 4项以上存在问题 | 高 |
| 2-3项存在问题 | 中 |
| 1项存在问题 | 低 |

---

## 七、风险应对策略库

| 风险类型 | 应对策略 | 具体措施 |
|---------|---------|---------|
| 人员缺口 | 协调/外包/延期 | 1. 申请临时人员 2. 外包开发 3. 低优功能延期 |
| 技术复杂度 | Review/缓冲/培训 | 1. 技术方案提前Review 2. 增加缓冲时间 3. 技术培训 |
| 依赖阻塞 | 协调/并行/替代 | 1. 协调上游优先 2. 下游并行准备 3. 寻找替代方案 |
| 时间约束 | 缓冲/延期功能 | 1. 低优先级作为缓冲 2. 非核心功能延期下批次 |
| 资源冲突 | 错峰/共享 | 1. 调整排期错峰使用 2. 建立共享预约机制 |

---

## 使用说明

1. **排期设计时**: 全面检查五类风险
2. **每周立会时**: 检查新增风险项
3. **里程碑验收时**: 检查风险解决状态
4. **发版前**: 检查所有高风险项是否解决
