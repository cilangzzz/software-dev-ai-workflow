# 工时估算模板

## 一、复杂度评估标准

| 复杂度等级 | 基础工时范围 | 识别特征 | 缓冲系数 |
|-----------|-------------|---------|---------|
| **高** | 3-5天 | 新增数据库表、并发控制机制、权限体系设计、第三方系统集成、短信/推送模块 | 1.5 |
| **中** | 1.5-3天 | 流程调整、字段扩展、组件复用、权限控制 | 1.3 |
| **低** | 0.5-1.5天 | UI调整、校验逻辑、Bug修复、CSS调整、默认值修改 | 1.2 |

---

## 二、三点估算法公式

```
预估工时 = (乐观时间 + 4×最可能时间 + 悠观时间) / 6
实际排期 = 预估工时 × 缓冲系数
```

### 示例计算

| 场景 | 乐观 | 最可能 | 悠观 | 预估工时 | 缓冲系数 | 最终排期 |
|------|------|--------|------|---------|---------|---------|
| 高复杂度功能 | 2天 | 4天 | 8天 | (2+16+8)/6=4.3天 | 1.5 | 6.5天 |
| 中复杂度功能 | 1天 | 2天 | 4天 | (1+8+4)/6=2.2天 | 1.3 | 2.8天 |
| 低复杂度功能 | 0.5天 | 1天 | 2天 | (0.5+4+2)/6=1.1天 | 1.2 | 1.3天 |

## 四、人员可用工时计算

| 计算项 | 公式 | 示例 |
|-------|------|------|
| Sprint周期 | 14天（默认） | 2周迭代 |
| 有效工作日 | Sprint周期 - 休假 - 培训 | 14-2=12天 |
| 可用工时 | 有效工作日 × 0.8 | 12×0.8=9.6天 |
| 0.8系数说明 | 扣除会议、沟通、Review时间 | 约20%非开发时间 |

## 五、负载率计算标准

| 负载率范围 | 状态 | 说明 |
|-----------|------|------|
| <50% | 充裕 | 可增加任务 |
| 50-70% | 正常 | 正常工作状态 |
| 70-90% | 预警 | 避免增加新任务 |
| >90% | 严重 | 必须调整或协调 |

---

## 六、工时偏差分析模板

| 偏差率范围 | 状态 | 处理方式 |
|-----------|------|---------|
| <30% | 正常 | 不需要特别处理 |
| 30-50% | 预警 | 记录原因，持续关注 |
| >50% | 严重 | 复盘会议，分析改进 |

### 偏差原因分类

| 原因类型 | 占比评估 | 典型案例 |
|---------|---------|---------|
| 需求变更 | | 功能范围增加 |
| 技术难度 | | 未预料的复杂实现 |
| 依赖阻塞 | | 等待第三方接口 |
| 环境问题 | | 开发环境不稳定 |
| 个人效率 | | 学习新技术时间 |

---

## 使用说明

1. **评估复杂度**: 先根据功能特征判断复杂度等级
2. **三点估算**: 使用公式计算预估工时
3. **应用缓冲**: 根据复杂度应用缓冲系数
4. **分配人员**: 计算负载率，识别缺口
5. **偏差监控**: 实际完成后记录偏差，分析原因
