# 会议模板

## 一、每日立会模板

### 立会记录

```markdown
### 立会记录 - [日期]

**参与人员**: [人员列表]
**会议时长**: 15分钟

**每人汇报**（每人2分钟）:
| 人员 | 昨天完成 | 今天计划 | 阻塞问题 |
|------|---------|---------|---------|
| [姓名] | [任务ID+状态] | [任务ID+预计完成] | [问题描述，无则填"无"] |

**阻塞汇总**:
- [阻塞问题1] - 需协调人: [姓名]
- [阻塞问题2] - 需协调人: [姓名]

**今日行动**:
- [行动项1] - 负责人: [姓名]
- [行动项2] - 负责人: [姓名]

**风险预警**:
- [预警内容]
```

### 立会规则
1. **时间限制**: 严格15分钟，每人2分钟
2. **汇报内容**: 只讲三件事（昨天完成、今天计划、阻塞问题）
3. **阻塞处理**: 识别阻塞后，会后单独协调解决
4. **不深入讨论**: 立会只同步状态，不讨论技术细节

## 二、需求评审会议模板

### 会议记录

```markdown
### 需求评审会议 - [需求名称]

**会议信息**:
| 项目 | 内容 |
|------|------|
| 会议主题 | [需求名称]评审 |
| 会议时间 | [日期+时间] |
| 参与人员 | 产品、研发、测试、设计 |
| 会议时长 | [实际时长] |

**需求概要**:
- 需求编码: [编码]
- 需求类型: [新增/优化/修复]
- 优先级: [高/中/低]
- 预估工时: [工时]

**评审要点**:
| 评审项 | 结论 | 备注 |
|--------|------|------|
| 需求清晰度 | 通过/待补充 | [说明] |
| 技术可行性 | 通过/有风险 | [说明] |
| 工时评估 | 确认/调整 | [说明] |
| 测试方案 | 确认/待设计 | [说明] |

**评审结论**:
- 状态: [通过/待补充/拒绝]
- 待补充内容: [如有]

**待办事项**:
| 待办 | 负责人 | 完成时间 |
|------|--------|---------|
| [事项] | [姓名] | [日期] |
```

## 五、变更评审会议模板

### 会议记录

```markdown
### 变更评审会议 - CR-[编号]

**会议信息**:
| 项目 | 内容 |
|------|------|
| 变换编号 | CR-[编号] |
| 会议时间 | [日期+时间] |
| 参与人员 | 产品、研发、项目经理 |
| 会议时长 | [实际时长] |

**变更内容**:
| 项目 | 内容 |
|------|------|
| 变换类型 | [新增/删除/修改] |
| 变换内容 | [详细描述] |
| 提出人 | [姓名] |

**影响评估**:
| 维度 | 影响 | 说明 |
|------|------|------|
| 工时影响 | +/-X天 | [说明] |
| 进度影响 | +/-X天 | [说明] |
| 依赖影响 | [任务列表] | [说明] |

**评审结论**:
- 状态: [批准/拒绝/待确认]
- 条件: [如有条件]
- 审批人: [姓名]

**执行计划**（如批准）:
| 步骤 | 负责人 | 完成时间 |
|------|--------|---------|
```

---

## 使用说明

1. **立会**: 每日固定时间，严格15分钟
2. **评审会**: 需求确认后召开，产出评审结论
3. **验收会**: 里程碑节点召开，产出验收报告
4. **周报会**: 每周固定时间，汇报进度
5. **变更会**: 收到变更请求后召开，产出审批结论
