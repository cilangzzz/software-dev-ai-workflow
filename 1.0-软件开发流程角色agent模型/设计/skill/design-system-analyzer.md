# Skill: design-system-analyzer

## 基本信息
- **名称**: design-system-analyzer
- **版本**: 1.0.0
- **所属部门**: 设计部
- **优先级**: P0

## 功能描述
分析设计系统的一致性和完整性。检测设计Token（颜色、字体、间距等）的使用一致性，发现设计偏差和问题。

## 触发条件
- 命令触发: `/design-system-analyzer`
- 自然语言触发:
  - "分析设计系统"
  - "检查设计一致性"
  - "设计Token检查"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| design_source | string | 是 | 设计文件路径或导出数据 |
| check_types | string | 否 | 检查类型：color / typography / spacing / all |

## 执行流程
1. **Token提取** - 从设计源提取Token定义
2. **使用分析** - 分析Token使用情况
3. **一致性检查** - 检查设计一致性
4. **偏差检测** - 检测偏离设计规范的元素
5. **报告生成** - 生成分析报告和优化建议

## 使用示例

### 示例：颜色一致性检查
**输入**:
```
design_source: design/tokens.json
check_types: color
```

**输出**:
```markdown
# 设计系统分析报告

## 概述
- 分析范围: 全部页面
- 设计Token数: 45
- 一致性评分: 82/100

## Token使用分析
### 颜色Token
| Token名称 | 使用次数 | 占比 | 状态 |
|-----------|----------|------|------|
| primary-500 | 156 | 32% | ✅ |
| gray-100 | 89 | 18% | ✅ |
| #3B82F6 | 23 | 5% | ⚠️ 未在规范中 |
| #FF5733 | 8 | 2% | ⚠️ 未在规范中 |

## 一致性问题
### 🔴 高优先级
| 问题 | 位置 | 影响 | 建议 |
|------|------|------|------|
| 使用了未定义颜色 #3B82F6 | 按钮组件 | 品牌色不一致 | 改用 primary-500 |
| 主色值与Token不同 | 导航栏 | 视觉不统一 | 使用 primary-600 |

### 🟡 中优先级
| 问题 | 位置 | 建议 |
|------|------|------|
| 间距值非标准倍数 | 卡片组件 | 使用 spacing-4 (16px) |

## 设计建议
- 建立颜色使用规范文档
- 在设计工具中锁定非规范颜色
- 定期进行设计审计
```

## 质量标准
- Token识别准确率 ≥ 95%
- 一致性问题检出率 ≥ 90%
- 建议可操作性 100%

## 依赖工具
- Read - 读取设计Token文件
- Write - 输出分析报告
- Grep - 搜索Token使用

## 注意事项
- 分析结果需要设计师确认
- 部分偏差可能是有意设计
- 建议结合实际业务场景判断
