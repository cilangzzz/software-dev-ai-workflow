# Skill: project-status

## 基本信息
- **名称**: project-status
- **版本**: 1.0.0
- **所属部门**: 项目管理
- **优先级**: P0

## 功能描述
生成项目状态报告，包含进度、风险、问题等关键信息。支持自动收集项目数据并生成可视化报告。

## 触发条件
- 命令触发: `/project-status`
- 自然语言触发:
  - "项目状态"
  - "生成进度报告"
  - "项目周报"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| project_path | string | 否 | 项目路径，默认当前目录 |
| report_type | string | 否 | 报告类型：daily / weekly / milestone |
| format | string | 否 | 输出格式：markdown / html |

## 执行流程
1. **数据收集** - 收集任务、代码、文档状态
2. **进度计算** - 计算完成率和偏差
3. **风险识别** - 识别项目风险和阻塞项
4. **报告生成** - 生成结构化状态报告

## 输出格式

| 指标 | 计划 | 实际 | 偏差 |
|------|------|------|------|

## 使用示例

### 示例：周报生成
**输入**:
```
report_type: weekly
```

**输出**:
```markdown
# 项目状态周报

## 概览
- 项目名称: 电商平台2.0
- 报告时间: 2024-01-15 (第3周)
- 整体状态: 🟡 关注

## 进度概览
| 指标 | 计划 | 实际 | 偏差 |
|------|------|------|------|
| 整体完成率 | 60% | 55% | -5% |
| 本周任务 | 15 | 12 | -3 |
| 遗留缺陷 | 5 | 8 | +3 |

## 里程碑状态
| 里程碑 | 计划日期 | 预计日期 | 状态 |
|--------|----------|----------|------|
| MVP版本 | 2024-01-20 | 2024-01-25 | ⚠️ 延期风险 |
| UAT测试 | 2024-02-01 | 2024-02-05 | ⚠️ 待评估 |
| 正式上线 | 2024-02-15 | 2024-02-15 | ✅ 按计划 |

## 风险和问题
### 🔴 高风险
| 风险 | 影响 | 缓解措施 | 负责人 |
|------|------|---------|--------|
| 支付接口对接延期 | 影响核心功能上线 | 增加人力支援，并行开发 | 张三 |

### 🟡 中风险
| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 测试环境不稳定 | 测试效率降低 | 协调运维优先修复 |

## 本周进展
- ✅ 用户模块开发完成
- ✅ 商品列表页开发完成
- ✅ 数据库设计评审通过
- 🔄 订单模块开发中 (70%)

## 下周计划
- [ ] 完成订单模块开发
- [ ] 支付接口对接
- [ ] 开始集成测试
- [ ] 准备UAT环境

## 需要决策
1. 支付渠道选择：微信支付 vs 支付宝优先
2. 是否需要增加测试人力
```

## 质量标准
- 数据准确性 ≥ 95%
- 报告完整性 100%
- 生成时间 < 30秒

## 依赖工具
- Read - 读取项目配置
- Bash - 收集Git统计
- Write - 输出报告

## 注意事项
- 需要项目有基本的管理数据
- 状态评估可能需要人工确认
- 敏感信息注意权限控制
