# 流程度量指标

## 概述

本文档定义流程效率、质量、合规性等度量指标和计算方法，用于评估流程执行效果和持续改进。

## 效率指标

### 1. 阶段周期时间

**定义**: 项目在某阶段从开始到结束的时间

**计算公式**: `阶段周期时间 = 阶段结束日期 - 阶段开始日期`

**基准值**:

| 阶段 | 基准周期 | 预警阈值 |
|------|---------|---------|
| 项目启动 | 1-2周 | >3周 |
| 需求分析 | 2-4周 | >6周 |
| 设计 | 2-3周 | >5周 |
| 开发 | 4-8周 | >12周 |
| 测试 | 2-4周 | >6周 |
| 部署 | 1周 | >2周 |

### 2. 流程等待时间

**定义**: 流程活动中等待他人响应或资源就绪的时间

**计算公式**: `等待时间 = 总周期时间 - 实际工作时间`

**基准值**: 等待时间占比 < 30%

### 3. 流程吞吐量

**定义**: 单位时间内完成的流程实例数量

**计算公式**: `吞吐量 = 完成的流程实例数 / 时间周期`

**示例**: 每月完成5个需求评审流程

### 4. 门控通过时间

**定义**: 从提交门控检查到通过检查的时间

**计算公式**: `门控通过时间 = 门控通过时间 - 门控提交时间`

**基准值**: < 2个工作日

## 质量指标

### 1. 缺陷密度

**定义**: 单位产出物中发现的缺陷数量

**计算公式**: `缺陷密度 = 缺陷总数 / 产出物规模`

**示例**:

| 产出物类型 | 度量单位 | 基准值 |
|-----------|---------|--------|
| 需求文档 | 缺陷/页 | < 0.5 |
| 代码 | 缺陷/KLOC | < 5 |
| 测试用例 | 缺陷/用例 | < 0.1 |

### 2. 返工率

**定义**: 需要返工的产出物比例

**计算公式**: `返工率 = 返工产出物数 / 总产出物数 × 100%`

**基准值**: < 15%

### 3. 一次通过率

**定义**: 第一次提交即通过的产出物比例

**计算公式**: `一次通过率 = 一次通过数 / 总提交数 × 100%`

**基准值**:

| 检查点 | 基准值 |
|--------|--------|
| 需求评审 | > 80% |
| 设计评审 | > 75% |
| 代码审查 | > 85% |
| 门控检查 | > 70% |

### 4. 缺陷修复周期

**定义**: 缺陷从发现到关闭的平均时间

**计算公式**: `缺陷修复周期 = Σ(缺陷关闭时间 - 缺陷发现时间) / 缺陷数`

**基准值**:

| 缺陷等级 | 基准值 |
|---------|--------|
| 严重 | < 4小时 |
| 高 | < 1天 |
| 中 | < 3天 |
| 低 | < 1周 |

## 合规指标

### 1. 流程遵从率

**定义**: 按流程规范执行的流程实例比例

**计算公式**: `遵从率 = 合规执行数 / 总执行数 × 100%`

**基准值**: > 95%

### 2. 文档完整率

**定义**: 产出物文档符合完整性要求的比例

**计算公式**: `完整率 = 完整文档数 / 应产出文档数 × 100%`

**基准值**: 100%

### 3. 审计通过率

**定义**: 流程审计中发现的不符合项比例

**计算公式**: `审计通过率 = (1 - 不符合项数 / 检查项数) × 100%`

**基准值**: > 90%

### 4. 变更合规率

**定义**: 按变更流程执行的变更比例

**计算公式**: `变更合规率 = 合规变更数 / 总变更数 × 100%`

**基准值**: 100%

## 指标采集方法

### 自动采集

| 指标 | 数据来源 | 采集方式 |
|------|---------|---------|
| 周期时间 | 项目管理系统 | 自动计算 |
| 门控通过时间 | gate-check Skill | 自动记录 |
| 缺陷密度 | 缺陷管理系统 | 自动统计 |
| 流程遵从率 | 工作流引擎 | 自动统计 |

### 人工采集

| 指标 | 数据来源 | 采集频率 |
|------|---------|---------|
| 等待时间 | 项目经理记录 | 每周 |
| 返工率 | 审查记录 | 每阶段 |
| 文档完整率 | 文档检查 | 每阶段 |
