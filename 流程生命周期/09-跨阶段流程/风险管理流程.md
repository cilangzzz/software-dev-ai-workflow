# 风险管理流程

## 流程概述

风险管理流程贯穿项目全生命周期，确保项目风险得到及时识别、评估、应对和监控。

## 风险管理生命周期

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  风险识别 │──▶│  风险评估 │──▶│  风险应对 │──▶│  风险监控 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘
     ▲                                                   │
     └───────────────────────────────────────────────────┘
                         持续迭代
```

## 各阶段风险关注点

| 阶段 | 典型风险 | 风险类别 |
|------|---------|---------|
| 项目启动 | 目标不明确、资源不足 | 战略风险 |
| 需求分析 | 需求蔓延、需求冲突 | 需求风险 |
| 设计 | 技术选型错误、架构缺陷 | 技术风险 |
| 开发 | 进度延期、人员流失 | 执行风险 |
| 测试 | 质量不达标、测试覆盖不足 | 质量风险 |
| 部署 | 部署失败、数据丢失 | 运维风险 |
| 运维 | 系统故障、安全漏洞 | 运维风险 |

## 风险评估矩阵

### 概率等级
| 等级 | 描述 | 分值 |
|------|------|------|
| 很低 | 发生概率<10% | 1 |
| 低 | 发生概率10%-30% | 2 |
| 中 | 发生概率30%-50% | 3 |
| 高 | 发生概率50%-70% | 4 |
| 很高 | 发生概率>70% | 5 |

### 影响等级
| 等级 | 描述 | 分值 |
|------|------|------|
| 很低 | 影响可忽略 | 1 |
| 低 | 轻微影响 | 2 |
| 中 | 中等影响 | 3 |
| 高 | 严重影响 | 4 |
| 很高 | 致命影响 | 5 |

### 风险等级判定
| 风险分数 | 等级 | 处理策略 |
|---------|------|---------|
| 1-6 | 低 | 接受监控 |
| 7-12 | 中 | 制定应对计划 |
| 13-19 | 高 | 优先处理 |
| 20-25 | 极高 | 立即升级处理 |

## 风险应对策略

| 策略 | 描述 | 适用场景 |
|------|------|---------|
| 规避 | 改变计划消除风险 | 高影响风险 |
| 转移 | 将风险转嫁给第三方 | 合同风险 |
| 缓解 | 降低风险概率或影响 | 技术风险 |
| 接受 | 接受风险并制定应急预案 | 低影响风险 |

## 风险监控机制

### 定期检查
- 周报中更新风险状态
- 里程碑评审时复盘风险
- 季度进行风险审计

### 预警触发
- 风险状态变为"发生中"
- 新增高风险项
- 风险应对措施失效
