# 质量门控流程

## 流程概述

质量门控是项目各阶段之间的质量检查点，确保每个阶段的产出物符合质量标准后才进入下一阶段。

## 各阶段门控标准

### Gate 0: 启动就绪
| 检查项 | 标准 | 检查方式 |
|--------|------|---------|
| 项目章程 | 已签署 | 人工检查 |
| 团队组建 | 核心成员到位 | 人工检查 |
| 环境准备 | 工具已配置 | 自动检查 |

### Gate 1: 需求就绪
| 检查项 | 标准 | 检查方式 |
|--------|------|---------|
| PRD完整性 | 核心章节完整 | AI辅助检查 |
| 用户故事 | INVEST标准 | AI辅助检查 |
| 验收标准 | Gherkin格式 | AI辅助检查 |
| 需求评审 | 已通过 | 人工检查 |

### Gate 2: 设计就绪
| 检查项 | 标准 | 检查方式 |
|--------|------|---------|
| 架构设计 | 已评审通过 | 人工检查 |
| 技术选型 | ADR已记录 | AI辅助检查 |
| 数据库设计 | 已评审通过 | 人工检查 |
| API设计 | 文档完整 | AI辅助检查 |

### Gate 3: 开发就绪
| 检查项 | 标准 | 检查方式 |
|--------|------|---------|
| 代码审查 | Review通过率100% | 自动检查 |
| 单元测试 | 覆盖率≥80% | 自动检查 |
| 代码规范 | 无严重违规 | 自动检查 |
| 安全扫描 | 无高危漏洞 | 自动检查 |

### Gate 4: 测试就绪
| 检查项 | 标准 | 检查方式 |
|--------|------|---------|
| 功能测试 | 通过率100% | 自动检查 |
| 性能测试 | 指标达标 | 自动检查 |
| 缺陷修复 | 严重缺陷为0 | 自动检查 |
| UAT验收 | 已通过 | 人工检查 |

### Gate 5: 发布就绪
| 检查项 | 标准 | 检查方式 |
|--------|------|---------|
| 部署方案 | 已评审通过 | 人工检查 |
| 回滚预案 | 已演练验证 | 人工检查 |
| 监控配置 | 已配置完成 | 自动检查 |
| 文档更新 | 已完成 | AI辅助检查 |

## 门控执行流程

1. **触发门控检查**
   - 阶段负责人提交门控申请
   - 或自动化触发（如代码合并）

2. **执行检查项**
   - 自动检查项由CI/CD执行
   - AI辅助检查项由gate-check Skill执行
   - 人工检查项由相关人员确认

3. **评估结果**
   - 统计通过率
   - 识别阻塞问题
   - 生成门控报告

4. **决策处理**
   - 通过：进入下一阶段
   - 有条件通过：设定补救期限
   - 不通过：返回当前阶段修复
