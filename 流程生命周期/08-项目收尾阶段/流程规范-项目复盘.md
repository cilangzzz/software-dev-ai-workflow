# 流程规范：项目复盘

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 项目复盘 |
| 所属阶段 | 项目收尾阶段 |
| 流程负责人 | 项目经理 |
| 前置流程 | 知识沉淀 |
| 后置流程 | 项目结束 |

## 流程目的

总结项目经验教训，为后续项目提供改进参考。

## 触发条件

- 知识沉淀完成
- 团队解散前

## 执行步骤

### Step 1: 复盘准备

- [ ] 收集项目数据
- [ ] 收集团队反馈
- [ ] 制定复盘议程

### Step 2: 复盘会议

- [ ] 回顾项目目标
- [ ] 评估目标达成情况
- [ ] 分析成功经验
- [ ] 分析失败教训
- [ ] 提出改进建议

### Step 3: 复盘报告

- [ ] 编写复盘报告
- [ ] 分享复盘成果
- [ ] 跟踪改进措施

## 复盘框架

| 维度 | 问题 |
|------|------|
| 目标达成 | 项目目标是否达成？偏差原因？ |
| 成功经验 | 项目中有哪些亮点？可以复用？ |
| 失败教训 | 项目中有哪些问题？如何避免？ |
| 改进建议 | 有哪些改进建议？如何落地？ |

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 复盘报告 | 文档 | 项目文档库 |
| 改进措施 | 文档 | 改进跟踪系统 |

## 质量标准

- 复盘参与全面
- 问题分析深入
- 改进措施可行
