# 流程规范：测试执行

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 测试执行 |
| 所属阶段 | 测试阶段 |
| 流程负责人 | 测试工程师 |
| 前置流程 | 测试规划 |
| 后置流程 | 测试报告 |

## 流程目的

按照测试计划和用例执行测试，发现和记录缺陷。

## 触发条件

- 测试计划已批准
- 测试环境已就绪
- 测试版本已发布

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 测试计划 | 测试规划 | 必须 |
| 测试用例 | 测试规划 | 必须 |
| 测试版本 | 开发阶段 | 必须 |

## 执行步骤

### Step 1: 冒烟测试

- [ ] 验证基本功能可用
- [ ] 验证核心流程通畅
- [ ] 决定是否继续测试

### Step 2: 功能测试

- [ ] 执行功能测试用例
- [ ] 记录测试结果
- [ ] 提交缺陷报告

### Step 3: 集成测试

- [ ] 测试模块间接口
- [ ] 测试系统集成
- [ ] 测试数据流转

### Step 4: 性能测试（如需要）

- [ ] 执行性能测试
- [ ] 分析性能指标
- [ ] 提出优化建议

### Step 5: 安全测试（如需要）

- [ ] 执行安全扫描
- [ ] 执行渗透测试
- [ ] 记录安全问题

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 测试执行记录 | 文档 | 测试管理系统 |
| 缺陷报告 | 文档 | 缺陷管理系统 |

## 质量标准

- 测试用例执行率 100%
- 严重缺陷修复率 100%
- 测试记录完整

## 相关Skill

| Skill | 用途 |
|-------|------|
| test-executor | 测试执行管理 |
| bug-analyzer | 缺陷分析 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 冒烟测试不通过 | 退回开发修复 |
| 严重缺陷过多 | 退回开发修复 |
