# 流程规范：测试规划

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 测试规划 |
| 所属阶段 | 测试阶段 |
| 流程负责人 | 测试负责人 |
| 前置流程 | 开发阶段完成 |
| 后置流程 | 测试执行 |

## 流程目的

制定全面的测试计划，明确测试范围、策略和资源安排。

## 触发条件

- 开发代码已提交
- 功能开发完成

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| PRD文档 | 需求分析 | 必须 |
| 设计文档 | 设计阶段 | 必须 |
| 发布版本 | 开发阶段 | 必须 |

## 执行步骤

### Step 1: 测试范围分析

- [ ] 分析功能需求
- [ ] 识别测试范围
- [ ] 确定测试重点

### Step 2: 测试策略制定

- [ ] 确定测试类型（功能/性能/安全）
- [ ] 确定测试方法（手工/自动化）
- [ ] 确定测试环境

### Step 3: 测试用例设计

- [ ] 编写功能测试用例
- [ ] 编写边界测试用例
- [ ] 编写异常测试用例

### Step 4: 资源安排

- [ ] 安排测试人员
- [ ] 准备测试环境
- [ ] 准备测试数据

### Step 5: 计划评审

- [ ] 组织测试计划评审
- [ ] 确认计划可行性
- [ ] 获取计划批准

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 测试计划 | 文档 | 项目文档库 |
| 测试用例 | 文档 | 测试管理系统 |

## 质量标准

- 测试范围覆盖全部需求
- 测试用例设计合理
- 资源安排充分

## 相关Skill

| Skill | 用途 |
|-------|------|
| test-case-generator | 测试用例生成 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 测试时间不足 | 调整测试优先级或申请延期 |
| 测试环境不可用 | 协调资源解决 |
