# 流程规范：开发协作

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 开发协作 |
| 所属阶段 | 开发阶段 |
| 流程负责人 | 技术负责人 |
| 前置流程 | - |
| 后置流程 | - |

## 流程目的

规范团队协作方式，确保信息同步和高效配合。

## 触发条件

- 项目开发过程中

## 协作机制

### 每日站会

| 项目 | 说明 |
|------|------|
| 时间 | 每日上午，15分钟内 |
| 参与人 | 开发团队 |
| 内容 | 昨天完成、今天计划、遇到问题 |

### 代码同步

| 项目 | 说明 |
|------|------|
| 频率 | 每天至少一次 |
| 方式 | 合并到开发分支 |
| 要求 | 不提交未完成代码 |

### 问题沟通

| 问题类型 | 沟通方式 |
|---------|---------|
| 技术问题 | 技术讨论群/面对面 |
| 需求问题 | 与产品沟通 |
| 进度风险 | 站会汇报 |

### 文档协作

| 文档类型 | 协作方式 |
|---------|---------|
| 技术文档 | Wiki协作编辑 |
| 接口文档 | 统一文档平台 |
| 代码注释 | 代码仓库管理 |

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 站会记录 | 文档 | 项目文档库 |
| 协作问题清单 | 表格 | 项目管理系统 |

## 质量标准

- 信息同步及时
- 问题响应快速
- 协作顺畅无阻塞
