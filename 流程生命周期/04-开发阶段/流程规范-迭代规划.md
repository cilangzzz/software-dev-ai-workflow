# 流程规范：迭代规划

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 迭代规划 |
| 所属阶段 | 开发阶段 |
| 流程负责人 | 技术负责人 |
| 前置流程 | 设计评审 |
| 后置流程 | 编码开发 |

## 流程目的

合理规划开发迭代，确保任务分配合理、进度可控。

## 触发条件

- 设计评审通过
- 开发准备就绪

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 设计文档 | 设计阶段 | 必须 |
| 用户故事 | 需求分析 | 必须 |
| 团队资源 | 项目启动 | 必须 |

## 执行步骤

### Step 1: 任务拆分

- [ ] 将需求拆分为开发任务
- [ ] 评估任务工作量
- [ ] 识别任务依赖关系

### Step 2: 迭代规划

- [ ] 确定迭代周期
- [ ] 分配迭代任务
- [ ] 设定迭代目标

### Step 3: 任务分配

- [ ] 根据能力分配任务
- [ ] 确认责任人
- [ ] 设定完成期限

### Step 4: 风险识别

- [ ] 识别技术风险
- [ ] 识别进度风险
- [ ] 制定应对措施

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 迭代计划 | 文档 | 项目管理系统 |
| 任务看板 | 看板 | 项目管理系统 |

## 质量标准

- 任务颗粒度适中（1-3天）
- 任务分配均衡
- 依赖关系明确

## 相关Skill

| Skill | 用途 |
|-------|------|
| project-status | 项目状态报告 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 工作量超预期 | 调整迭代范围或申请资源 |
| 关键人员变动 | 重新分配任务 |
