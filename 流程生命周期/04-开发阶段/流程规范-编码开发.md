# 流程规范：编码开发

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 编码开发 |
| 所属阶段 | 开发阶段 |
| 流程负责人 | 研发工程师 |
| 前置流程 | 迭代规划 |
| 后置流程 | 代码审查 |

## 流程目的

按照设计文档和编码规范实现功能代码。

## 触发条件

- 迭代任务已分配
- 开发环境已就绪

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 设计文档 | 设计阶段 | 必须 |
| 任务描述 | 迭代规划 | 必须 |
| 编码规范 | 项目规范 | 必须 |

## 执行步骤

### Step 1: 开发准备

- [ ] 拉取最新代码
- [ ] 创建开发分支
- [ ] 理解任务需求

### Step 2: 编码实现

- [ ] 编写业务代码
- [ ] 编写单元测试
- [ ] 本地验证通过

### Step 3: 代码提交

- [ ] 代码格式化
- [ ] 编写提交信息
- [ ] 推送代码

### Step 4: 合并请求

- [ ] 创建合并请求(MR/PR)
- [ ] 关联任务编号
- [ ] 添加变更说明

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 源代码 | 代码 | 代码仓库 |
| 单元测试 | 代码 | 代码仓库 |
| 合并请求 | 记录 | 代码仓库 |

## 质量标准

- 代码符合编码规范
- 单元测试覆盖率达标
- 无编译错误和警告

## 相关Skill

| Skill | 用途 |
|-------|------|
| implement | 功能实现辅助 |
| scaffold | 项目脚手架 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 需求理解偏差 | 与产品确认后调整 |
| 技术难题 | 寻求技术支持或技术预研 |
