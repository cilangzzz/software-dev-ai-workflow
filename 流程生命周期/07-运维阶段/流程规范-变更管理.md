# 流程规范：变更管理

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 变更管理 |
| 所属阶段 | 运维阶段 |
| 流程负责人 | 运维工程师 |
| 前置流程 | 变更申请 |
| 后置流程 | 变更验证 |

## 流程目的

规范线上变更流程，降低变更风险，确保变更可控。

## 触发条件

- 需要修改线上系统配置
- 需要更新线上版本

## 执行步骤

### Step 1: 变更申请

- [ ] 提交变更申请单
- [ ] 说明变更内容
- [ ] 说明变更原因
- [ ] 评估变更风险

### Step 2: 变更评审

- [ ] 变更评审会议
- [ ] 评估变更影响
- [ ] 确定变更时间
- [ ] 审批变更申请

### Step 3: 变更准备

- [ ] 编写变更方案
- [ ] 准备回滚方案
- [ ] 通知相关人员

### Step 4: 变更执行

- [ ] 按方案执行变更
- [ ] 记录变更过程
- [ ] 监控变更效果

### Step 5: 变更验证

- [ ] 验证变更结果
- [ ] 确认系统正常
- [ ] 关闭变更工单

## 变更类型

| 类型 | 说明 | 审批级别 |
|------|------|---------|
| 标准变更 | 低风险、常规变更 | 运维负责人审批 |
| 正常变更 | 一般风险变更 | 变更评审会审批 |
| 紧急变更 | 紧急修复变更 | 值班负责人审批，事后补评审 |

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 变更申请单 | 文档 | 变更管理系统 |
| 变更记录 | 日志 | 变更管理系统 |

## 质量标准

- 变更申请完整
- 变更过程有记录
- 变更结果有验证
