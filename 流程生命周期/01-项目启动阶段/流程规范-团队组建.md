# 流程规范：团队组建

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 团队组建 |
| 所属阶段 | 项目启动阶段 |
| 流程负责人 | 项目经理 |
| 前置流程 | 项目立项 |
| 后置流程 | 环境准备 |

## 流程目的

组建具备完成项目所需能力的团队，明确各角色职责和协作方式。

## 触发条件

- 项目章程已签署
- 资源预算已确认

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 项目章程 | 项目立项 | 必须 |
| 资源预算 | 财务 | 必须 |
| 技能需求清单 | 技术负责人 | 必须 |

## 执行步骤

### Step 1: 团队规模确定

- [ ] 根据项目范围估算所需人力
- [ ] 确定团队规模和结构
- [ ] 制定人员招聘/调配计划

### Step 2: 核心成员到位

- [ ] 确认项目经理
- [ ] 确认技术负责人
- [ ] 确认产品负责人
- [ ] 确认测试负责人

### Step 3: 角色职责定义

- [ ] 编写角色职责矩阵（RACI）
- [ ] 明确汇报关系
- [ ] 确定决策权限

### Step 4: 团队沟通机制

- [ ] 确定例会时间
- [ ] 确定沟通工具
- [ ] 建立信息同步机制

### Step 5: Kickoff会议

- [ ] 召开项目启动会
- [ ] 介绍项目背景和目标
- [ ] 宣布团队组成和职责

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 团队组织架构图 | 图表 | 项目文档库 |
| RACI矩阵 | 表格 | 项目文档库 |
| Kickoff会议纪要 | 文档 | 项目文档库 |

## 质量标准

- 核心成员到位率 100%
- 角色职责清晰无歧义
- 团队成员已签字确认职责

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 关键岗位人员不足 | 升级至人力资源部门 |
| 技能匹配度不足 | 安排培训或调整人员 |
