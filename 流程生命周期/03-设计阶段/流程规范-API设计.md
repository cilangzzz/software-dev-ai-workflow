# 流程规范：API设计

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | API设计 |
| 所属阶段 | 设计阶段 |
| 流程负责人 | 研发工程师 |
| 前置流程 | 数据库设计 |
| 后置流程 | 设计评审 |

## 流程目的

设计清晰的API接口规范，确保前后端协作顺畅。

## 触发条件

- 数据库设计完成
- 接口需求已明确

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| PRD文档 | 需求分析 | 必须 |
| 数据库设计文档 | 数据库设计 | 必须 |

## 执行步骤

### Step 1: API规划

- [ ] 梳理接口需求
- [ ] 定义接口分组
- [ ] 规划版本策略

### Step 2: 接口设计

- [ ] 定义接口路径
- [ ] 设计请求参数
- [ ] 设计响应格式
- [ ] 定义错误码

### Step 3: 接口文档编写

- [ ] 编写接口说明
- [ ] 提供请求示例
- [ ] 提供响应示例

### Step 4: 接口评审

- [ ] 组织前后端评审
- [ ] 确认接口可行性
- [ ] 修订完善文档

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| API设计文档 | 文档 | 项目文档库 |
| OpenAPI/Swagger文件 | 文件 | 代码仓库 |

## 质量标准

- 遵循RESTful规范
- 命名风格统一
- 文档完整清晰
