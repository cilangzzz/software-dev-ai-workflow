# Skill: gate-check

## 基本信息
- **名称**: gate-check
- **版本**: 1.0.0
- **所属部门**: 项目管理
- **优先级**: P0

## 功能描述
执行阶段质量门控检查，确保交付物符合质量标准。支持需求、设计、开发、测试、发布等各阶段的质量门禁。

## 触发条件
- 命令触发: `/gate-check`
- 自然语言触发:
  - "质量门控检查"
  - "阶段验收检查"
  - "Gate检查"

## 输入参数
| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| stage | string | 是 | 阶段：requirement / design / develop / test / release |
| deliverables | string | 否 | 交付物路径列表 |
| strict | boolean | 否 | 严格模式，默认false |

## 执行流程
1. **加载检查清单** - 根据阶段加载对应检查项
2. **执行自动检查** - 运行自动化检查脚本
3. **收集人工检查项** - 提示需要人工确认的项目
4. **评估结果** - 计算通过率和阻塞项
5. **生成报告** - 输出门控报告

## 各阶段检查项

### 需求阶段
- PRD文档完整性
- 用户故事定义
- 验收标准明确
- 需求评审通过

### 设计阶段
- 设计稿完整性
- 交互说明清晰
- 技术方案评审
- 数据库设计评审

### 开发阶段
- 代码审查通过
- 单元测试覆盖率
- 静态分析通过
- 安全扫描通过

### 测试阶段
- 测试用例覆盖率
- 缺陷修复率
- 性能测试通过
- 安全测试通过

### 发布阶段
- 部署方案确认
- 回滚预案就绪
- 监控配置完成
- 文档更新完成

## 使用示例

**输入**:
```
stage: develop
deliverables: src/
strict: true
```

## 质量标准
- 检查项覆盖率 100%
- 检查执行准确率 ≥ 95%
- 阻塞问题识别准确率 100%

## 依赖工具
- Bash - 执行检查脚本
- Read - 读取检查配置
- Write - 输出报告

## 注意事项
- 门控标准需要与团队协商确定
- 严格模式下所有检查项必须通过
- 建议在CI/CD中集成门控检查
