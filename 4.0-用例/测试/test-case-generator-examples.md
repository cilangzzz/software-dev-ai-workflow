# 测试用例生成器示例

## 用例1：登录功能测试用例

### 场景描述
测试工程师需要为用户登录功能生成完整的测试用例集，覆盖正常、异常、边界和安全场景。

### 使用Skill
- **Skill**: test-case-generator
- **触发方式**: `/test-case-generator` 或自然语言 "生成测试用例"

### 完整提示词
```
/test-case-generator

功能：用户登录
- 支持手机号+密码登录
- 支持验证码登录
- 密码错误5次锁定账户30分钟
- 支持"记住我"功能

来源类型：requirement
测试类型：functional
```

### 输入参数
| 参数名 | 类型 | 必填 | 值 |
|--------|------|------|-----|
| source | string | 是 | "功能：用户登录\n- 支持手机号+密码登录\n- 支持验证码登录\n- 密码错误5次锁定账户30分钟\n- 支持\"记住我\"功能" |
| source_type | string | 否 | "requirement" |
| test_type | string | 否 | "functional" |

### 执行过程

#### 步骤1：内容解析
解析需求内容，识别测试对象：
- 测试对象: 用户登录模块
- 登录方式: 手机号+密码、验证码
- 特殊规则: 密码错误5次锁定30分钟
- 扩展功能: 记住我

#### 步骤2：功能点提取
识别测试对象的功能点：

**核心功能点**：
1. 手机号+密码登录
2. 验证码登录
3. 账户锁定机制
4. 记住我功能

**输入字段**：
- 手机号
- 密码
- 验证码
- 记住我复选框

#### 步骤3：场景分析
分析正常、异常、边界场景：

**正常场景**（5个）：
- 手机号+密码登录成功
- 验证码登录成功
- 记住我功能生效
- 退出登录
- 登录后跳转

**异常场景**（8个）：
- 密码错误提示
- 未注册手机号提示
- 验证码错误
- 验证码过期
- 账户锁定
- 空输入校验
- 格式错误校验
- 验证码重发限制

**边界场景**（3个）：
- 手机号边界值
- 密码长度边界
- 错误次数边界

**安全场景**（2个）：
- SQL注入测试
- XSS测试

#### 步骤4：用例生成
生成结构化测试用例：
- P0核心功能用例: 6个
- P1边界异常用例: 8个
- P2性能安全用例: 4个
- **总计**: 18个

#### 步骤5：优先级排序
按风险和重要性排序：
- P0: 核心功能（登录成功、锁定验证）
- P1: 边界和异常（格式校验、错误提示）
- P2: 性能和安全（响应时间、安全测试）

## 用例2：购物车测试用例

### 场景描述
为购物车功能生成完整的测试用例，覆盖核心操作和边界场景。

### 使用Skill
- **Skill**: test-case-generator
- **触发方式**: `/test-case-generator` 或自然语言 "创建测试案例"

### 完整提示词
```
/test-case-generator

功能：购物车
- 添加商品到购物车
- 修改商品数量
- 删除商品
- 选择/取消选择商品
- 查看商品详情
- 去结算

来源类型：requirement
```

### 输入参数
| 参数名 | 类型 | 必填 | 值 |

| source | string | 是 | "功能：购物车\n- 添加商品到购物车\n- 修改商品数量\n- 删除商品\n- 选择/取消选择商品\n- 查看商品详情\n- 去结算" |
| source_type | string | 否 | "requirement" |

### 执行过程

#### 步骤1：内容解析
解析需求内容，识别测试对象：
- 测试对象: 购物车模块
- 核心操作: 添加、修改、删除、选择、结算

#### 步骤2：功能点提取
识别测试对象的功能点：

**核心功能点**：
1. 添加商品到购物车
2. 修改商品数量（增加/减少）
3. 删除商品
4. 全选/取消全选
5. 单选商品
6. 去结算

**关联功能**：
- 库存校验
- 价格计算
- 数据持久化

#### 步骤3：场景分析
分析正常、异常、边界场景：

**正常场景**：
- 添加商品成功
- 修改数量成功
- 删除商品成功
- 全选/取消全选
- 去结算

**异常场景**：
- 数量超过库存
- 商品下架
- 商品售罄
- 未登录添加

**边界场景**：
- 数量输入非法值
- 重复添加商品
- 不同规格商品

#### 步骤4：用例生成
生成结构化测试用例：
- P0核心功能用例: 8个
- P1边界异常用例: 8个
- P2性能体验用例: 4个
- **总计**: 20个

#### 步骤5：优先级排序
按风险和重要性排序：
- P0: 核心操作（添加、删除、结算）
- P1: 边界和异常（库存、下架、售罄）
- P2: 性能和体验（大量商品、跨设备同步）
