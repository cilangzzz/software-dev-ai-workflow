# 流程规范：需求采集

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 需求采集 |
| 所属阶段 | 需求分析阶段 |
| 流程负责人 | 产品经理 |
| 前置流程 | 项目启动 |
| 后置流程 | 需求分析 |

## 流程目的

全面、系统地收集业务需求，确保不遗漏关键需求点。

## 触发条件

- 项目启动阶段完成
- 干系人已识别

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 项目章程 | 项目立项 | 必须 |
| 干系人登记表 | 项目启动 | 必须 |
| 业务背景资料 | 业务方 | 建议 |

## 执行步骤

### Step 1: 需求采集规划

- [ ] 确定需求采集范围
- [ ] 制定需求采集计划
- [ ] 准备采集工具和模板

### Step 2: 干系人访谈

- [ ] 与业务方进行深度访谈
- [ ] 记录业务场景和痛点
- [ ] 收集期望和约束条件

### Step 3: 竞品分析

- [ ] 识别主要竞品
- [ ] 分析竞品功能特点
- [ ] 提取可借鉴的功能点

### Step 4: 需求整理

- [ ] 整理原始需求记录
- [ ] 去重和分类
- [ ] 形成需求清单

### Step 5: 需求确认

- [ ] 与干系人确认需求理解
- [ ] 标注疑问和待确认项
- [ ] 补充缺失信息

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 需求采集记录 | 文档 | 项目文档库 |
| 干系人访谈纪要 | 文档 | 项目文档库 |
| 需求清单 | 表格 | 项目文档库 |

## 质量标准

- 覆盖所有核心干系人
- 需求采集完整率 ≥ 95%
- 疑问项已明确标注

## 相关Skill

| Skill | 用途 |
|-------|------|
| requirement-analyzer | 需求整理和分析 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 干系人时间冲突 | 提前预约，多次沟通 |
| 需求模糊 | 追问具体场景 |
| 需求冲突 | 记录并升级决策 |
