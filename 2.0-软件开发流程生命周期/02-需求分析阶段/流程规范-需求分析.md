# 流程规范：需求分析

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 需求分析 |
| 所属阶段 | 需求分析阶段 |
| 流程负责人 | 产品经理 |
| 前置流程 | 需求采集 |
| 后置流程 | 需求评审 |

## 流程目的

将采集的需求转化为结构化的产品需求文档，明确功能范围和优先级。

## 触发条件

- 需求采集完成
- 需求清单已整理

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 需求采集记录 | 需求采集 | 必须 |
| 需求清单 | 需求采集 | 必须 |

## 执行步骤

### Step 1: 需求分类

- [ ] 按功能模块分类
- [ ] 区分功能需求和非功能需求
- [ ] 识别边界条件和异常场景

### Step 2: 需求优先级排序

- [ ] 评估业务价值
- [ ] 评估实现复杂度
- [ ] 确定优先级（P0/P1/P2）

### Step 3: PRD文档编写

- [ ] 编写需求概述
- [ ] 编写功能详细说明
- [ ] 编写非功能需求
- [ ] 编写数据需求

### Step 4: 用户故事拆分

- [ ] 将需求拆分为用户故事
- [ ] 确保符合INVEST原则
- [ ] 编写验收标准

### Step 5: 需求追溯矩阵

- [ ] 建立需求与用户故事的映射
- [ ] 建立需求与测试用例的映射

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 产品需求文档(PRD) | 文档 | 项目文档库 |
| 用户故事列表 | 文档 | 项目文档库 |
| 需求追溯矩阵 | 表格 | 项目文档库 |

## 质量标准

- PRD文档结构完整
- 用户故事符合INVEST原则
- 验收标准明确可测试

## 相关Skill

| Skill | 用途 |
|-------|------|
| requirement-analyzer | 生成PRD框架 |
| user-story-generator | 生成用户故事 |
| acceptance-criteria-writer | 编写验收标准 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 需求不明确 | 返回需求采集补充 |
| 技术可行性存疑 | 邀请技术负责人评估 |
