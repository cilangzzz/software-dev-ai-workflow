# 流程规范：环境准备

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 环境准备 |
| 所属阶段 | 项目启动阶段 |
| 流程负责人 | 技术负责人 |
| 前置流程 | 团队组建 |
| 后置流程 | 需求分析阶段 |

## 流程目的

准备项目开发、测试、部署所需的基础环境和工具，确保团队可以正常开展工作。

## 触发条件

- 团队核心成员已到位
- 技术方向已初步确定

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 技术选型初步方案 | 技术负责人 | 建议 |
| 环境需求清单 | 各负责人 | 必须 |

## 执行步骤

### Step 1: 开发环境准备

- [ ] 创建代码仓库
- [ ] 配置分支策略
- [ ] 准备开发工具（IDE、调试工具等）
- [ ] 配置代码规范检查工具

### Step 2: 项目管理工具配置

- [ ] 创建项目管理空间
- [ ] 配置任务看板
- [ ] 设置里程碑和迭代

### Step 3: 协作工具配置

- [ ] 配置文档协作工具
- [ ] 配置即时通讯群组
- [ ] 配置邮件通知

### Step 4: 测试环境准备

- [ ] 申请测试服务器资源
- [ ] 配置测试数据库
- [ ] 准备测试数据

### Step 5: CI/CD基础配置

- [ ] 配置持续集成流水线
- [ ] 配置构建环境
- [ ] 配置制品仓库

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 环境配置清单 | 文档 | 项目文档库 |
| 工具访问地址 | 文档 | 项目文档库 |
| 环境验证报告 | 文档 | 项目文档库 |

## 质量标准

- 所有环境可正常访问
- CI/CD流水线可正常执行
- 团队成员权限配置正确

## 相关Skill

| Skill | 用途 |
|-------|------|
| scaffold | 项目脚手架初始化 |
| ci-pipeline-assistant | CI/CD配置辅助 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 资源申请受阻 | 升级至管理层协调 |
| 工具权限问题 | 联系IT支持 |
