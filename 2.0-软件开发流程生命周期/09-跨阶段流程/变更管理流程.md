# 变更管理流程

## 流程概述

变更管理流程贯穿项目全生命周期，确保所有变更可控、可追溯。

## 变更类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 需求变更 | 功能需求的增删改 | 新增功能、修改需求 |
| 设计变更 | 技术方案的调整 | 架构调整、技术选型变更 |
| 资源变更 | 人员或资源的调整 | 人员变动、预算调整 |
| 进度变更 | 时间节点的调整 | 里程碑延期、迭代调整 |

## 变更流程

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  变更申请 │──▶│  影响评估 │──▶│  变更评审 │──▶│  变更执行 │──▶│  变更验证 │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

## 执行步骤

### Step 1: 变更申请

- [ ] 提交变更申请单
- [ ] 说明变更原因
- [ ] 说明变更内容
- [ ] 说明期望时间

### Step 2: 影响评估

- [ ] 评估对进度的影响
- [ ] 评估对成本的影响
- [ ] 评估对质量的影响
- [ ] 评估对范围的影响

### Step 3: 变更评审

- [ ] 组织变更评审会议
- [ ] 讨论变更必要性
- [ ] 评估替代方案
- [ ] 决策变更是否接受

### Step 4: 变更执行

- [ ] 更新相关文档
- [ ] 通知相关干系人
- [ ] 执行变更内容

### Step 5: 变更验证

- [ ] 验证变更结果
- [ ] 更新变更记录
- [ ] 关闭变更工单

## 变更审批权限

| 变更影响 | 审批级别 |
|---------|---------|
| 轻微影响 | 项目经理审批 |
| 中等影响 | 部门负责人审批 |
| 重大影响 | 项目总监审批 |

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 变更申请单 | 文档 | 变更管理系统 |
| 影响评估报告 | 文档 | 项目文档库 |
| 变更评审记录 | 文档 | 项目文档库 |
