# 流程规范：发布规划

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 发布规划 |
| 所属阶段 | 部署阶段 |
| 流程负责人 | 运维负责人 |
| 前置流程 | 测试阶段完成 |
| 后置流程 | 部署执行 |

## 流程目的

制定详细的发布计划，确保发布过程可控、风险可控。

## 触发条件

- UAT验收通过
- 发布版本已确定

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 测试报告 | 测试阶段 | 必须 |
| 发布内容 | 产品经理 | 必须 |
| 运维需求 | 技术负责人 | 必须 |

## 执行步骤

### Step 1: 发布内容确认

- [ ] 确认发布版本号
- [ ] 确认发布功能列表
- [ ] 确认发布时间窗口

### Step 2: 发布策略制定

- [ ] 确定发布方式（全量/灰度）
- [ ] 确定发布批次
- [ ] 制定发布时间表

### Step 3: 风险评估

- [ ] 评估发布风险
- [ ] 制定应对措施
- [ ] 准备回滚方案

### Step 4: 资源准备

- [ ] 确认运维人员
- [ ] 确认监控告警
- [ ] 确认值班安排

### Step 5: 计划评审

- [ ] 组织发布计划评审
- [ ] 获取各方确认
- [ ] 正式发布通知

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 发布计划 | 文档 | 项目文档库 |
| 发布通知 | 文档 | 项目文档库 |

## 质量标准

- 发布时间窗口合理
- 回滚方案完整
- 相关方已通知

## 相关Skill

| Skill | 用途 |
|-------|------|
| deploy-analyzer | 部署方案分析 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 发布窗口冲突 | 调整发布时间 |
| 关键人员请假 | 安排替代人员 |
