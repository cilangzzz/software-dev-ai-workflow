# 流程规范：测试报告

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 测试报告 |
| 所属阶段 | 测试阶段 |
| 流程负责人 | 测试负责人 |
| 前置流程 | 测试执行完成 |
| 后置流程 | UAT验收 |

## 流程目的

汇总测试结果，评估系统质量，为发布决策提供依据。

## 触发条件

- 测试执行完成
- 缺陷已处理

## 输入物

| 输入物 | 来源 | 必要性 |
|--------|------|--------|
| 测试执行记录 | 测试执行 | 必须 |
| 缺陷报告 | 缺陷管理 | 必须 |
| 测试计划 | 测试规划 | 必须 |

## 执行步骤

### Step 1: 数据汇总

- [ ] 汇总测试用例执行情况
- [ ] 汇总缺陷统计
- [ ] 汇总测试覆盖率

### Step 2: 质量评估

- [ ] 评估功能完整性
- [ ] 评估系统稳定性
- [ ] 评估性能指标

### Step 3: 风险分析

- [ ] 识别遗留问题
- [ ] 评估发布风险
- [ ] 提出风险应对建议

### Step 4: 报告编写

- [ ] 编写测试报告
- [ ] 附上测试数据
- [ ] 给出测试结论

### Step 5: 报告评审

- [ ] 组织报告评审
- [ ] 确认测试结论
- [ ] 获取签字确认

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 测试执行报告 | 文档 | 项目文档库 |
| 测试覆盖率报告 | 文档 | 项目文档库 |

## 质量标准

- 数据准确完整
- 结论有数据支撑
- 风险提示清晰

## 相关Skill

| Skill | 用途 |
|-------|------|
| test-executor | 测试报告生成 |

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 测试未通过 | 详细说明问题并建议延期 |
| 严重缺陷未修复 | 阻止发布 |
