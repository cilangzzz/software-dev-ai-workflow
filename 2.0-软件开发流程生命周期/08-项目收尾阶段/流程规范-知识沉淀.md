# 流程规范：知识沉淀

## 基本信息

| 属性 | 值 |
|------|-----|
| 流程名称 | 知识沉淀 |
| 所属阶段 | 项目收尾阶段 |
| 流程负责人 | 项目经理 |
| 前置流程 | 项目验收 |
| 后置流程 | 项目复盘 |

## 流程目的

整理和沉淀项目知识资产，为后续项目提供参考。

## 触发条件

- 项目验收通过

## 执行步骤

### Step 1: 知识盘点

- [ ] 盘点项目文档
- [ ] 盘点技术资产
- [ ] 盘点经验教训

### Step 2: 知识整理

- [ ] 整理项目文档
- [ ] 编写技术总结
- [ ] 编写最佳实践

### Step 3: 知识入库

- [ ] 归档项目文档
- [ ] 更新知识库
- [ ] 分享团队知识

## 知识资产分类

| 类型 | 说明 |
|------|------|
| 项目文档 | PRD、设计文档、测试报告等 |
| 技术资产 | 代码库、工具脚本、配置模板等 |
| 经验教训 | 项目总结、最佳实践、避坑指南等 |

## 输出物

| 输出物 | 格式 | 存放位置 |
|--------|------|---------|
| 知识沉淀清单 | 文档 | 项目文档库 |
| 技术总结 | 文档 | 知识库 |

## 质量标准

- 知识资产分类清晰
- 文档完整可读
- 知识库已更新
