# 需求分析器用例

## 用例1：用户反馈转PRD

### 输入
```
用户反馈：希望能在手机上查看订单状态，有时候在外面也需要确认订单走到哪一步了，
还有就是支付的时候老是报错，特别是用微信支付的时候。有时候想取消订单但找不到入口。
```

### 输出
```markdown
# 产品需求文档

## 需求概述
用户需要在移动端查看订单状态、解决微信支付报错问题、以及便捷的取消订单入口。

## 背景与目标
- **背景**: 用户外出时无法及时查看订单进度，支付体验不佳，取消订单操作复杂
- **目标**: 提升用户移动端订单管理体验，降低客服咨询量

## 用户故事

### 故事1: 移动端订单查询
作为外出用户，我希望能在手机上查看订单状态，以便随时了解订单进度。

### 故事2: 微信支付修复
作为支付用户，我希望微信支付能正常工作，以便顺利完成订单。

### 故事3: 取消订单入口
作为购物用户，我希望能够便捷地取消订单，以便处理不需要的商品。

## 功能需求

### 核心功能
- 移动端订单列表页面（显示订单编号、时间、金额、状态）
- 订单详情页（显示订单进度节点）
- 微信支付异常修复
- 取消订单入口优化

### 边界条件
- 支持iOS和Android
- 订单状态实时更新（延迟<3秒）
- 支持断网情况下查看缓存订单
- 取消订单需在发货前操作

## 非功能需求
- 性能要求: 订单列表加载<2秒
- 安全要求: 订单信息需登录验证
- 兼容性: 支持主流手机浏览器

## 待澄清问题
1. 订单状态具体包含哪些节点？
2. 微信支付报错的具体错误信息是什么？
3. 取消订单是否需要确认弹窗？
4. 取消后退款流程如何处理？

## 建议优先级
- P0: 微信支付异常修复（影响交易转化）
- P1: 移动端订单查询（用户体验）
- P1: 取消订单入口优化（用户体验）
- P2: 订单推送通知（增强功能）
```

## 用例2：竞品功能分析

### 输入
```
竞品分析：竞品A的购物车可以保存30天，支持跨设备同步。
竞品B有一键清空购物车功能，还可以分享购物车给好友。
我们的购物车只保存7天，不支持跨设备同步。
```

### 输出
```markdown
# 竞品功能分析报告

## 对比概览
| 功能 | 我们的产品 | 竞品A | 竞品B |
|------|-----------|-------|-------|
| 购物车保存时长 | 7天 | 30天 | 15天 |
| 跨设备同步 | ❌ | ✅ | ✅ |
| 一键清空 | ❌ | ❌ | ✅ |
| 分享购物车 | ❌ | ❌ | ✅ |

## 差距分析

### 核心差距
1. **购物车保存时长不足** - 用户可能因出差/旅行导致购物车清空
2. **缺少跨设备同步** - 用户换设备后购物车丢失
3. **缺少便捷操作** - 无一键清空和分享功能

### 影响评估
- 购物车保存时长: 预计影响15%用户的购买决策
- 跨设备同步: 预计影响20%多设备用户
- 一键清空: 便捷功能，影响较小

## 功能建议

### P0 - 必须实现
1. 购物车保存时长延长至30天
   - 技术成本: 低
   - 开发工期: 1天
   - 收益: 减少15%购物车丢失

### P1 - 建议实现
2. 跨设备同步
   - 技术成本: 中
   - 开发工期: 5天
   - 收益: 提升多设备用户体验

### P2 - 可选实现
3. 一键清空购物车
   - 技术成本: 低
   - 开发工期: 0.5天
   - 收益: 操作便捷性提升

4. 分享购物车
   - 技术成本: 中
   - 开发工期: 3天
   - 收益: 社交传播，潜在获客
```

## 用例3：会议记录转需求

### 输入
```
会议记录：
产品会讨论了会员体系改造：
1. 普通会员升级为铜牌会员需要消费满500元
2. 银牌会员需要消费满2000元或邀请5个好友注册
3. 金牌会员需要消费满5000元或邀请20个好友注册
4. 不同等级享受不同折扣：铜牌95折，银牌9折，金牌85折
5. 还要做一个生日特权，所有会员生日当月有额外优惠券
```

### 输出
```markdown
# 产品需求文档 - 会员体系改造

## 需求概述
重构会员等级体系，建立多维度升级条件，配套差异化权益和生日特权。

## 会员等级规则

| 等级 | 升级条件 | 享受折扣 |
|------|----------|----------|
| 普通会员 | 注册用户 | 无 |
| 铜牌会员 | 累计消费≥500元 | 95折 |
| 银牌会员 | 累计消费≥2000元 或 邀请≥5人注册 | 9折 |
| 金牌会员 | 累计消费≥5000元 或 邀请≥20人注册 | 85折 |

## 生日特权
- 适用对象: 所有会员等级
- 特权内容: 生日当月获得专属优惠券
- 优惠券类型: 待定义（满减券？折扣券？）

## 功能需求

### 会员等级模块
- 等级计算逻辑（消费金额累计 + 邀请人数统计）
- 等级升降自动处理
- 等级进度展示（距离下一等级还需多少）

### 权益模块
- 折扣自动计算应用
- 生日优惠券自动发放
- 权益到期提醒

### 邀请模块
- 邀请链接生成
- 邀请记录统计
- 邀请成功通知

## 待澄清问题
1. 生日优惠券具体金额/类型？
2. 会员等级是否有有效期？
3. 降级规则如何定义？
4. 邀请的好友是否需要完成首单才算有效邀请？

## 建议优先级
- P0: 等级计算逻辑、折扣应用
- P1: 邀请模块、等级进度展示
- P2: 生日特权、权益提醒
```
